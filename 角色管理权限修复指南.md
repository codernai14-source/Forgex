# 角色管理页面按钮权限缺失问题修复指南

## 问题现象

角色管理页面的以下按钮不显示：
- 编辑按钮
- 删除按钮
- 菜单授权按钮
- 用户授权按钮

## 问题原因

**系统管理员角色（admin）在数据库中没有正确关联角色管理的按钮权限。**

权限验证流程：
1. 前端通过 `v-permission="'sys:role:edit'"` 等指令控制按钮显示
2. 权限数据来自用户的 Sa-Token Session
3. Session 中的权限通过三表联查获取：
   - `sys_user_role_perm`（用户 - 角色关联表）
   - `sys_role_menu_perm`（角色 - 菜单权限关联表）
   - `sys_menu_perm`（菜单权限表）

## 修复步骤

### 步骤 1：连接数据库

使用数据库客户端（如 DBeaver、Navicat、DataGrip 等）连接到开发数据库。

### 步骤 2：执行诊断查询

打开 `fix_role_permissions.sql` 文件，执行以下步骤：

#### 2.1 查看租户信息
```sql
SELECT id, tenant_name, tenant_code FROM sys_tenant;
```
记录你的租户 ID（通常是第一个租户）。

#### 2.2 查看角色管理菜单和按钮
```sql
SELECT 
    id,
    parent_id,
    name,
    path,
    perm_key,
    type,
    order_num
FROM sys_menu_perm
WHERE perm_key LIKE 'sys:role:%'
   OR path = 'role'
ORDER BY parent_id, order_num;
```

**预期结果应该包含 6 条记录：**
| id | parent_id | name | perm_key | type |
|----|-----------|------|----------|------|
| X | 0 | 角色管理 | sys:role:view | menu |
| X+1 | X | 新增角色 | sys:role:add | button |
| X+2 | X | 编辑角色 | sys:role:edit | button |
| X+3 | X | 删除角色 | sys:role:delete | button |
| X+4 | X | 菜单授权 | sys:role:authMenu | button |
| X+5 | X | 用户授权 | sys:role:authUser | button |

如果缺少这些记录，说明菜单权限数据本身就有问题，需要重新执行系统初始化。

#### 2.3 查看系统管理员角色
```sql
SELECT id, role_name, role_key, tenant_id
FROM sys_role
WHERE role_key = 'admin';
```
记录 admin 角色的 ID。

#### 2.4 检查 admin 角色的权限
将下面的 `{admin_role_id}` 和 `{tenant_id}` 替换为实际值后执行：

```sql
SELECT 
    m.perm_key,
    m.name,
    m.type,
    rmp.role_id
FROM sys_role_menu_perm rmp
JOIN sys_menu_perm m ON rmp.menu_id = m.id
WHERE rmp.role_id = {admin_role_id}
  AND rmp.tenant_id = {tenant_id}
  AND m.perm_key LIKE 'sys:role:%';
```

**如果查询结果缺少以下任何权限，则需要执行修复脚本：**
- sys:role:view (角色管理菜单)
- sys:role:add (新增按钮)
- sys:role:edit (编辑按钮)
- sys:role:delete (删除按钮)
- sys:role:authMenu (菜单授权按钮)
- sys:role:authUser (用户授权按钮)

### 步骤 3：执行修复脚本

如果步骤 2.4 的查询结果缺少权限，执行以下修复脚本：

**请将以下占位符替换为实际 ID：**
- `{admin_role_id}`: admin 角色 ID
- `{tenant_id}`: 租户 ID
- `{role_menu_id}`: 角色管理菜单 ID（perm_key='sys:role:view' 的记录）
- `{role_add_button_id}`: 新增按钮 ID（perm_key='sys:role:add'）
- `{role_edit_button_id}`: 编辑按钮 ID（perm_key='sys:role:edit'）
- `{role_delete_button_id}`: 删除按钮 ID（perm_key='sys:role:delete'）
- `{role_auth_menu_button_id}`: 菜单授权按钮 ID（perm_key='sys:role:authMenu'）
- `{role_auth_user_button_id}`: 用户授权按钮 ID（perm_key='sys:role:authUser'）

```sql
-- 添加角色管理菜单权限
INSERT INTO sys_role_menu_perm (role_id, menu_id, tenant_id, created_at, updated_at)
SELECT {admin_role_id}, {role_menu_id}, {tenant_id}, NOW(), NOW()
WHERE NOT EXISTS (
    SELECT 1 FROM sys_role_menu_perm 
    WHERE role_id = {admin_role_id} 
      AND menu_id = {role_menu_id} 
      AND tenant_id = {tenant_id}
);

-- 添加新增角色按钮权限
INSERT INTO sys_role_menu_perm (role_id, menu_id, tenant_id, created_at, updated_at)
SELECT {admin_role_id}, {role_add_button_id}, {tenant_id}, NOW(), NOW()
WHERE NOT EXISTS (
    SELECT 1 FROM sys_role_menu_perm 
    WHERE role_id = {admin_role_id} 
      AND menu_id = {role_add_button_id} 
      AND tenant_id = {tenant_id}
);

-- 添加编辑角色按钮权限

-- 添加删除角色按钮权限

-- 添加菜单授权按钮权限

-- 添加用户授权按钮权限

SELECT {admin_role_id}, {role_auth_user_button_id}, {tenant_id}, NOW(), NOW()
WHERE NOT EXISTS (
    SELECT 1 FROM sys_role_menu_perm 
    WHERE role_id = {admin_role_id} 
      AND menu_id = {role_auth_user_button_id} 
      AND tenant_id = {tenant_id}

```

### 步骤 4：验证修复结果

重新执行步骤 2.4 的查询，确认现在包含了所有 6 个权限。

### 步骤 5：清除缓存并重新登录（重要！）

权限数据缓存在 Sa-Token Session 中，需要：

1. **清除 Redis 缓存**（在 Redis 客户端执行）：
   ```bash
   # 查找 admin 用户的 Session 键
   KEYS *{admin_user_id}*
   
   # 删除对应的 Session（如果有多个，全部删除）
   DEL sa_session:{session_id}
   ```

2. **重新登录系统**：
   - 退出登录
   - 清除浏览器缓存（Ctrl+Shift+Delete）
   - 重新登录

3. **刷新页面**：
   - 按 Ctrl+F5 强制刷新
   - 或者清除浏览器缓存后刷新

### 步骤 6：验证按钮显示

刷新页面后，角色管理页面应该显示以下按钮：
- ✅ 新增角色名称（橙色按钮）
- ✅ 编辑（蓝色链接按钮）
- ✅ 菜单授权（蓝色链接按钮）
- ✅ 用户授权（蓝色链接按钮）
- ✅ 删除（红色链接按钮）

## 快速修复脚本（推荐）

如果你想快速修复，可以直接执行以下脚本（需要根据实际情况替换 ID）：

```sql
-- 1. 先查询所有需要的 ID
SELECT 
    r.id AS admin_role_id,
    r.role_name,
    (SELECT id FROM sys_menu_perm WHERE path = 'role' AND tenant_id = r.tenant_id) AS role_menu_id,
    (SELECT id FROM sys_menu_perm WHERE perm_key = 'sys:role:add' AND tenant_id = r.tenant_id) AS add_button_id,
    (SELECT id FROM sys_menu_perm WHERE perm_key = 'sys:role:edit' AND tenant_id = r.tenant_id) AS edit_button_id,
    (SELECT id FROM sys_menu_perm WHERE perm_key = 'sys:role:delete' AND tenant_id = r.tenant_id) AS delete_button_id,
    (SELECT id FROM sys_menu_perm WHERE perm_key = 'sys:role:authMenu' AND tenant_id = r.tenant_id) AS auth_menu_button_id,
    (SELECT id FROM sys_menu_perm WHERE perm_key = 'sys:role:authUser' AND tenant_id = r.tenant_id) AS auth_user_button_id
FROM sys_role r
WHERE r.role_key = 'admin'
LIMIT 1;

-- 2. 根据查询结果，手动替换下面的 ID 后执行修复脚本
```

## 常见问题

### Q1: 执行修复脚本后按钮还是不显示？

**A:** 可能是缓存问题，请确保：
1. 清除了 Redis 中的 Session 缓存
2. 重新登录了系统
3. 强制刷新了页面（Ctrl+F5）

### Q2: 步骤 2.2 查询不到角色管理的菜单记录？

**A:** 说明菜单数据本身就有问题，需要：
1. 检查系统初始化是否成功执行
2. 或者手动在 `sys_menu_perm` 表中插入角色管理菜单和按钮记录

### Q3: 我是多租户系统，需要为每个租户都执行修复吗？

**A:** 是的，每个租户的权限数据是独立的，需要为每个租户的 admin 角色都执行修复脚本。

## 技术说明

### 权限数据结构

```
sys_user (用户表)
  ↓ (sys_user_role_perm)
sys_role (角色表)
  ↓ (sys_role_menu_perm)
sys_menu_perm (菜单权限表)
  ↓ (perm_key)
前端 v-permission 指令
```

### 前端权限控制

```vue
<a-button
  type="link"
  size="small"
  @click="openEdit(record)"
  v-permission="'sys:role:edit'"
>
  {{ $t('common.edit') }}
</a-button>
```

### 后端权限验证

```java
@RequirePerm("sys:role:edit")
@PostMapping("/update")
public R<Void> update(@RequestBody @Validated SysRoleDTO roleDTO) {
    // ...
}
```
