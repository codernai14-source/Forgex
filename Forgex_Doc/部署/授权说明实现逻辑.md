# 授权说明实现逻辑

> 分类：部署  
> 版本：**V0.6.1**  
> 更新时间：**2026-04-22**

本文档说明 Forgex 授权文件的生成、装载与校验链路，内容贴合 Forgex_Build/license-tools 目录下的实际实现。

## 一、授权主链路

```text
系统启动
  -> 读取 license-dir 配置
  -> 若无 request-info.json 则自动生成
  -> 授权签发端根据 request-info.json 生成 license.lic
  -> license.lic 放回授权目录
  -> 网关校验授权状态
  -> 放行或阻断受保护接口
```

## 二、授权工具架构

Forgex_Build/license-tools 目录包含授权相关的所有工具：

| 目录/文件 | 说明 |
|---|---|
| `issuer/FxLicenseGenerator/` | 授权签发工具，用于授权中心生成正式授权 |
| `request-client/FxLicenseRequest/` | 现场请求授权客户端，用于生成 request-info.json 和导入 license.lic |
| `FxLicenseCore/` | 授权核心库，包含模型、服务和工具类 |
| `activate-dev-license.bat` | 开发环境一键授权脚本 |
| `repair-dev-license.ps1` | 开发授权修复脚本 |
| `collect-license-diagnostics.ps1` | 授权诊断收集脚本 |
| `keys/` | 生产密钥对目录 |
| `dev-local-keys/` | 开发密钥对目录 |

### 2.1 FxLicenseGenerator 授权签发工具

FxLicenseGenerator 是 C# 实现的 CLI 工具，提供两个核心命令：

#### gen-keypair 命令

生成 Ed25519 密钥对：

```bash
dotnet run --project issuer/FxLicenseGenerator -- gen-keypair --out-dir ./keys
```

输出文件：
- `public-key.base64`：公钥文件，用于后端验签
- `private-key.pkcs8.base64`：私钥文件，用于授权签发

#### issue 命令

签发授权文件：

```bash
dotnet run --project issuer/FxLicenseGenerator -- issue \
  --request-info D:/mine_product/forgex/forgex/license/request-info.json \
  --private-key ./keys/private-key.pkcs8.base64 \
  --output D:/mine_product/forgex/forgex/license/license.lic \
  --customer-name 本机开发环境 \
  --edition dev \
  --modules gateway,auth,sys,basic,job,integration,workflow,report \
  --duration-days 365
```

可选参数列表：

| 参数 | 说明 | 默认值 |
|---|---|---|
| `--output` | license.lic 输出路径 | request-info.json 同目录 |
| `--license-id` | 授权编号 | 自动生成 LIC-{UUID} |
| `--product` | 产品名称 | Forgex |
| `--edition` | 版本类型 | standard |
| `--customer-name` | 客户名称 | 客户码 |
| `--modules` | 授权模块列表，逗号分隔 | gateway,auth,sys,basic,job,integration,workflow,report |
| `--max-users` | 最大用户数 | 无限制 |
| `--max-tenants` | 最大租户数 | 无限制 |
| `--effective-at` | 生效时间（ISO_OFFSET_DATE_TIME） | 当前时间 |
| `--expire-at` | 到期时间（ISO_OFFSET_DATE_TIME） | 生效时间 + duration-days |
| `--duration-days` | 授权天数 | 365 |
| `--grace-days` | 宽限期天数 | 0 |
| `--issue-serial` | 签发序号 | 1 |
| `--remark` | 备注 | 空 |

### 2.2 FxLicenseRequest 现场请求客户端

FxLicenseRequest 是 C# 实现的 CLI 工具，提供三个核心命令：

#### generate-request 命令

生成请求授权文件：

```bash
dotnet run --project request-client/FxLicenseRequest -- generate-request \
  --output D:/mine_product/forgex/forgex/license/request-info.json \
  --instance-code DEV_LOCAL \
  --edition dev
```

可选参数：

| 参数 | 说明 | 默认值 |
|---|---|---|
| `--instance-code` | 实例编码 | DEFAULT |
| `--customer-code-prefix` | 客户码前缀 | FXC |
| `--machine-code` | 覆盖机器码 | 自动采集 |
| `--product` | 产品名称 | Forgex |
| `--edition` | 版本类型 | standard |

#### print-machine-code 命令

输出当前机器码：

```bash
dotnet run --project request-client/FxLicenseRequest -- print-machine-code
```

可用于验证机器码采集结果，或指定自定义机器码。

#### import-license 命令

导入授权文件：

```bash
dotnet run --project request-client/FxLicenseRequest -- import-license \
  --license-file D:/mine_product/forgex/forgex/license/license.lic \
  --target-dir D:/mine_product/forgex/forgex/license
```

或通过授权字符串导入：

```bash
dotnet run --project request-client/FxLicenseRequest -- import-license \
  --activation-code "<授权字符串>" \
  --target-dir D:/mine_product/forgex/forgex/license
```

可选参数：

| 参数 | 说明 | 默认值 |
|---|---|---|
| `--license-file-name` | 授权文件名 | license.lic |
| `--history-file-name` | 历史文件名 | activation-history.json |

### 2.3 FxLicenseCore 核心库

FxLicenseCore 包含授权相关的核心模型、服务和工具类：

#### 模型定义（Models）

**LicenseRequestInfo**：请求授权信息模型

```text
- Product：产品名称
- Edition：版本类型
- CustomerCode：客户码（由 instance-code + machine-code 计算生成）
- MachineCode：机器码（SHA-256 摘要）
- OsType：操作系统类型
- Hostname：主机名
- GeneratedAt：生成时间
- ToolVersion：工具版本
```

**LicensePayload**：授权载荷模型

```text
- LicenseId：授权编号
- Product：产品名称
- Edition：版本类型
- CustomerCode：客户码
- CustomerName：客户名称
- MachineCode：机器码
- Modules：授权模块列表
- MaxUsers：最大用户数（可选）
- MaxTenants：最大租户数（可选）
- IssuedAt：签发时间
- EffectiveAt：生效时间
- ExpireAt：到期时间（可选）
- DurationDays：授权时长（天）
- GraceDays：宽限期（天）
- IssueSerial：签发序号
- Remark：备注
```

**LicenseIssueOptions**：签发选项模型，承载命令行输入参数。

#### 服务类（Services）

**MachineFingerprintService**：机器指纹服务

负责采集稳定机器码，算法如下：

```text
Windows:
  - COMPUTERNAME
  - PROCESSOR_IDENTIFIER
  - SystemDrive
  - MAC 地址列表（排除 Loopback、Tunnel）
  - 主机名

Linux:
  - /etc/machine-id
  - /sys/class/dmi/id/product_uuid
  - HOSTNAME
  - MAC 地址列表
  - 主机名

最终：将所有采集值用 "|" 连接，计算 SHA-256 摘要（十六进制大写）
```

**RequestInfoService**：请求授权信息服务

- 生成 request-info.json 文件
- 读取 request-info.json 文件
- 计算客户码算法：`{customerCodePrefix}-{UUID8位后缀}`，其中 UUID 基于 `{instanceCode}|{machineCode}` 生成

**LicenseIssuingService**：授权签发服务

- 组装 LicensePayload 载荷
- 使用 Ed25519 私钥签名
- 输出 license.lic 文件

**LicenseImportService**：授权导入服务

- 导入 license.lic 文件或授权字符串
- 写入 activation-history.json 历史记录

#### 工具类（Utilities）

**KeyMaterialHelper**：密钥与签名工具类

- `GenerateKeyPair`：生成 Ed25519 密钥对
- `ReadPrivateKey`：读取 PKCS#8 私钥
- `ReadPublicKey`：读取公钥
- `Sign`：Ed25519 签名
- `Verify`：Ed25519 验签
- `EncodeLicense`：组装授权文件文本（格式：`{payload.base64url}.{signature.base64url}`）
- `DecodePayload`：解码授权载荷
- `DecodeSignature`：解码授权签名

## 三、授权文件格式

### 3.1 request-info.json 格式

```json
{
  "product": "Forgex",
  "edition": "standard",
  "customerCode": "FXC-AB12CD34",
  "machineCode": "A1B2C3D4E5F6...",
  "osType": "windows",
  "hostname": "forgex-demo",
  "generatedAt": "2026-04-22T10:00:00+08:00",
  "toolVersion": "1.0.0"
}
```

### 3.2 license.lic 格式

授权文件采用 Base64Url 编码的两段式结构：

```text
{payload_base64url}.{signature_base64url}
```

- payload：LicensePayload JSON 序列化后的 Base64Url 编码
- signature：Ed25519 签名的 Base64Url 编码

### 3.3 activation-history.json 格式

每次导入授权都会追加一条记录：

```json
[
  {
    "activatedAt": "2026-04-22T10:30:00+08:00",
    "licenseId": "LIC-AB12CD34...",
    "customerCode": "FXC-AB12CD34",
    "machineCode": "A1B2C3D4E5F6...",
    "effectiveAt": "2026-04-22T10:00:00+08:00",
    "expireAt": "2027-04-22T10:00:00+08:00",
    "edition": "prod",
    "modules": ["gateway", "auth", "sys", "basic", "job", "integration", "workflow", "report"],
    "licensePath": "/opt/Forgex_PROD/license/license.lic"
  }
]
```

## 四、开发环境授权

### 4.1 一键授权脚本

执行 `activate-dev-license.bat` 可自动完成开发授权：

1. 检查 `FORGEX_PROFILE` 是否为 `dev`，非 dev 环境禁止执行
2. 检查 dotnet SDK 是否安装
3. 构建授权工具解决方案
4. 若开发密钥不存在，自动生成密钥对
5. 若 request-info.json 存在则复用，避免机器码变更
6. 签发开发授权（3650 天，remark 标记 `DEV_ONLY_LOCAL_LICENSE`）
7. 导入授权文件
8. 复制公钥到授权目录
9. 设置用户环境变量

执行方式：

```bat
cd Forgex_Build\license-tools
activate-dev-license.bat
```

### 4.2 授权修复脚本

`repair-dev-license.ps1` 用于修复开发授权问题：

- 自动探测多个候选授权目录
- 选择评分最高的授权目录作为规范目录
- 复制授权文件到规范目录
- 同步授权文件到所有候选目录
- 重新激活授权（可选）

执行方式：

```powershell
cd Forgex_Build\license-tools
powershell -ExecutionPolicy Bypass -File repair-dev-license.ps1
```

可选参数：

| 参数 | 说明 |
|---|---|
| `-DryRun` | 仅预览操作，不实际执行 |
| `-SkipReactivate` | 跳过重新激活授权 |
| `-TargetHome` | 指定目标 FORGEX_HOME |

### 4.3 授权诊断脚本

`collect-license-diagnostics.ps1` 用于收集授权诊断信息：

输出内容：
- FORGEX 相关环境变量（Process/User/Machine 三级）
- Java 进程快照（命令行参数、JVM 配置）
- 候选授权目录状态检查
- 授权文件格式验证

执行方式：

```powershell
cd Forgex_Build\license-tools
powershell -ExecutionPolicy Bypass -File collect-license-diagnostics.ps1
```

输出文件默认保存在桌面：`forgex-license-diagnostics-{timestamp}.md`

## 六、运行期校验

网关会在登录态判断前先校验授权状态：

```text
校验步骤：
  1. 读取 license-dir 下的 license.lic
  2. 使用公钥验证签名
  3. 解析 LicensePayload
  4. 校验机器码匹配
  5. 校验生效时间与到期时间
  6. 计算宽限期状态
  7. 返回授权状态（有效/无效/过期/宽限期）
```

授权状态返回值：
- **有效**：授权在有效期内，机器码匹配
- **无效**：签名验证失败、文件格式错误、机器码不匹配
- **过期**：已超过到期时间且无宽限期
- **宽限期**：已超过到期时间但在宽限期内

## 七、密钥管理

### 7.1 密钥类型

| 密钥 | 文件名 | 用途 |
|---|---|
| 公钥 | `public-key.base64` | 后端验签，分发给所有部署实例 |
| 私钥 | `private-key.pkcs8.base64` | 授权签发，仅授权中心持有 |

### 7.2 密钥目录

| 目录 | 说明 |
|---|---|
| `license-tools/keys/` | 生产密钥对，用于签发正式授权 |
| `license-tools/dev-local-keys/` | 开发密钥对，仅用于开发环境 |

**安全要求**：
- 私钥文件严禁泄露，仅在授权签发端使用
- 开发密钥对生成的授权仅在 `FORGEX_PROFILE=dev` 时生效
- 生产密钥对用于签发 test、yanshi、prod 环境授权

## 十、Windows Control Center 实现补充

`Forgex_Build/tools/control-center/ForgexControlCenter` 是 Windows 安装包内置的桌面控制台。它复用 `FxLicenseCore`，不定义新的授权格式。

授权相关实现：
- 使用 `MachineFingerprintService` 获取客户机器码。
- 使用 `RequestInfoService` 生成 `license/request-info.json`。
- 使用 `LicenseImportService` 导入授权中心返回的 `license.lic`。
- 导入授权时同步维护 `activation-history.json`。

服务控制相关实现：
- 安装脚本生成 `config/forgex-control.json`，记录实例编码、安装目录、中间件地址、服务 JAR、端口和日志目录。
- 控制台优先查找同名 Windows 服务并通过 ServiceController 启停。
- 未注册 Windows 服务时，控制台使用 Java 进程模式启停服务，并记录 pid 文件。
- 启动 Java 服务时注入 `FORGEX_HOME`、`FORGEX_LICENSE_DIR`、`FORGEX_NACOS_ADDR`、`FORGEX_NACOS_NAMESPACE`、`FORGEX_REDIS_ADDR`、`FORGEX_ROCKETMQ_NAME_SERVER`、`FORGEX_MYSQL_URL` 等环境变量。

安全边界：
- 客户安装包只分发 `public-key.base64`。
- `private-key.pkcs8.base64` 仅用于授权中心签发，不进入客户安装包。
