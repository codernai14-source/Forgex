# 授权说明使用方式

> 分类：部署  
> 版本：**V0.6.0**  
> 更新时间：**2026-04-17**

本文档说明开发、测试、演示、生产环境的授权申请、导入与验证方式。

## 开发环境建议

```bash
FORGEX_PROFILE=dev
FORGEX_INSTANCE_CODE=DEV_LOCAL
FORGEX_HOME=D:/mine_product/forgex/forgex
FORGEX_LICENSE_DIR=D:/mine_product/forgex/forgex/license
FORGEX_LICENSE_ENABLED=true
```

步骤：

1. 启动一次系统生成 `request-info.json`
2. 提交授权申请
3. 将 `license.lic` 放入授权目录
4. 调 `POST /api/sys/license/refresh`
5. 用 `GET /api/sys/license/status` 验证

## 各环境原则

| 环境 | 建议 |
|---|---|
| `dev` | 可使用开发专用授权 |
| `test` | 独立测试授权 |
| `yanshi` | 独立演示授权 |
| `prod` | 正式生产授权，最严格 |

## 常见排查

| 现象 | 排查点 |
|---|---|
| 系统提示未授权 | `license.lic` 是否存在、是否匹配机器码 |
| 授权刷新不生效 | 公钥、授权目录、网关服务是否重载 |
| 解析失败 | 文件 BOM、公钥缺失、签名无效 |
