# 授权说明使用方式

> 分类：部署  
> 版本：**V0.6.1**  
> 更新时间：**2026-04-22**

本文档说明开发、测试、演示、生产环境的授权申请、导入与验证方式，内容贴合 Forgex_Build/license-tools 目录下的实际实现。

## 一、环境变量配置

### 1.1 开发环境配置

```text
FORGEX_PROFILE=dev
FORGEX_INSTANCE_CODE=DEV_LOCAL
FORGEX_HOME=D:/mine_product/forgex/forgex
FORGEX_LICENSE_DIR=D:/mine_product/forgex/forgex/license
FORGEX_LICENSE_ENABLED=true
FORGEX_LICENSE_PUBLIC_KEY_FILE_NAME=public-key.base64
```

### 1.2 Linux 生产环境配置

参考 `Forgex_Build/delivery/linux/scripts/.env.template`：

```text
FORGEX_INSTANCE_CODE=ACME_PROD
FORGEX_PROFILE=prod
FORGEX_HOME=/opt/Forgex_ACME_PROD
FORGEX_LICENSE_DIR=/opt/Forgex_ACME_PROD/license
FORGEX_UPLOAD_DIR=/opt/Forgex_ACME_PROD/data/uploads
FORGEX_LOG_DIR=/opt/Forgex_ACME_PROD/logs
FORGEX_BACKUP_DIR=/opt/Forgex_ACME_PROD/backup
```

### 1.3 Windows 安装环境配置

Windows 安装器会自动生成 `config/install-config.yml`：

安装器支持中文 / English 语言选择。前端会在交付包构建阶段执行 `npm run build` 并复制 `dist` 到安装目录的 `frontend/`；Windows 交付包会内置 `nginx/nginx.exe`、`nginx/conf/mime.types` 和 `nginx/forgex.conf.template`，安装时会生成 `nginx/forgex.conf`，用于托管前端静态文件并将 `/api/` 代理到网关。安装完成后可直接打开 `Forgex 控制中心`，在控制中心内生成 `request-info.json`、导入 `license.lic`，通过右上角语言下拉在中文和 English 之间切换，并优先使用安装目录内置 Nginx 启停前端；未检测到 Nginx 时会回退到内置静态 Web 服务。

## 二、授权申请流程

### 2.1 标准授权流程

适用于 test、yanshi、prod 环境：

```text
步骤 1：客户现场生成请求文件
  -> 使用 FxLicenseRequest 生成 request-info.json
  -> 提交给授权签发端

步骤 2：授权签发端签发授权
  -> 使用 FxLicenseGenerator 签发 license.lic
  -> 返回授权文件或授权字符串给客户现场

步骤 3：客户现场导入授权
  -> 使用 FxLicenseRequest 导入 license.lic
  -> 验证授权状态
```

### 2.2 开发环境授权流程

仅适用于 dev 环境，使用一键授权脚本：

```bat
cd Forgex_Build\license-tools
activate-dev-license.bat
```

脚本自动完成以下操作：
1. 检查环境是否为 dev
2. 构建授权工具
3. 生成开发密钥对（首次）
4. 生成/复用 request-info.json
5. 签发开发授权（3650 天）
6. 导入授权并设置环境变量

**重要限制**：开发授权带有 `DEV_ONLY_LOCAL_LICENSE` 标记，仅在 `FORGEX_PROFILE=dev` 时生效。

## 三、授权工具使用详解

### 3.1 构建授权工具

在 `Forgex_Build/license-tools` 目录执行：

```bash
dotnet build FxLicenseTools.sln
```

需要安装 .NET SDK 9。

### 3.2 生成密钥对（授权中心）

首次部署时生成生产密钥对：

```bash
dotnet run --project issuer/FxLicenseGenerator -- gen-keypair --out-dir ./keys
```

输出文件：
- `keys/public-key.base64`：公钥，分发给所有部署实例
- `keys/private-key.pkcs8.base64`：私钥，仅授权中心持有

### 3.3 生成请求文件（客户现场）

客户现场使用请求客户端生成请求文件：

```bash
dotnet run --project request-client/FxLicenseRequest -- generate-request \
  --output /opt/Forgex_PROD/license/request-info.json \
  --instance-code ACME_PROD \
  --edition prod
```

请求文件包含：
- 机器码（自动采集）
- 客户码（自动计算）
- 操作系统信息
- 生成时间

### 3.4 签发授权（授权中心）

授权中心收到 request-info.json 后签发授权：

```bash
dotnet run --project issuer/FxLicenseGenerator -- issue \
  --request-info request-info.json \
  --private-key ./keys/private-key.pkcs8.base64 \
  --output license.lic \
  --customer-name "ACME科技有限公司" \
  --edition prod \
  --modules gateway,auth,sys,basic,job,integration,workflow,report \
  --max-users 100 \
  --max-tenants 5 \
  --duration-days 365 \
  --grace-days 30 \
  --remark "正式授权"
```

### 3.5 导入授权（客户现场）

客户现场收到授权文件后导入：

**方式一：文件导入**

```bash
dotnet run --project request-client/FxLicenseRequest -- import-license \
  --license-file license.lic \
  --target-dir /opt/Forgex_PROD/license
```

**方式二：授权字符串导入**

```bash
dotnet run --project request-client/FxLicenseRequest -- import-license \
  --activation-code "<授权字符串>" \
  --target-dir /opt/Forgex_PROD/license
```

导入后会自动更新 `activation-history.json`。

### 3.6 公钥分发

公钥文件需要分发到所有部署实例的授权目录：

```text
# Linux
cp keys/public-key.base64 /opt/Forgex_PROD/license/public-key.base64

# Windows
copy keys\public-key.base64 D:\Forgex_PROD\license\public-key.base64
```

## 四、授权验证

### 4.1 系统启动验证

系统启动时自动校验授权：
1. 读取 `FORGEX_LICENSE_DIR` 下的 license.lic
2. 使用公钥验证签名
3. 校验机器码匹配
4. 校验时间有效性

### 4.2 API 接口验证

**刷新授权状态**

```http
POST /api/sys/license/refresh
```

重新加载授权文件并更新内存中的授权状态。

**查询授权状态**

```http
GET /api/sys/license/status
```

返回当前授权信息：

```json
{
  "licenseId": "LIC-AB12CD34...",
  "customerCode": "FXC-AB12CD34",
  "customerName": "ACME科技有限公司",
  "edition": "prod",
  "modules": ["gateway", "auth", "sys", ...],
  "effectiveAt": "2026-04-22T10:00:00+08:00",
  "expireAt": "2027-04-22T10:00:00+08:00",
  "graceDays": 30,
  "status": "valid"
}
```

### 4.3 授权诊断

遇到授权问题时，运行诊断脚本：

```powershell
cd Forgex_Build\license-tools
powershell -ExecutionPolicy Bypass -File collect-license-diagnostics.ps1
```

诊断报告包含：
- 环境变量配置（Process/User/Machine 三级）
- Java 进程 JVM 参数
- 候选授权目录文件状态
- 授权文件格式验证

## 五、各环境授权原则

| 环境 | 授权类型 | 签发方式 | 有效期 |
|---|---|---|---|
| `dev` | 开发授权 | activate-dev-license.bat 一键生成 | 3650 天 |
| `test` | 测试授权 | 授权中心正式签发 | 根据需求设定 |
| `yanshi` | 演示授权 | 授权中心正式签发 | 根据需求设定 |
| `prod` | 生产授权 | 授权中心正式签发 | 根据合同设定 |

**重要说明**：
- 开发授权只能在 `FORGEX_PROFILE=dev` 时生效
- test/yanshi/prod 必须使用授权中心签发的正式授权
- 开发密钥与生产密钥完全隔离

## 六、授权修复

### 6.1 开发授权修复

若开发授权出现问题，运行修复脚本：

```powershell
cd Forgex_Build\license-tools
powershell -ExecutionPolicy Bypass -File repair-dev-license.ps1
```

脚本功能：
- 自动探测多个候选授权目录
- 选择评分最高的目录作为规范目录
- 复制授权文件到规范目录
- 同步授权文件到其他候选目录
- 设置正确的环境变量

### 6.2 预览修复操作

使用 DryRun 模式预览：

```powershell
powershell -ExecutionPolicy Bypass -File repair-dev-license.ps1 -DryRun
```

### 6.3 指定目标目录

```powershell
powershell -ExecutionPolicy Bypass -File repair-dev-license.ps1 -TargetHome D:\forgex\forgex
```

## 七、常见问题排查

| 现象 | 排查点 | 解决方案 |
|---|---|---|
| 系统提示未授权 | license.lic 是否存在 | 检查 FORGEX_LICENSE_DIR 配置 |
| 授权验证失败 | 公钥是否正确 | 确保公钥与签发私钥配对 |
| 机器码不匹配 | request-info.json 机器码与授权不匹配 | 重新生成请求文件并签发 |
| 授权刷新不生效 | 网关是否重载 | 重启 Gateway 服务 |
| 解析失败 | 文件 BOM 或格式问题 | 运行诊断脚本检查 |
| 开发授权不生效 | FORGEX_PROFILE 不是 dev | 设置 FORGEX_PROFILE=dev |
| 环境变量混乱 | 多级环境变量冲突 | 运行诊断脚本检查各级别值 |

### 7.1 诊断检查清单

1. **检查环境变量一致性**

```powershell
$env:FORGEX_PROFILE
$env:FORGEX_LICENSE_DIR
$env:FORGEX_LICENSE_ENABLED
```

2. **检查授权文件存在**

```bash
ls $FORGEX_LICENSE_DIR/license.lic
ls $FORGEX_LICENSE_DIR/public-key.base64
ls $FORGEX_LICENSE_DIR/request-info.json
```

3. **检查 Java 进程参数**

```bash
# Linux
ps aux | grep java | grep forgex

# Windows（PowerShell）
Get-CimInstance Win32_Process -Filter "Name='java.exe'" | Select-Object CommandLine
```

确认 JVM 参数 `-Dforgex.deployment.license-dir` 与实际授权目录一致。

4. **运行诊断脚本**

```powershell
powershell -ExecutionPolicy Bypass -File collect-license-diagnostics.ps1
```

## 八、授权目录结构

标准授权目录结构：

```text
license/
├── license.lic              # 授权文件
├── public-key.base64        # 公钥文件
├── request-info.json        # 请求文件
└── activation-history.json  # 导入历史（可选）
```

## 十、Windows Control Center 授权流程

Windows 客户现场优先使用 Forgex Control Center 处理授权，不需要客户手工运行 `FxLicenseRequest` 命令。

流程：
1. 客户安装 Forgex 后，打开桌面快捷方式 `Forgex Control Center`。
2. 点击 `Generate Request`，生成 `license/request-info.json`。
3. 将 `request-info.json` 复制给授权中心。
4. 授权中心使用 `FxLicenseGenerator` 和私钥签发 `license.lic`。
5. 客户现场点击 `Import License`，选择授权中心返回的 `license.lic`。
6. 控制台写入 `license/license.lic`，并维护 `license/activation-history.json`。
7. 通过控制台重启 Forgex Java 服务，使授权状态生效。

命令行方式仍然保留：

```powershell
D:\Forgex_ACME_PROD\tools\control-center\ForgexControlCenter.exe --install-root D:\Forgex_ACME_PROD --status
D:\Forgex_ACME_PROD\tools\control-center\ForgexControlCenter.exe --install-root D:\Forgex_ACME_PROD --start-all
D:\Forgex_ACME_PROD\tools\control-center\ForgexControlCenter.exe --install-root D:\Forgex_ACME_PROD --stop-all
```

客户安装包只包含 `public-key.base64`，不包含 `private-key.pkcs8.base64`。私钥只允许保存在授权签发端。
