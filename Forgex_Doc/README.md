# Forgex 文档中心

> 版本：**预览版 V0.5.0**  
> 更新时间：**2026-04-13**

Forgex V0.5.0 开始统一使用 `Forgex_Doc` 作为文档总入口，配合根目录 `README.md` / `README.en.md` 提供可导航、可维护、可扩展的文档结构。

## 一、目录结构

```text
Forgex_Doc
├─ README.md                文档中心首页
├─ 开发规范/                项目架构设计、规范文档、开发约束
├─ 后端/                    后端公共能力与功能模块手册
├─ 前端/                    前端公共组件与页面能力手册
├─ 数据库/                  脚本索引、设计规范、修复脚本导航
├─ 安卓端/                  Android 公共组件与接入说明（预留）
├─ 部署/                    部署方法与环境说明（预留）
└─ nacos配置/               现有配置资料（保留）
```

## 三、V0.5.0 文档目标

本次版本重点不只是“补文档”，而是建立一套清晰的文档分层：

1. **根 README 负责产品概览与功能总览**。
2. **`Forgex_Doc` 负责面向开发者的正式文档导航**。
3. **`doc/开发规范/04-功能模块` 保留详细模块说明与历史沉淀**。
4. **后续新增功能优先写入 `Forgex_Doc`，再按需下沉到模块级专题文档**。

---

如需继续扩展文档，建议遵循以下规则：

- 产品概览更新：同步修改根目录 `README.md` / `README.en.md`
- 研发规范更新：优先修改 `Forgex_Doc/开发规范`
- 组件/能力更新：优先修改 `Forgex_Doc/后端` 或 `Forgex_Doc/前端`
- 深度专题：按模块沉淀到 `doc/开发规范/04-功能模块`
