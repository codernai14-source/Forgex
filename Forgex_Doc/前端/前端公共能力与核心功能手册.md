# 前端公共能力与核心功能手册

> 版本：**预览版 V0.5.0**  
> 更新时间：**2026-04-13**

本文档聚焦 V0.5.0 版本中前端最重要的公共能力，说明它们的职责、实现逻辑与接入方式。

## 一、统一 HTTP 请求层：`src/api/http.ts`

### 1.1 当前对外客户端

| 客户端 | 用途 | 特点 |
|---|---|---|
| `http` | 默认请求客户端 | 适合大多数页面请求 |
| `httpSuccess` | 强制弹成功提示 | 适合写操作或需要明确成功反馈的场景 |
| `silentHttp` | 静默请求 | 适合轮询、预加载、后台刷新 |

### 1.2 已支持的能力

- 请求头统一注入
- 语言透传
- 登录失效处理
- 后端成功/失败消息显示
- 消息防重复
- 静默错误控制
- 自定义错误消息

### 1.3 推荐使用方式

1. 查询接口：优先使用 `http`
2. 新增/修改/删除：优先让后端返回 `message`，前端继续使用 `http` 或 `httpSuccess`
3. 后台轮询/无感刷新：使用 `silentHttp`

## 二、配置驱动列表页：`FxDynamicTable`

### 2.1 组件定位

`FxDynamicTable` 负责把“查询区 + 表格列 + 分页 + 列设置”标准化，业务页面只需提供：

- `tableCode`
- `request`
- `dictOptions`（可选）

### 2.2 能力范围

| 能力 | 说明 |
|---|---|
| 查询区自动渲染 | 根据后端列配置渲染查询字段 |
| 表格列自动渲染 | 读取表格配置中心的列定义 |
| 用户列设置 | 与列设置组件联动保存个性化偏好 |
| 字典项渲染 | 与字典/标签配置联动 |
| 分页排序 | 标准分页与排序参数回调 |
| 国际化重载 | 切换语言后可重新读取配置与文案 |

### 2.3 推荐适用场景

- 用户、角色、租户、消息模板等标准管理页
- 列结构变化较频繁的业务列表页
- 需要用户个性化列显隐/排序的页面

## 三、列设置：`ColumnSettingButton`

### 3.1 职责

列设置组件负责让用户对当前表格进行：

- 显示/隐藏列
- 调整列顺序
- 保存个人偏好
- 恢复默认配置

### 3.2 联动方式

```text
FxDynamicTable -> ColumnSettingButton -> 用户列配置接口
-> 保存到后端 -> 再次加载时应用用户偏好
```

## 七、前后端交互原则

1. **后端写操作优先返回 `message`，前端少写重复提示。**
2. **页面不要自己重复封装 axios，统一走 `src/api/http.ts`。**
3. **列表页优先用动态表格，不再重复手写查询区和列配置逻辑。**
4. **组件文案优先走国际化，不在页面中硬编码文本。**

## 九、V0.5.0 前端开发建议

- 新模块页面先判断能否复用已有公共组件。
- 所有写操作接口都检查后端是否有明确成功消息。
- 页面消息尽量由公共 HTTP 层统一托管。
- 公共组件文档优先沉淀到 `Forgex_Doc/前端`，复杂专题再补充旧目录手册。
