# 系统页面引导使用方式

> 分类：前端 / 组件与页面
> 版本：**V0.6.5**
> 更新时间：**2026-04-29**

本文档说明系统管理模块页面如何接入页面级引导，重点覆盖页面配置、按钮锚点、首次打开触发和跳过引导。

## 代码位置

```text
Forgex_MOM/Forgex_Fronted/src/components/common/FxGuideTour.vue
Forgex_MOM/Forgex_Fronted/src/guide/systemPageGuides.ts
Forgex_MOM/Forgex_Fronted/src/layouts/MainLayout.vue
Forgex_MOM/Forgex_Fronted/src/stores/guide.ts
```

系统管理页面的引导文案集中维护在：

```text
src/guide/systemPageGuides.ts
```

普通系统菜单首次打开时，由主布局在路由切换后调用菜单打开次数接口；如果返回 `firstOpen=true`，会根据当前路由生成对应页面引导。

用户在个人资料页点击“重新播放系统引导”后，会同时把 `system.main/v1` 和所有 `system.menu.* / v2` 页面级引导重置为 `PENDING`。再次进入系统管理页面时，即使菜单打开次数接口不再返回首次打开，也会根据 `PENDING` 状态直接启动当前页面引导。

## 新增系统页面引导

新增系统页面时，优先按以下步骤接入：

1. 在 `systemPageGuides.ts` 的 `pageGuides` 中增加页面配置。
2. 第一段说明写在 `title` 和 `intro` 中，用于页面打开后的居中说明。
3. 需要指向按钮或区域的步骤，放到 `actions` 或 `extraSteps`。
4. 页面关键按钮使用稳定的 `data-guide-id`，不要用按钮中文文本做选择器。
5. 如果页面使用 `FxDynamicTable`，查询区、工具栏、列设置、表格区、分页区已有公共锚点。

非表格页面接入时，需要显式关闭不适用的默认区域，例如 `includeQuery: false`、`includeToolbar: false`、`includeColumnSetting: false`、`includeTable: false`、`includePagination: false`，再通过 `extraSteps` 指向页面真实存在的模块锚点。

示例：

```ts
const pageGuides = {
  user: {
    title: '用户管理',
    intro: '用户管理用于维护登录账号、人员状态、部门岗位和角色关系。',
    actions: [
      { id: 'sys-user-add', title: '新增用户', description: '点击这里创建新的系统账号。' },
    ],
  },
}
```

页面按钮示例：

```vue
<a-button data-guide-id="sys-user-add" type="primary" @click="openAddDialog">
  新增
</a-button>
```

## 用户管理页要求

用户管理页已配置更细的引导步骤，包含：

- 页面作用说明
- 查询区
- 工具栏
- 新增
- 同步第三方
- 从第三方拉取
- 导入
- 下载模板
- 批量删除
- 导出
- 行编辑
- 启停
- 重置密码
- 分配角色
- 删除
- 列设置
- 表格区
- 分页区

因为行操作需要被引导气泡直接定位，用户管理页将 `FxActionGroup` 的 `max-inline` 设置为 `5`，确保五个行操作都在 DOM 中可见。

## 角色授权页

角色菜单授权、人员授权当前从角色管理页进入，属于角色管理的内嵌工作流。
页面自身也有独立引导配置：

- `role/MenuGrant`
- `role/UserGrant`

角色管理页在切换到授权内嵌视图后，会使用同一套 `FxGuideTour` 和 `guideStates` 自动触发对应引导。这样即使授权页没有发生实际路由切换，也能展示专属说明。

菜单授权页顶部信息区保持紧凑展示，角色名称放在右侧信息块，表格默认隐藏查询表单以保留树形授权区高度。勾选父目录时，前端保存前会把父目录递归展开为其下菜单和按钮 ID，同时合并后端已有授权中未加载模块的 ID，避免只保存当前模块时清空其他模块权限。

## 跳过引导

跳过按钮显示在引导气泡底部，不再使用右下角悬浮按钮。
用户点击“跳过引导”后，当前 `guideCode` 会写入 `SKIPPED`；点击“完成”后写入 `COMPLETED`。

状态仍保存在用户引导偏好中：

```text
guideStates[guideCode]
```

如果需要重新测试某个页面引导，可以清空对应 `guideCode` 的状态，或把状态改回 `PENDING`。`FxGuideTour` 支持 `startKey` 显式启动信号，主布局会在页面级引导被重置后递增该信号，避免仅重置状态但组件没有重新打开。

## 常见排查

### 首次打开没有弹出引导

优先检查：

1. 当前路径是否在 `/workspace/sys/` 下。
2. 菜单打开次数接口是否返回 `firstOpen=true`。
3. `guideStates[guideCode]` 是否已经是 `SKIPPED` 或 `COMPLETED`。
4. `systemPageGuides.ts` 是否能把当前路由归一化到正确页面 key。
5. 如果是“重新播放系统引导”，检查当前页面对应 `guideStates[guideCode]` 是否已被重置为 `PENDING`。

页面第一步通常没有 `target`，这类步骤会以 `placement: 'center'` 居中展示；如果错误地把无目标步骤解析成空目标，Tour 会认为目标不存在并可能不显示。

### 某一步自动跳过

这通常说明目标元素不存在或当前权限下不可见。
页面级引导会自动跳过找不到目标的步骤，避免阻塞后续引导。

如果在系统管理主页这类非表格页面中点击下一步直接跳到最后一步，优先检查该页面是否关闭了查询区、列设置、分页区等表格默认步骤。

### 控制台出现 `<rect>` 负宽度错误

这是 Tour 遮罩高亮区域在视口边缘被默认 gap 扩展后产生的 SVG 计算问题。公共引导组件已把 `gap.offset` 固定为 `0`，新增引导时不要在贴边目标上重新设置负向或过大的高亮间距。

### 气泡定位不准

优先检查：

1. `data-guide-id` 是否拼写一致。
2. 目标按钮是否因为权限不可见。
3. 操作是否被折叠到下拉菜单中。
4. 页面是否还在异步加载，目标 DOM 尚未渲染。
