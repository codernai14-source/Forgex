# 消息模板预览与接收人选择使用方式

> 分类：前端 / 组件与页面
> 版本：**V0.6.5**
> 更新时间：**2026-04-17**

本文档说明在页面中如何使用 `TemplatePreview` 和 `ReceiverSelector`，重点覆盖代码接入和页面操作方式。

## ReceiverSelector 如何使用

### 最小示例

```vue
<template>
  <ReceiverSelector v-model="receiver" />
</template>

<script setup lang="ts">
import { reactive } from 'vue'
import ReceiverSelector from '@/components/common/ReceiverSelector.vue'

const receiver = reactive({
  receiverType: undefined,
  receiverIds: [],
})
</script>
```

### 页面如何使用

用户在页面上的操作路径：

1. 先选择接收类型
2. 如果是用户、角色、部门、岗位，则继续从下拉中多选
3. 如果是自定义，则只展示说明，不需要继续选择

### 多接收人配置场景

在消息模板页面中，通常不是只配置一条接收规则，而是配置一个数组：

```ts
formData.receivers.push({
  receiverType: undefined,
  receiverIds: [],
})
```

然后每一项都渲染一个 `ReceiverSelector`。

### 页面如何保存

保存时直接把组件输出结果作为接收人配置提交：

```ts
receivers: formData.receivers.map((receiver) => ({
  ...receiver,
  receiverIds: Array.isArray(receiver.receiverIds) ? receiver.receiverIds : [],
}))
```

## ReceiverSelector 使用建议

- 外层表单自己做“至少一条接收人”的校验
- `CUSTOM` 类型不要强制要求 `receiverIds`
- 接收人数组建议由页面统一维护，不要让组件自己负责增删

## TemplatePreview 如何使用

### 最小示例

```vue
<template>
  <a-button @click="previewVisible = true">预览</a-button>

  <TemplatePreview
    v-model:visible="previewVisible"
    :contents="contents"
    :message-type="messageType"
  />
</template>

<script setup lang="ts">
import { ref } from 'vue'
import TemplatePreview from '@/components/common/TemplatePreview.vue'

const previewVisible = ref(false)
const messageType = ref('NOTICE')
const contents = ref([
  {
    platform: 'INTERNAL',
    contentTitleI18nJson: '{"zh-CN":"系统通知"}',
    contentBodyI18nJson: '{"zh-CN":"您好，${userName}，您有一条新的通知"}',
    linkUrl: '/workspace/message',
  },
])
</script>
```

### 页面如何使用

通常是：

1. 用户先配置模板内容
2. 页面把当前 `formData.contents` 直接传给预览组件
3. 用户点击预览按钮，检查内容和占位符效果

### 在消息模板页中的推荐调用时机

只有在内容数组不为空时才允许预览：

```ts
function handlePreview() {
  if (formData.contents.length === 0) {
    message.warning('请先配置模板内容')
    return
  }
  previewVisible.value = true
}
```

## 页面如何组织消息模板内容

推荐每个平台一条内容：

```ts
{
  platform: 'INTERNAL',
  contentTitleI18nJson: '',
  contentBodyI18nJson: '',
  linkUrl: '',
}
```

其中：

- 标题可以为空
- 正文通常必填
- `contentBodyI18nJson` 推荐使用 `I18nInput`

## 页面使用建议

- 接收人组件负责单条规则编辑，外层页面负责规则数组增删
- 预览组件只做查看，不要在预览弹窗里再改数据
- 模板正文建议配合占位符工具栏输入

## 常见排查

### ReceiverSelector 选中后表单没更新

优先检查：

1. 是否使用了 `v-model`
2. 外层是否又手动覆盖了该对象

### 预览弹窗显示的不是最新内容

优先检查：

1. 传给 `TemplatePreview` 的是否是当前表单数据
2. 是否在点击预览前把表单值同步到了 `formData.contents`
