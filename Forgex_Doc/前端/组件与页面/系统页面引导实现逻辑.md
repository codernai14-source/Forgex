# 系统页面引导实现逻辑

> 分类：前端 / 组件与页面  
> 版本：**V0.6.0**  
> 更新时间：**2026-04-29**

本文档说明系统管理模块页面级引导的实现逻辑，包括引导气泡、页面配置注册表、菜单首次打开触发、锚点定位和跳过状态持久化。

## 功能定位

系统页面引导用于在用户首次进入系统管理模块页面时，先解释页面用途，再指向查询区、工具栏、关键按钮、表格区和行操作。

它解决的问题包括：

- 不同系统页面展示不同引导文案
- 首次打开菜单时自动出现引导
- 权限导致按钮不可见时自动跳过对应步骤
- 跳过和完成状态可持久化
- 公共表格区域有统一定位锚点

## FxGuideTour

`FxGuideTour.vue` 基于 Ant Design Vue Tour 实现。  
当前使用 `renderPanel` 自定义面板内容，面板底部统一包含：

- 当前进度
- 跳过引导
- 上一步
- 下一步 / 完成

因此跳过按钮在引导气泡内部，不再使用页面右下角悬浮按钮。

注意：当前 Ant Design Vue 版本会在 `<a-tour>` 顶层覆盖 `renderPanel`，所以公共组件会把自定义 `renderPanel` 注入到每个 step 上，确保真正渲染自定义面板和“跳过引导”按钮。

组件会把传入步骤解析成 `GuideResolvedStep`：

- 有 `target` 的步骤会查找对应 DOM。
- 没有 `target` 的步骤视为居中说明步骤，并保持 `target: undefined`，让 Ant Design Vue Tour 按 `placement: 'center'` 渲染。
- 找不到目标时自动跳到下一个可用步骤。

组件还支持 `startKey` 作为显式启动信号。父组件每次递增 `startKey` 时，`FxGuideTour` 会重新执行 `openTour()`，用于处理“重新播放”或状态被重置为 `PENDING` 但 `autoStart` 布尔值没有变化的场景。

Tour 遮罩的 `gap.offset` 统一设为 `0`。这样目标元素贴近视口边缘时，不会因为默认高亮间距把 `left` 计算成负数，从而触发 SVG `<rect>` 的负宽度控制台错误。

## 引导步骤类型

`FxGuideStep` 支持 `category` 字段，用于区分步骤类型：

```ts
category?: 'intro' | 'navigation' | 'form' | 'action' | 'table' | 'detail'
```

常用语义：

| category | 用途 |
|---|---|
| `intro` | 页面作用说明 |
| `form` | 查询区或筛选区 |
| `action` | 工具栏按钮或主操作 |
| `table` | 表格、列设置、分页 |
| `detail` | 行操作或明细操作 |
| `navigation` | 页面导航、模块切换、页签 |

## 页面配置注册表

系统模块页面配置集中在：

```text
src/guide/systemPageGuides.ts
```

对外导出：

```ts
resolveSystemPageGuide(path: string)
```

该函数会：

1. 去掉 `/workspace/sys/` 前缀。
2. 去掉查询参数。
3. 对隐藏授权页等动态路径做归一化。
4. 生成 `guideCode`。
5. 返回 `{ guideCode, version, steps }`。

当前系统页面级引导版本为 `v2`。

## 默认步骤生成

每个页面第一步固定为页面说明：

```ts
{
  title: meta.title,
  description: meta.intro,
  placement: 'center',
  useMask: false,
  category: 'intro',
}
```

如果页面没有关闭默认区域，后续会自动追加：

- 查询区：`fx-table-query`
- 工具栏：`fx-table-toolbar`
- 列设置：`fx-table-column-setting`
- 表格区：`fx-table-content`
- 分页区：`fx-table-pagination`

页面自己的按钮步骤通过 `actions` 或 `extraSteps` 补充。

系统管理主页、模块首页等非表格页面应显式关闭查询区、工具栏、列设置、表格区、分页区等默认步骤，只保留页面说明和页面真实存在的锚点，避免点击下一步时连续跳过不存在的表格步骤。

## 首次打开触发链路

系统模块普通菜单的触发链路在 `MainLayout.vue`：

1. 路由进入 `/workspace/sys/*`。
2. 先解析当前页面的 `guideCode`，如果状态是 `PENDING` 或开启婴儿模式，直接启动当前页面引导，同时后台静默上报打开次数。
3. 如果状态不是 `PENDING`，调用 `reportUserMenuOpen(path)`。
4. 后端返回 `{ openCount, firstOpen }`。
5. 当 `firstOpen=true` 时，调用 `resolveSystemPageGuide(path)`。
6. 根据 `guideStore.shouldAutoStartGuide(guideCode, version)` 判断是否自动打开。
7. 用户点击跳过或完成后写入 `guideStates`。

偏好配置异步加载完成后，主布局会再次检查当前路由。如果用户直接刷新在系统管理页面，或“重新播放系统引导”后当前页面状态为 `PENDING`，会立即调用页面级引导启动逻辑，不依赖打开次数接口再次返回 `firstOpen=true`。

打开次数接口只用于系统模块自动触发，本次没有改动接口协议。

## 角色授权内嵌页

角色菜单授权和人员授权从角色管理页内嵌进入，不一定发生路由切换。  
因此角色管理页在 `activeView` 切换为 `menuGrant` 或 `userGrant` 后，会局部挂载 `FxGuideTour`，并复用：

- `resolveSystemPageGuide('/workspace/sys/role/MenuGrant')`
- `resolveSystemPageGuide('/workspace/sys/role/UserGrant')`
- `guideStore.shouldAutoStartGuide`
- `markGuideSkipped`
- `markGuideCompleted`

这样两个授权工作流仍然使用统一引导状态，不需要新增接口。

## DOM 锚点

公共表格组件提供稳定锚点：

| 锚点 | 位置 |
|---|---|
| `fx-table-query` | 查询区 |
| `fx-table-toolbar` | 工具栏 |
| `fx-table-toolbar-left` | 工具栏左侧业务操作 |
| `fx-table-column-setting` | 列设置 |
| `fx-table-content` | 表格内容 |
| `fx-table-pagination` | 分页区 |

业务页面按钮统一使用 `data-guide-id`。  
行操作如果通过 `FxActionGroup` 渲染，可以在 `ActionItem` 中传入 `guideId`，组件会渲染到对应操作元素上。

## 状态持久化

引导状态由 `src/stores/guide.ts` 管理，核心结构是：

```ts
guideStates[guideCode] = {
  status: 'SKIPPED' | 'COMPLETED' | 'PENDING',
  version,
  updatedAt,
}
```

判断逻辑：

- 没有状态时允许自动打开。
- `PENDING` 允许自动打开。
- `SKIPPED` 和 `COMPLETED` 不再自动打开。
- 引导版本变化时可以重新打开。
- 婴儿模式开启时允许重新打开。

个人资料页的“重新播放系统引导”会重置 `system.main/v1` 和 `listSystemPageGuideCodes()` 返回的所有系统页面 `system.menu.* / v2`，因此系统主页引导和页面级引导都可以重新播放。

## 为什么这样设计

### 1. 页面文案集中维护

所有系统页面引导都在 `systemPageGuides.ts` 中注册，减少散落在各页面里的说明文案。

### 2. 页面锚点稳定

使用 `data-guide-id` 定位，避免依赖中文按钮文本、DOM 层级或样式类名。

### 3. 权限场景不阻塞

找不到目标元素时自动跳过该步骤。按钮因权限不可见时，引导仍可继续。

### 4. 内嵌工作流可复用同一机制

角色授权没有强依赖独立路由，但仍复用相同 `guideCode`、版本和状态持久化逻辑。

## 常见排查

### 配置了步骤但不显示

优先检查：

1. `guideCode` 是否已经是 `SKIPPED` 或 `COMPLETED`。
2. `path` 是否被 `normalizeSystemGuidePath` 归一化到正确 key。
3. 步骤是否都指向了不可见元素。
4. 如果是重新播放不显示，检查 `guideStates[guideCode]` 是否已经变为 `PENDING`，以及 `startKey` 是否在主布局中递增。
5. 如果第一步是页面说明，确认该步骤没有传入空函数目标，应由 `target: undefined` 配合 `placement: 'center'` 渲染。

### 目标按钮存在但 Tour 找不到

优先检查：

1. 是否写成 `data-guide-id="xxx"`。
2. `systemPageGuides.ts` 的 `id` 是否完全一致。
3. 操作是否被折叠到下拉菜单。
4. 该按钮是否受权限控制，当前账号不可见。
