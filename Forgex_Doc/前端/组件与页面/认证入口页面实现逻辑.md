# 认证入口页面实现逻辑

> 版本：`V0.6.5`

本文档说明登录页、注册页等认证入口页面的前端布局与浏览器自动填充样式处理。

## 入口文件

- 登录页：`Forgex_MOM/Forgex_Fronted/src/views/auth/login/index.vue`
- 注册页：`Forgex_MOM/Forgex_Fronted/src/views/auth/register/index.vue`

## 背景与主题

认证入口页面复用系统基础配置中的登录背景、系统名称、系统 Logo、主色和辅色。图片或视频背景固定铺满视口，遮罩和网格层固定覆盖，避免页面滚动时背景与表单脱节。

登录页使用 `cyber-input` 作为原生输入框样式入口。注册页基于 Ant Design Vue 表单组件，通过 `register-form :deep(...)` 统一覆盖输入框、密码框、占位文字、聚焦边框和按钮色彩。

## 自动填充样式

Chrome、Edge 等 Chromium 浏览器在账号密码自动填充后会注入 `:-webkit-autofill` 默认背景色。认证入口页面使用页面级 `--auth-field-bg` 变量，并在普通输入框、密码框内部输入框上覆盖：

- `-webkit-text-fill-color` 保持暗色主题文字颜色；
- `box-shadow` / `-webkit-box-shadow` 使用大内阴影覆盖浏览器默认自动填充底色；
- `caret-color` 保持主题色光标；
- 长时 `transition` 避免自动填充背景闪回。

同时为登录密码使用 `autocomplete="current-password"`，注册密码使用 `autocomplete="new-password"`，账号、姓名、电话、邮箱和验证码使用对应语义，减少浏览器误判字段。

## 注册页布局

注册页桌面端采用左右分区布局：左侧品牌信息，右侧注册面板。桌面端表单面板宽度为 560px 左右，将账号/用户名、密码/确认密码、手机号/邮箱组合为两列，降低表单整体高度。

桌面端外层容器隐藏滚动，面板在极端高度不足时内部滚动，避免浏览器出现整页滚动条和底部空白。移动端恢复外层纵向滚动，并将表单切回单列，保证小屏不裁切内容。

## 验证方式

修改认证入口页面样式后，至少执行：

```bash
npm run build
```

建议再通过浏览器检查桌面与移动视口：

- 桌面端确认 `document` / `body` 高度不超过视口高度，页面右侧不出现外层滚动条；
- 自动填充账号密码后，输入框背景仍保持暗色主题；
- 移动端可滚动查看完整表单与底部操作区。
