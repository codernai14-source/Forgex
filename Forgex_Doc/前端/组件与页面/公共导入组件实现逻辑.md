# 公共导入组件实现逻辑

> 分类：前端 / 组件与页面  
> 版本：**V0.6.0**  
> 更新时间：**2026-05-01**

本文档说明 `CommonImportDialog` 的实现方式，重点覆盖配置加载、本地解析、基础校验、多 Sheet 分组和统一提交。

## 功能定位

`CommonImportDialog.vue` 是 Excel 文件导入的统一前端入口。它不承载具体业务落库规则，只负责把用户选择的文件解析成后端统一接口需要的数据结构。

它解决的是：

- 业务页面重复写上传和模板下载逻辑。
- 导入模式、预览、校验结果展示不一致。
- 单 Sheet 和多 Sheet 文件传参结构不统一。
- 导入按钮和提示文案难以统一国际化。

## 数据流

组件打开后的主流程是：

```text
接收 tableCode
  -> 调用 /sys/excel/importConfig/detailByCode
  -> 用户选择导入方式
  -> 用户选择或拖拽文件
  -> xlsx 本地解析文件
  -> 按导入配置匹配表头
  -> 过滤空行并校验必填字段
  -> 生成预览数据和 importData
  -> 调用 /sys/excel/import/execute
  -> 展示 FxExcelImportResultDTO 统计
```

## 配置加载

组件通过 `importConfigDetailByCode` 获取导入配置。

导入配置中的关键字段包括：

- `items`：导入列配置。
- `sheetCode`：Sheet 稳定编码。
- `sheetName`：Sheet 展示名称。
- `importField`：提交给后端的数据字段名。
- `i18nJson`：用于匹配当前语言下的表头。
- `required`：是否必填。

如果旧数据仍返回 `fields`，配置弹窗层会做兼容，但公共导入组件统一按 `items` 使用。

## 文件解析

组件使用 `xlsx` 依赖解析导入文件。

支持格式：

- `.xlsx`
- `.xls`
- `.csv`

基础限制：

- 文件大小不超过 10MB。
- CSV 只允许用于单 Sheet 导入。
- 多 Sheet Excel 会按导入配置中的 `sheetCode` 和 `sheetName` 读取。

## 表头匹配

表头匹配优先使用当前语言下的列名。

匹配顺序为：

1. 当前 locale 对应的 `i18nJson`。
2. 中文列名。
3. 英文列名。
4. `importField`。

为了兼容模板标题行、说明行等场景，组件会扫描前几行，选择和配置匹配度最高的一行作为表头行。

## 本地校验

本地校验只覆盖通用基础规则：

- 文件类型。
- 文件大小。
- Sheet 是否存在。
- 表头是否匹配。
- 必填字段是否为空。
- 空行过滤后是否存在有效数据。

业务唯一性、外键、字典合法性、覆盖边界仍由后端业务处理器判断。

## 预览与结果

组件会按 Sheet 展示前 5 行预览，并提供查看全部数据弹窗。

确认导入后，组件展示后端返回的统计字段：

- `totalCount`
- `createdCount`
- `updatedCount`
- `skippedCount`
- `failedCount`

## 统一提交结构

组件提交到后端的数据结构为：

```ts
{
  tableCode: string
  importMode: 'ADD' | 'UPDATE' | 'COVER'
  importConfig: object
  importData: Record<string, Array<Record<string, unknown>>>
}
```

后端会重新按 `tableCode` 查询配置并做权限校验，前端传入的 `importConfig` 只作为处理器执行上下文的一部分，不作为权限判断依据。

## 适合的使用边界

适合：

- 用户导入。
- 供应商多 Sheet 导入。
- 语言类型导入。
- 后续配置驱动的 Excel 文件导入。

不适合：

- JSON 参数导入。
- 第三方系统同步或拉取。
- 不依赖 Excel 模板的自由格式文件上传。
