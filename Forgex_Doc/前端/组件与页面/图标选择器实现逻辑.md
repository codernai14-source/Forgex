# 图标选择器实现逻辑

> 分类：前端 / 组件与页面
> 版本：**V0.6.5**
> 更新时间：**2026-04-17**

本文档说明 `IconPicker` 的实现方式，重点解释它如何维护图标名称、如何与业务表单联动，以及代码具体在哪里。

## 功能定位

`IconPicker.vue` 是一个“图标名称选择器”，不是图片上传器，也不是 SVG 编辑器。

它解决的是后台配置场景里很常见的一类问题：

- 菜单图标配置
- 模块图标配置
- 其他需要保存图标组件名的表单字段

它最终保存的不是图标文件，而是一个字符串，例如：

```ts
'UserOutlined'
'AppstoreOutlined'
```

## 组件核心思路

这个组件的核心思路非常简单：

1. 从 `@ant-design/icons-vue` 全量导入图标
2. 过滤出真正可用的图标导出项
3. 表单里保存“图标名称字符串”
4. 预览时再根据名称反查对应组件

所以它本质上是：

```text
图标名字符串 <-> 图标组件
```

之间的一个桥接层。

## 图标来源逻辑

组件内部有：

```ts
import * as Icons from '@ant-design/icons-vue'
```

然后通过 `iconNames` 计算属性筛选：

- `Outlined`
- `Filled`
- `TwoTone`

结尾的导出项。

这样做的好处是：

- 不需要手写图标清单
- 升级图标库后能自动跟随
- 只保留真正可渲染的图标

## 值模型

组件对外使用的是：

```vue
v-model:value
```

值类型是：

```ts
string | undefined
```

例如：

```ts
formData.icon = 'AppstoreOutlined'
```

组件内部用 `innerValue` 作为中间状态，并通过：

```ts
emit('update:value', value)
```

同步回父组件。

## 预览逻辑

组件右侧按钮默认显示当前选中的图标预览。

实现方式是：

1. 读取当前字符串值 `innerValue`
2. 在 `Icons` 中按名称取组件
3. 如果找到则渲染该图标
4. 如果找不到则显示默认的 `AppstoreOutlined`

这使得表单在编辑状态下能直接回显已经保存过的图标。

## 弹窗选择逻辑

点击按钮后会打开一个 `a-modal`。

弹窗内部做了两件事：

- 输入关键字筛选图标名
- 点击某个图标项后立刻写回值并关闭弹窗

筛选逻辑基于图标名字符串本身，不需要额外数据源。

## 为什么保存图标名而不是图标对象

主要有三个原因：

### 1. 后端存储更稳定

数据库保存字符串远比保存复杂对象简单。

### 2. 页面回显更自然

只要知道名字，就能通过统一工具函数或组件反向解析图标。

### 3. 和菜单、模块配置模型更匹配

菜单、模块本身就是配置对象，图标只是其中一个普通字段。

## 在菜单管理中的实际使用

在 `src/views/system/menu/index.vue` 中，菜单表单直接将它作为一个普通字段控件使用：

```vue
<a-form-item :label="t('system.menu.icon')" name="icon">
  <IconPicker
    v-model:value="formData.icon"
    :placeholder="t('system.menu.form.iconPlaceholder')"
  />
</a-form-item>
```

保存时，`formData.icon` 直接作为菜单字段一起提交给后端。

## 适合的使用边界

适合：

- 菜单
- 模块
- 配置型页面

不适合：

- 业务图片上传
- 用户头像选择
- 自定义 SVG 设计器

## 常见排查

### 页面能保存但图标不显示

优先检查：

1. 保存的是否是合法图标名
2. 展示页是否通过统一图标工具函数渲染

### 编辑回显时没有预览

优先检查：

1. `formData.icon` 是否真的有值
2. 图标名是否拼写正确

### 搜索不到某个图标

优先检查：

1. 该图标是否属于 `@ant-design/icons-vue`
2. 名称是否以 `Outlined/Filled/TwoTone` 结尾
