# 认证入口页面使用方式

> 版本：`V0.6.5`

本文档面向维护登录页、注册页样式和交互的前端开发者。

## 修改入口

- 登录页样式调整优先修改 `src/views/auth/login/index.vue` 中的 `.cyber-input` 和认证页面局部样式。
- 注册页样式调整优先修改 `src/views/auth/register/index.vue` 中的 `.register-wrap`、`.register-shell`、`.register-panel`、`.register-form` 和响应式规则。
- 新增账号、密码、验证码等输入项时，应同步设置合适的 `autocomplete` 属性。

## 自动填充处理

不要只修改普通 `background`，因为浏览器自动填充后的背景通常由 `:-webkit-autofill` 伪类接管。新增输入框样式时，应确认它处于页面级自动填充覆盖规则范围内。

常见语义：

| 字段 | autocomplete |
|---|---|
| 登录账号 | `username` |
| 登录密码 | `current-password` |
| 注册密码 / 确认密码 | `new-password` |
| 姓名 | `name` |
| 手机号 | `tel` |
| 邮箱 | `email` |
| 验证码 / 邀请码 | `off` |

## 布局约定

桌面端注册页应尽量在一屏内展示主要表单内容。新增字段时优先考虑组合成两列表单；若字段明显增加，应让注册面板内部滚动，而不是恢复整页滚动。

移动端保持单列表单和页面纵向滚动，确保验证码图片、提交按钮和返回登录入口都能自然访问。

## 检查清单

- 浏览器自动填充账号或密码后，输入框背景不变白。
- 桌面端注册页没有外层白色滚动条和底部大块空白。
- 移动端表单不被裁切，继续滑动可看到完整底部内容。
- 构建命令 `npm run build` 通过。
