# 部门树与组织选择使用方式

> 分类：前端 / 组件与页面
> 版本：**V0.6.5**
> 更新时间：**2026-04-17**

本文档说明开发页面时如何接入 `DeptTree`，重点包含代码如何使用、页面如何联动，以及哪些场景适合用它。

## 代码位置

```text
Forgex_MOM/Forgex_Fronted/src/components/system/DeptTree.vue
Forgex_MOM/Forgex_Fronted/src/views/system/department/index.vue
```

## 适用场景

推荐在以下页面优先使用：

- 左树右表
- 部门筛选用户
- 组织筛选岗位
- 组织详情页

不建议在以下场景硬套：

- 只需要一个简单下拉选择器
- 组织编辑逻辑完全独立的弹窗

## 最小接入示例

```vue
<template>
  <a-row :gutter="16">
    <a-col :span="6">
      <DeptTree ref="deptTreeRef" @select="handleDeptSelect" />
    </a-col>

    <a-col :span="18">
      <FxDynamicTable
        ref="tableRef"
        table-code="UserTable"
        :request="handleRequest"
        row-key="id"
      />
    </a-col>
  </a-row>
</template>

<script setup lang="ts">
import { ref } from 'vue'
import DeptTree from '@/components/system/DeptTree.vue'
import FxDynamicTable from '@/components/common/FxDynamicTable.vue'
import { getUserList } from '@/api/system/user'

const deptTreeRef = ref()
const tableRef = ref()
const currentDeptId = ref('')

function handleDeptSelect(keys: string[], node: any) {
  currentDeptId.value = String(node?.id || '')
  tableRef.value?.refresh?.()
}

async function handleRequest(payload: any) {
  const res = await getUserList({
    pageNum: payload.page.current,
    pageSize: payload.page.pageSize,
    deptId: currentDeptId.value || undefined,
    ...payload.query,
  })

  return {
    records: res.records || [],
    total: Number(res.total || 0),
  }
}
</script>
```

## 页面如何使用选中事件

`DeptTree` 最常用的事件就是：

```vue
@select="handleDeptSelect"
```

推荐写法：

```ts
function handleDeptSelect(keys: string[], node: any) {
  const deptId = String(node?.id || keys?.[0] || '')
  currentDeptId.value = deptId
  loadDetailOrList()
}
```

这样写的好处是：

- 既能拿到完整节点对象
- 也兼容只用 `keys[0]`

## 页面如何使用新增按钮

如果页面希望树头部显示“新增”按钮，可以传：

```vue
<DeptTree :show-add="true" @add="handleAddDept" />
```

页面侧自己决定点击后干什么，例如：

- 打开新增弹窗
- 打开右侧表单
- 默认以当前节点为父节点新增

示例：

```ts
function handleAddDept() {
  openDeptForm({
    parentId: currentDeptId.value || '0',
  })
}
```

## 页面如何刷新树

组件暴露了 `loadTree()`，保存成功后推荐这样刷新：

```ts
await deptTreeRef.value?.loadTree?.()
```

常见时机：

- 新增组织成功
- 编辑组织成功
- 删除组织成功
- 切换租户后重新加载组织树

## 页面如何在右侧显示详情

如果页面是详情模式，推荐在选中节点后再调用详情接口，而不是指望树节点上带全量业务字段。

示例：

```ts
async function handleDeptSelect(keys: string[], node: any) {
  const id = String(node?.id || '')
  if (!id) {
    currentDept.value = null
    return
  }

  currentDept.value = await getDepartment({
    id,
    tenantId: sessionStorage.getItem('tenantId'),
  })
}
```

这种方式更稳，因为树接口通常是轻量数据。

## 页面如何与表格联动

在左树右表场景中，推荐把组织筛选参数收口到右侧表格的 `request` 里。

```ts
async function handleRequest(payload) {
  return getPositionList({
    pageNum: payload.page.current,
    pageSize: payload.page.pageSize,
    deptId: currentDeptId.value || undefined,
    ...payload.query,
  })
}
```

这样树只是改变筛选条件，不直接侵入列表实现。

## 页面如何处理“默认未选中”

推荐页面一开始就兼容“没有选中任何组织”：

```ts
if (!keys.length) {
  currentDeptId.value = ''
  selectedDept.value = null
  return
}
```

否则在首次进入页面时，右侧很容易报空值错误。

## 页面使用建议

- 左树右表页面优先使用 `DeptTree`
- 右侧查询接口自己控制，不要把业务查询塞进树组件
- 保存组织后记得手动调用 `loadTree()`
- 如果树节点很多，尽量保持接口返回轻量结构

## 常见排查

### 页面接了树但右侧不刷新

优先检查：

1. `handleDeptSelect` 是否更新了筛选条件
2. 更新后是否调用了 `tableRef.refresh()`

### 点击新增没反应

优先检查：

1. 是否传了 `:show-add="true"`
2. 是否监听了 `@add`

### 刷新树后当前选中丢失

处理建议：

- 保存前记录当前节点 ID
- 刷新后主动重新选中或重新加载详情
