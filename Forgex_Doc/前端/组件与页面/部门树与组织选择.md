# 部门树与组织选择

> 分类：前端 / 组件与页面  
> 版本：**V0.5.0**  
> 更新时间：**2026-04-13**

## 一、概述

Forgex 前端部门树公共组件位于：

- `Forgex_MOM/Forgex_Fronted/src/components/system/DeptTree.vue`

它封装了系统组织架构的常见左侧树能力，用于在页面上完成：

- 部门树展示
- 组织节点选择
- 左树右表联动
- 快速新增入口
- 不同组织类型图标渲染

这类组件在用户、角色授权、职位、邀请码等页面中都有实际使用。

---

## 二、组件定位

`DeptTree` 的职责不是“编辑组织结构”，而是做一个稳定的**组织选择面板**。

典型布局：

```text
+----------------------+--------------------------------+
| DeptTree             | 右侧列表 / 表单 / 授权区域      |
| - 树形组织结构       | - 按部门筛选用户               |
| - 选中当前节点       | - 展示部门详情                 |
| - 新增按钮（可选）   | - 角色分配 / 职位配置等        |
+----------------------+--------------------------------+
```

## 四、组件能力

### 4.1 当前已支持的能力

- 自动读取当前租户 `tenantId`
- 调用后端部门树接口加载树形数据
- 默认展开全部节点
- 支持当前节点选中态
- 支持不同组织类型图标
- 支持“新增”按钮开关
- 对外暴露刷新方法

### 4.2 节点图标规则

组件会根据 `orgType` 自动切换图标：

| `orgType` | 图标 |
|---|---|
| `group` | 集团图标 |
| `company` | 公司图标 |
| `subsidiary` | 子公司图标 |
| `department` | 部门图标 |
| `team` | 团队图标 |
| 其他 | 默认部门图标 |

### 4.3 禁用态样式

若节点 `status === false`，组件会使用禁用文字样式显示，便于区分停用组织。

## 五、Props 与事件

### 5.1 Props

| 属性 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `showAdd` | `boolean` | `true` | 是否显示新增按钮 |

### 5.2 事件

| 事件 | 参数 | 说明 |
|---|---|---|
| `select` | `(keys, node)` | 用户选择树节点时触发 |
| `add` | 无 | 点击新增按钮时触发 |

### 5.3 暴露方法

组件通过 `defineExpose` 暴露：

| 方法 / 属性 | 说明 |
|---|---|
| `loadTree()` | 重新加载部门树 |
| `selectedKeys` | 当前选中节点键值 |

## 六、数据来源与接口

部门树通过如下接口获取：

```text
POST /sys/department/tree
```

前端 API 封装位于：

- `src/api/system/department.ts`

当前还提供：

- `listDepartments`
- `getDepartment`
- `createDepartment`
- `updateDepartment`
- `deleteDepartment`

### 6.1 租户依赖

组件会从：

```text
sessionStorage.getItem('tenantId')
```

读取当前租户 ID，然后查询对应组织树。

因此在以下情况下需要注意：

- 尚未选择租户时，部门树不会加载；
- 登录态切换租户后，建议重新调用 `loadTree()`；
- 如果页面支持跨租户管理，建议改为由父组件显式控制租户参数。

## 七、推荐用法

### 7.1 左树右表

```vue
<template>
  <div class="page-layout">
    <DeptTree ref="deptTreeRef" @select="handleDeptSelect" />
    <FxDynamicTable :request="loadUserPage" table-code="sys_user_table" />
  </div>
</template>
```

```ts
const currentDeptId = ref<string>('')

function handleDeptSelect(keys: string[], node: any) {
  currentDeptId.value = String(node?.id || '')
  tableRef.value?.reload()
}
```

### 7.2 与新增弹窗联动

```vue
<DeptTree @add="openCreateDepartment" />
```

点击新增后可直接打开 `BaseFormDialog` 来创建部门。

## 八、适用页面

目前这类能力尤其适用于：

- 用户管理：按组织筛选用户
- 角色授权：按组织查看可分配人员
- 职位管理：挂载到部门
- 邀请码管理：注册后自动归属部门
- 组织维护：左树右表的组织结构页面

---

## 九、实践建议

1. **优先把 `DeptTree` 作为左侧筛选导航，而不是复杂编辑器**；
2. **右侧业务列表的筛选条件由选中节点驱动**；
3. **切换租户后主动刷新树数据**；
4. **组织编辑、删除、排序等重操作建议放到右侧详情区或弹窗中完成**；
5. **如果页面需要部门平铺下拉，可将树结构预处理为扁平选项，不要强行滥用树组件。**
