# 多语言输入与动态布局

> 分类：前端 / 国际化与布局  
> 版本：**V0.5.0**  
> 更新时间：**2026-04-13**

## 一、概述

本分类下当前有两类容易混在一起、但职责不同的公共能力：

1. **多语言输入**：解决表单中多语言名称、标题、正文的编辑；
2. **动态布局**：解决首页/工作台这类组件化页面的配置与拖拽。

其中多语言输入的核心组件位于：

- `Forgex_MOM/Forgex_Fronted/src/components/common/I18nInput.vue`

而首页拖拽布局专题见：

- [个人首页与可拖拽布局](./个人首页与可拖拽布局.md)

---

## 二、多语言输入组件定位

`I18nInput` 主要用来处理如下字段：

- 菜单名称
- 字典显示名
- 模板标题
- 模板内容
- 模块名称
- 各类带多语言 JSON 存储的文本字段

它的目标不是“切换界面语言”，而是帮助业务页面**录入和维护多语言内容本身**。

---

## 三、核心能力

### 3.1 三种展示模式

| 模式 | 说明 | 适用场景 |
|---|---|---|
| `simple` | 单输入框 / 文本域 + 图标唤起配置 | 最常用，适合普通表单 |
| `table` | 直接按语言列表表格编辑 | 管理页、配置页 |
| `modal` | 只显示弹窗编辑面板 | 需要把入口与编辑区完全分离时 |

### 3.2 两种输入类型

| `type` | 说明 |
|---|---|
| `input` | 单行文本 |
| `textarea` | 多行文本，适合模板正文、说明类字段 |

### 3.3 支持的附加能力

- 自动读取启用的语言列表
- 默认语言回填
- JSON 字符串双向绑定
- 表格化编辑多语言文本
- 占位符工具栏
- 弹窗内长文本输入
- 响应式弹窗高度与宽度调整

---

## 四、数据格式

组件通过 `v-model` 绑定一个 JSON 字符串，典型格式如下：

```json
{
  "zh-CN": "用户名",
  "en-US": "Username",
  "zh-TW": "使用者名稱"
}
```

### 4.1 组件内部处理方式

1. 读取 `modelValue`；
2. 解析为对象；
3. 根据语言列表构造表格行；
4. 编辑后再生成 JSON 字符串；
5. 通过 `update:modelValue` 回写父组件。

### 4.2 默认语言行为

在 `simple` 模式下，默认只显示**默认语言**对应的输入框；
更完整的多语言编辑通过图标打开弹窗完成。

---

## 五、语言来源

组件不会写死语言列表，而是调用后端接口动态获取启用语言：

- `listEnabledLanguages()`

因此它天然跟随系统语言配置中心工作，适合：

- 新增语言后自动扩展输入面板；
- 停用语言后收敛录入项；
- 与后端 `fx_i18n_language_type` 保持一致。

---

## 六、Props 说明

| 属性 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `modelValue` | `string` | `''` | 多语言 JSON 字符串 |
| `mode` | `'simple' \| 'table' \| 'modal'` | `simple` | 展示模式 |
| `type` | `'input' \| 'textarea'` | `input` | 输入类型 |
| `placeholder` | `string` | `请输入` | 占位提示 |
| `defaultLang` | `string` | `zh-CN` | 默认语言代码 |
| `rows` | `number` | `4` | 多行文本行数 |
| `showPlaceholders` | `boolean` | `false` | 是否显示占位符快捷插入 |

---

## 七、占位符能力

当 `showPlaceholders = true` 且使用模板类场景时，组件可插入一些内置占位符，例如：

- `${userName}`
- `${userAccount}`
- `${tenantName}`
- `${currentTime}`
- `${title}`
- `${content}`
- `${linkUrl}`

### 7.1 适用场景

尤其适合：

- 消息模板配置
- 通知标题/正文配置
- 站内信内容模板
- 含动态变量的多语言文本

### 7.2 交互体验

- `simple + textarea` 模式下：底部显示占位符工具栏；
- 弹窗模式中：右侧显示占位符面板，点击可插入到当前聚焦语言行。

---

## 八、推荐使用方式

### 8.1 普通名称字段

```vue
<I18nInput v-model="form.nameI18nJson" placeholder="请输入名称" />
```

### 8.2 多语言正文

```vue
<I18nInput
  v-model="form.contentI18nJson"
  type="textarea"
  :rows="4"
  :show-placeholders="true"
/>
```

### 8.3 配置页直接表格编辑

```vue
<I18nInput v-model="form.dictValueI18nJson" mode="table" />
```

## 九、适用页面

当前在项目中，这类组件常见于：

- 模块管理
- 菜单管理
- 字典管理
- 消息模板管理
- 表格配置管理

这说明它已不仅是“补充小组件”，而是多语言配置表单的标准输入方式。

---

## 十、与后端的配合要求

为保证数据闭环，后端通常需要：

1. 提供启用语言列表接口；
2. 接收并保存多语言 JSON 字段；
3. 在读取时原样回填 JSON；
4. 在实际展示时根据 `LangContext` 解析对应语言。

因此它通常与以下能力配套使用：

- 后端多语言配置
- `fx_i18n_language_type`
- 统一返回与国际化
- 消息模板配置

## 十一、动态布局专题说明

本页标题里虽然包含“动态布局”，但首页拖拽布局已单独沉淀为独立文档，建议分开阅读：

- **做多语言表单**：优先读本页；
- **做首页 / 工作台拖拽配置**：看 [个人首页与可拖拽布局](./个人首页与可拖拽布局.md)。

---

## 十二、实践建议

1. **多语言字段优先统一存 JSON，不要拆成多列硬编码**；
2. **普通短文本默认使用 `simple` 模式**；
3. **长文本模板建议使用 `textarea + showPlaceholders`**；
4. **配置页批量编辑建议使用 `table` 模式**；
5. **不要把“界面语言切换”和“多语言内容录入”混为一谈。**
