# 个人首页与可拖拽布局使用方式

> 分类：前端 / 国际化与布局
> 版本：**V0.6.5**
> 更新时间：**2026-04-17**

本文档说明页面开发时如何使用个人首页设计器，重点覆盖代码如何接入、页面如何操作，以及管理端如何维护默认布局。

## 代码位置

```text
Forgex_MOM/Forgex_Fronted/src/components/personal-homepage/PersonalHomepageDesigner.vue
Forgex_MOM/Forgex_Fronted/src/views/home/index.vue
Forgex_MOM/Forgex_Fronted/src/views/home/favorite-management/index.vue
Forgex_MOM/Forgex_Fronted/src/api/system/personalHomepage.ts
```

## 当前用户首页如何接入

最简单的接入方式就是首页入口页直接挂组件：

```vue
<template>
  <div class="personal-home-view">
    <PersonalHomepageDesigner mode="current" />
  </div>
</template>
```

这是当前项目中的标准写法，见：

```text
Forgex_MOM/Forgex_Fronted/src/views/home/index.vue
```

## 管理端默认布局如何接入

如果是后台管理默认布局页面，推荐使用：

```vue
<PersonalHomepageDesigner
  mode="manage"
  title="首页默认布局管理"
  description="维护租户级或公共级首页默认布局"
  :show-scope-selector="true"
/>
```

页面上的使用方式：

- 选择租户级或公共级
- 打开编辑模式
- 调整卡片显隐、顺序、参数
- 点击保存

## 模块首页如何接入

基础信息、审批管理、系统管理的模块首页复用同一套拖拽配置能力，入口组件为：

```text
Forgex_MOM/Forgex_Fronted/src/components/module-homepage/ModuleHomepageDesigner.vue
```

页面中按模块编码挂载即可：

```vue
<ModuleHomepageDesigner module-code="basic" />
```

系统管理、审批管理如果已有业务图表或列表卡片，不要把页面退回成通用占位卡片，而是把真实业务卡片作为命名插槽放进 `ModuleHomepageDesigner`。插槽名称与 widget key 保持一致，布局、显隐、拖拽、缩放仍由设计器统一处理：

```vue
<ModuleHomepageDesigner
  module-code="sys"
  :hidden-widget-keys="legacyWidgetKeys"
  @layout-updated="handleHomepageLayoutUpdated"
>
  <template #systemCpu>
    <a-card class="chart-card">
      <div ref="cpuChartRef" class="echart-container"></div>
    </a-card>
  </template>
</ModuleHomepageDesigner>
```

使用约定：

- `hiddenWidgetKeys` 用于隐藏历史兼容 widget 或当前权限不可用的 widget，隐藏后不出现在网格和右侧配置面板。
- `layout-updated` 会在网格加载、拖拽、缩放、显隐变化或切换配置状态后触发；图表页应在该事件中 `nextTick` 后初始化或 resize ECharts。
- 基础信息首页目前继续使用模块首页通用卡片，只处理全宽与拖拽配置，不额外补图表。

当前已接入的模块首页：

| 模块 | 页面 | moduleCode |
|---|---|---|
| 基础信息 | `src/views/basic/dashboard/index.vue` | `basic` |
| 审批管理 | `src/views/workflow/dashboard/index.vue` | `approval` |
| 系统管理 | `src/views/system/dashboard/index.vue` | `sys` |

系统配置的“个人首页”页签左侧提供模块卡片：个人首页、基础信息、审批管理、系统管理。点击模块卡片后，右侧切换到对应首页设计器；个人首页仍走原个人配置，模块首页支持公共级和租户级配置。

模块首页接口复用个人首页配置 API，并通过可选 `moduleCode` 区分配置维度。未传 `moduleCode` 时保持原个人首页行为。

## 页面如何使用编辑模式

组件内部已经封装了编辑模式按钮，页面通常不需要自己再做一套“进入编辑”的逻辑。

用户在页面上的操作路径通常是：

1. 点击“进入编辑模式”
2. 拖拽卡片位置
3. 在右侧面板调整显隐和数量
4. 点击“保存布局”

如果是当前用户模式，还可以点击“恢复默认”。

## 页面如何使用保存能力

保存按钮由组件内部处理。
页面开发者只需要正确选择模式：

- `current`
- `manage`

组件就会自动调用对应接口。

## 页面如何扩展新的 widget

如果要新增首页卡片，推荐按以下顺序处理：

1. 在 `PERSONAL_HOMEPAGE_WIDGET_KEYS` 中加入新 key
2. 在 `createDefaultPersonalHomepageConfig()` 中补默认布局
3. 在 `PersonalHomepageDesigner.vue` 中补渲染逻辑
4. 如需业务数据，再补对应 API 调用
5. 在右侧配置面板中补参数设置

## 收藏管理页如何使用

收藏管理页并不直接使用设计器，而是作为个人首页体系的配套页面。

参考：

```text
Forgex_MOM/Forgex_Fronted/src/views/home/favorite-management/index.vue
```

页面上的使用方式：

- 查看当前全部收藏菜单
- 上移、下移排序
- 批量取消收藏
- 保存排序

这会间接影响首页中的“我的收藏”卡片展示效果。

## 页面开发建议

- 首页入口页尽量保持轻量，只挂设计器组件
- 业务数据接口放在设计器内部统一管理
- 新 widget 接入时先补默认配置，再补渲染
- 当前用户模式和管理模式不要混用同一套保存语义

## 常见排查

### 页面接了组件但首页空白

优先检查：

1. 是否正确传了 `mode`
2. 相关接口是否可用
3. 页面容器是否有足够高度

### 拖拽正常但保存没生效

优先检查：

1. 当前模式是否正确
2. 保存接口是否成功返回
3. 是否又被重新加载的旧配置覆盖

### 新 widget 加了配置但页面不显示

优先检查：

1. 是否在设计器模板中补了对应渲染分支
2. 是否被默认设置成了 `visible=false`
