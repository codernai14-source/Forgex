# 个人首页与可拖拽布局实现逻辑

> 分类：前端 / 国际化与布局  
> 版本：**V0.6.0**  
> 更新时间：**2026-04-17**

本文档说明个人首页设计器的实现逻辑，包括首页布局配置、组件拖拽、当前用户配置与租户/公共配置的关系，以及代码具体位置。

## 功能定位

Forgex 的个人首页不是写死的一张首页，而是一套“可配置的首页布局系统”。

它主要解决以下问题：

- 首页卡片可以拖拽排序
- 小组件可以显示或隐藏
- 当前用户可以保存自己的首页布局
- 管理端可以维护租户级、公共级默认布局
- 首页数据和布局配置分离

核心组件是：

```text
PersonalHomepageDesigner.vue
```

## 布局配置模型

布局配置类型定义在：

```text
Forgex_MOM/Forgex_Fronted/src/api/system/personalHomepage.ts
```

主要结构分为两层：

### layout

描述网格整体规则，例如：

- 列数
- 行高
- 横向纵向间距
- 平板和移动端列数

### widgets

描述每个卡片，例如：

- `key`
- `visible`
- `x/y`
- `w/h`
- `minW/minH`
- `orderNum`
- `params`

这意味着首页的“形”和“内容”是分开的：

- `layout` 决定怎么摆
- `widgets` 决定摆什么、显示什么

## 默认配置逻辑

默认配置在 `createDefaultPersonalHomepageConfig()` 中生成。

系统内置了以下小组件：

- `commonMenus`
- `myFavorites`
- `pendingApprovals`
- `calendar`
- `messages`
- `notices`
- `currentTime`

这套默认配置是首页的兜底方案。  
即使后端没有返回配置，前端也能先渲染出一张完整首页。

## 配置获取与保存链路

前端区分两套模式：

### current 模式

给当前登录用户使用。  
对应接口：

- `getCurrentPersonalHomepageConfig()`
- `saveCurrentPersonalHomepageConfig()`
- `resetCurrentPersonalHomepageConfig()`

### manage 模式

给后台管理默认布局使用。  
对应接口：

- `getManagePersonalHomepageConfig(scopeLevel)`
- `saveManagePersonalHomepageConfig(scopeLevel, config)`

这里的 `scopeLevel` 用来区分：

- `TENANT`
- `PUBLIC`

## 配置合并逻辑

前端拿到后端配置后，会先经过：

```ts
mergePersonalHomepageConfig(config)
```

它做的事情包括：

- 用默认布局补齐缺失字段
- 将后端已有 widget 合并到默认 widget
- 保证内置卡片始终存在
- 对某些卡片参数做强制修正

例如：

- 常用菜单固定 Top 6
- 我的收藏固定保留“更多”入口

这说明首页不是完全自由拖拽，而是在“可配置范围内可拖拽”。

## 网格拖拽实现

真正的拖拽布局由：

```text
vue-grid-layout-v3
```

实现。

`PersonalHomepageDesigner.vue` 中通过：

- `GridLayout`
- `GridItem`

渲染每个 widget。

工作流程如下：

1. 前端根据 `config.widgets` 生成 `gridLayout`
2. 用户拖拽或缩放卡片
3. `gridLayout` 变化后反向同步回 `config.widgets`
4. 点击保存时提交整个配置

因此页面上拖的是显示层，真正保存的是配置层。

## 数据与布局分离逻辑

个人首页有两类数据：

### 布局数据

比如卡片位置、显隐、宽高、限制条数。  
这类数据保存到个人首页配置接口。

### 业务数据

比如：

- 常用菜单
- 收藏菜单
- 待审批
- 未读消息

这类数据来自各自业务接口，例如：

- `getUserCommonMenus`
- `getUserFavoriteMenus`
- `pageMyPending`
- `listUnreadMessages`

这样设计的好处是：首页布局不依赖具体业务数据结构变化。

## 当前首页与收藏管理页的关系

`src/views/home/index.vue` 本身非常轻，只负责挂载：

```vue
<PersonalHomepageDesigner mode="current" />
```

收藏管理页则是个人首页体系的一个配套页面，用于：

- 查看全部收藏菜单
- 调整收藏顺序
- 批量取消收藏

也就是说，个人首页设计器负责“首页显示”，收藏管理页负责“收藏治理”。

## 为什么这样设计

### 1. 首页个性化与平台治理都能兼顾

用户可以改自己的首页，管理员也能提供默认首页。

### 2. 页面扩展性更强

新增一个 widget，不需要重写整张首页，只要补默认配置和渲染逻辑。

### 3. 布局持久化简单

前端只需要保存 JSON 配置，不需要存一堆分散字段。

## 常见排查

### 首页卡片拖了但刷新后恢复原样

优先检查：

1. 是否点击了保存
2. 保存接口是否成功
3. 当前模式是 `current` 还是 `manage`

### 某个 widget 不显示

优先检查：

1. `config.widgets` 中该项是否 `visible=false`
2. 该 widget 的业务数据是否加载失败

### 收藏菜单数量不对

优先检查：

1. 当前逻辑是否被固定为 Top 6
2. 是否在收藏管理页重新保存过排序

### 管理模式下保存了但普通用户首页没变化

优先检查：

1. 当前用户是否已有自己的 `current` 配置
2. 租户级或公共级配置是否只是默认值，没有覆盖用户级配置
