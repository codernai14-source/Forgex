# 个人首页与可拖拽布局

> 分类：前端 / 国际化与布局  
> 版本：**V0.5.0**  
> 更新时间：**2026-04-13**

## 一、概述

Forgex 前端已提供可拖拽首页配置能力，核心组件位于：

- `Forgex_MOM/Forgex_Fronted/src/components/personal-homepage/PersonalHomepageDesigner.vue`
- `Forgex_MOM/Forgex_Fronted/src/api/system/personalHomepage.ts`

这套能力用于支持：

- 当前用户个人首页布局定制
- 管理端维护公共 / 租户默认首页布局
- 组件拖拽排序、尺寸调整、显隐控制
- 首页卡片数据联动（常用菜单、收藏、待审批、消息、通知、时钟等）

---

## 二、能力定位

它不是普通静态首页，而是一个**配置驱动 + 可视化拖拽**的首页设计器。

整体结构可理解为：

```text
Hero 区
  + 工具栏（编辑 / 刷新 / 保存 / 重置）
  + GridLayout 可拖拽画布
  + 右侧配置面板（编辑模式下显示）
```

其中：

- 普通用户场景：编辑自己的首页布局
- 管理端场景：配置租户级 / 公共默认布局

---

## 三、核心概念

### 3.1 三种作用域

后端和前端共同支持首页配置作用域：

| 作用域 | 说明 |
|---|---|
| `USER` | 当前用户自己的首页配置 |
| `TENANT` | 当前租户默认配置 |
| `PUBLIC` | 全局公共默认配置 |

### 3.2 两种运行模式

组件 props 中定义了两种模式：

| 模式 | 说明 |
|---|---|
| `current` | 当前用户首页 |
| `manage` | 管理配置模式 |

对应行为差异：

- `current`：支持“恢复默认布局”，并加载个人摘要信息；
- `manage`：支持管理 `TENANT / PUBLIC` 作用域配置。

---

## 四、默认布局结构

前端默认配置由 `createDefaultPersonalHomepageConfig()` 生成，包含：

- `layout`
- `widgets`

### 4.1 布局字段

| 字段 | 说明 |
|---|---|
| `colNum` | 桌面端列数 |
| `rowHeight` | 每行高度 |
| `marginX` | 横向间距 |
| `marginY` | 纵向间距 |
| `tabletColNum` | 平板列数 |
| `mobileColNum` | 手机列数 |

### 4.2 默认内置组件

| key | 说明 |
|---|---|
| `commonMenus` | 常用菜单 |
| `myFavorites` | 我的收藏 |
| `pendingApprovals` | 我收到的审批 |
| `calendar` | 日历 |
| `messages` | 我收到的消息 |
| `notices` | 系统通知 |
| `currentTime` | 当前时间 |

每个组件都包含：

- 位置 `x/y`
- 尺寸 `w/h`
- 最小尺寸 `minW/minH`
- 排序号 `orderNum`
- 参数 `params`

## 五、拖拽与布局引擎

组件使用：

- `vue-grid-layout-v3`

核心行为：

- `GridLayout` 负责整个画布
- `GridItem` 代表单个首页组件
- 编辑模式下允许拖拽与缩放

### 5.1 编辑模式开关

当 `editMode = true` 时：

- `is-draggable = true`
- `is-resizable = true`
- 右侧配置面板显示
- 顶部出现保存按钮

### 5.2 布局同步逻辑

组件内部维护两条同步链：

```text
配置 → gridLayout
  syncGridFromConfig()

gridLayout → 配置
  syncConfigFromGrid()
```

这保证了：

- 从后端加载的布局能正确渲染；
- 用户拖动后的结果能实时回写到配置对象；
- 保存时直接提交统一配置结构。

## 六、接口与配置保存

前端 API 位于：

- `src/api/system/personalHomepage.ts`

### 6.1 当前用户配置

| 接口 | 说明 |
|---|---|
| `POST /sys/homepage/current/get` | 获取当前用户首页配置 |
| `POST /sys/homepage/current/save` | 保存当前用户首页配置 |
| `POST /sys/homepage/current/reset` | 恢复当前用户默认配置 |
| `GET /sys/homepage/summary` | 获取个人首页摘要信息 |

### 6.2 管理配置

| 接口 | 说明 |
|---|---|
| `POST /sys/homepage/manage/get` | 获取租户 / 公共默认首页配置 |
| `POST /sys/homepage/manage/save` | 保存租户 / 公共默认首页配置 |

### 6.3 关联数据接口

首页卡片数据还会联动：

- 常用菜单
- 收藏菜单
- 未读消息
- 待审批任务

对应 API 也都已收敛到 `personalHomepage.ts` 与消息 / 工作流模块接口中。

---

## 七、卡片数据联动

### 7.1 常用菜单与收藏

首页会读取：

- 当前用户常用菜单
- 当前用户收藏菜单
- 收藏切换 / 排序能力

并在组件中支持：

- 打开菜单
- 收藏/取消收藏
- 跳转收藏管理页面
- 展示模块名称与图标

### 7.2 待审批

读取工作流待办列表，按组件参数中的 `limit` 控制条数。

### 7.3 消息与通知

读取未读站内消息，并区分：

- 普通消息
- 通知 / 告警类消息

收到消息事件后还会刷新对应卡片。

### 7.4 当前时间与日历

- 当前时间卡片每秒刷新
- 日历根据当前语言环境做本地化格式化

## 八、国际化能力

首页设计器不是只有标题翻译，而是整套页面都受国际化驱动：

- Hero 标题与描述
- 组件名称、副标题、空状态
- 星期、日期格式
- 问候语（含中英文差异化称呼）
- 按钮文案与提示文本

### 8.1 语言切换后的表现

切换语言后：首页中的以下内容会同步变化：

- 卡片标题
- 时间/日期格式
- 问候语
- 工具栏文案
- 空状态与更多按钮文案

---

## 九、推荐接入方式

### 9.1 当前用户首页

适用于“我的首页”、“工作台首页”等入口。

```vue
<PersonalHomepageDesigner mode="current" />
```

### 9.2 管理配置页

适用于系统管理员维护默认布局：

```vue
<PersonalHomepageDesigner
  mode="manage"
  :show-scope-selector="true"
  initial-scope-level="TENANT"
/>
```

## 十、适合扩展的方向

这套体系非常适合后续继续扩展：

- 新增首页组件卡片
- 新增图表型组件
- 增加每个卡片更细的参数配置
- 支持租户模板一键同步到用户
- 支持更多响应式断点布局

### 10.1 新增组件时建议同步处理

1. 在默认配置中加入新 widget key；
2. 增加标题 / 副标题 / 空状态 i18n 文案；
3. 在 `PersonalHomepageDesigner.vue` 中补渲染分支；
4. 设计对应更多动作与参数项；
5. 后端配置结构保持兼容。

---

## 十一、实践建议

1. **用户首页与管理默认布局分开理解**：一个是个性化，一个是模板能力；
2. **新增组件时优先保持配置结构稳定**，避免历史布局失效；
3. **卡片内容应尽量复用已有 API，而不是为首页单独再造一套接口**；
4. **多语言标题优先走 i18n key，不要把中文标题硬编码在页面逻辑中**；
5. **保存成功提示建议继续交给统一 HTTP 层托管。**
