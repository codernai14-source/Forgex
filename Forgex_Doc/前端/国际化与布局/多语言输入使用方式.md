# 多语言输入使用方式

> 分类：前端 / 国际化与布局
> 版本：**V0.6.5**
> 更新时间：**2026-04-17**

本文档说明在页面中如何使用 `I18nInput` 与 `I18nJsonEditor`，重点覆盖代码接入方式和页面操作方式。

## 代码位置

```text
Forgex_MOM/Forgex_Fronted/src/components/common/I18nInput.vue
Forgex_MOM/Forgex_Fronted/src/components/common/I18nJsonEditor.vue
```

## I18nInput 最常见用法

### 1. 单行输入

适合菜单名称、标题类字段。

```vue
<I18nInput
  v-model="form.nameI18nJson"
  mode="simple"
  placeholder="请输入名称"
/>
```

页面上的使用方式：

- 默认只显示默认语言文本
- 点击右侧地球图标后打开完整多语言弹窗

### 2. 多行文本

适合模板正文、描述信息。

```vue
<I18nInput
  v-model="form.bodyI18nJson"
  mode="simple"
  type="textarea"
  :rows="6"
  placeholder="请输入正文"
/>
```

### 3. 带占位符工具栏

适合消息模板正文。

```vue
<I18nInput
  v-model="form.contentBodyI18nJson"
  mode="simple"
  type="textarea"
  :rows="6"
  :show-placeholders="true"
/>
```

## 页面如何使用 I18nInput

推荐写法是直接把它绑定到表单里的 `xxxI18nJson` 字段。

例如菜单页面：

```vue
<a-form-item label="菜单名称" name="nameI18nJson">
  <I18nInput
    v-model="formData.nameI18nJson"
    mode="simple"
  />
</a-form-item>
```

例如消息模板页面：

```vue
<a-form-item label="正文" required>
  <I18nInput
    v-model="formData.contents[index].contentBodyI18nJson"
    mode="simple"
    type="textarea"
    :rows="6"
    :show-placeholders="true"
  />
</a-form-item>
```

## 页面提交时怎么处理

通常不需要再自己 `JSON.stringify`，因为组件输出的本来就是字符串。

也就是说，保存时可以直接提交：

```ts
await saveApi({
  nameI18nJson: formData.nameI18nJson,
  contentBodyI18nJson: formData.contentBodyI18nJson,
})
```

## 页面回显时怎么处理

详情接口返回什么，表单里就直接接什么。
如果接口返回的是：

```ts
{
  nameI18nJson: '{"zh-CN":"菜单管理","en-US":"Menu"}'
}
```

那就直接赋给表单字段即可。

## I18nJsonEditor 如何使用

适合简单双语维护场景。

```vue
<I18nJsonEditor v-model="form.textI18nJson" />
```

页面上的使用方式：

- 左侧编辑中文
- 右侧编辑英文
- 下方实时预览 JSON

## 什么时候选 I18nInput，什么时候选 I18nJsonEditor

推荐使用 `I18nInput` 的场景：

- 系统启用语言是动态的
- 需要默认语言简洁输入
- 需要弹窗编辑
- 需要占位符

推荐使用 `I18nJsonEditor` 的场景：

- 只维护中英文
- 页面本身就是开发或配置场景
- 需要看到 JSON 结构预览

## 页面使用建议

- 普通业务表单优先用 `I18nInput`
- 多语言正文场景开启 `textarea`
- 模板类字段开启 `showPlaceholders`
- 只有在确实只维护双语时再用 `I18nJsonEditor`

## 常见排查

### 表单校验总是不过

优先检查：

1. 校验字段是否绑定到 `xxxI18nJson`
2. 页面是否把空字符串误当成对象处理

### 页面回显了原始 JSON，而不是默认语言文本

通常是没有使用 `I18nInput`，而是直接把 JSON 字符串塞进了普通输入框。

### 保存后丢失多语言内容

优先检查：

1. 保存接口是否接收 `xxxI18nJson`
2. 页面保存时是否被错误地覆盖成普通文本字段
