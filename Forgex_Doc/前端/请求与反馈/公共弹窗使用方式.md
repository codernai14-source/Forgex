# 公共弹窗使用方式

> 分类：前端 / 请求与反馈  
> 版本：**V0.6.0**  
> 更新时间：**2026-04-17**

本文档说明 `BaseFormDialog.vue` 在页面中如何接入、如何组织表单代码，以及如何与接口保存联动。

## 最小接入方式

```vue
<BaseFormDialog
  v-model:open="dialogVisible"
  title="新增用户"
  :loading="saving"
  @submit="handleSubmit"
>
  <UserForm ref="formRef" />
</BaseFormDialog>
```

## 标准页面写法

推荐拆成两层：

### 1. 页面层

负责：

- 控制 `dialogVisible`
- 传入当前编辑记录
- 监听保存成功后刷新列表

### 2. 表单弹窗层

负责：

- 渲染 `BaseFormDialog`
- 承载具体表单
- 在 `submit` 里调用校验和接口保存

这样页面不会因为弹窗表单太重而变得难维护。

## 标准提交流程

```text
打开弹窗
  -> 回显数据
  -> 用户编辑
  -> 点击确定
  -> 表单校验
  -> 调接口保存
  -> 成功后关闭弹窗
  -> 通知页面刷新列表
```

## 代码示例

```ts
const saving = ref(false)

async function handleSubmit() {
  const values = await formRef.value?.validate()
  saving.value = true
  try {
    if (values.id) {
      await updateThirdSystem(values)
    } else {
      await addThirdSystem(values)
    }
    dialogVisible.value = false
    emit('success')
  } finally {
    saving.value = false
  }
}
```

## 遮罩点击关闭

`BaseFormDialog` 默认 `maskClosable=false`，用于避免复杂表单在误点遮罩时丢失未保存内容。

如果业务明确要求点击弹窗外部关闭，在页面调用处显式开启：

```vue
<BaseFormDialog
  v-model:open="visible"
  title="新增数据"
  :mask-closable="true"
  @submit="handleSubmit"
/>
```

当前已采用该策略的页面：

```text
Forgex_MOM/Forgex_Fronted/src/views/system/codegenDatasource/components/CodegenDatasourceFormDialog.vue
Forgex_MOM/Forgex_Fronted/src/views/basic/supplier/index.vue
```

## 什么时候用 `modal`，什么时候用 `drawer`

| 场景 | 推荐模式 | 说明 |
|---|---|---|
| 字段少、操作集中 | `modal` | 短表单更紧凑 |
| 字段多、内容长、需要更多横向空间 | `drawer` | 更适合复杂配置页面 |

示例：

```vue
<BaseFormDialog
  v-model:open="visible"
  mode="drawer"
  title="编辑接口"
  :width="720"
  @submit="handleSubmit"
>
  <ApiConfigForm />
</BaseFormDialog>
```

## 页面使用建议

1. 提交 loading 统一绑定到 `:loading`。
2. 接口成功提示优先交给 `http` / `httpSuccess` 托管。
3. 弹窗内部不要重复写一套取消按钮逻辑。
4. 复杂表单优先拆子组件，不要把所有字段堆在列表页本身。

## 常见排查

| 现象 | 处理建议 |
|---|---|
| 点击确定后不保存 | 检查 `@submit` 是否绑定 |
| 表单校验没触发 | `formRef` 是否暴露了 `validate` |
| 保存成功后列表没刷新 | 页面层是否监听成功事件并 reload |
| 弹窗再次打开还是旧数据 | 是否在打开时重置表单或启用 `destroyOnClose` |
