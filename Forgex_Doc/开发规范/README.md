| 类别 | 内容 |
|---|---|
| 架构设计 | 仓库结构、服务边界、上下文透传、前后端协作关系 |
| Git 规范 | 提交消息、分支管理、协作流程 |
| 数据库规范 | 字段统一规范、命名与通用审计字段 |

| 注释规范 | Java / TypeScript 注释基线 |

| 测试规范 | AI 自动化测试范围、MCP 使用方式、前端点击流程与报告模板 |

- `doc/开发规范/公共表格组件使用说明与实现逻辑.md`：继续作为专题实现说明保留。

## 六、维护原则

1. **架构变更先更新这里，再修改模块文档。**
2. **通用规则写在这里，模块差异写到各自专题文档。**
3. **新的公共规范，应优先放在 `Forgex_Doc/开发规范`。**
