# 国际化提示消息使用指南

> 分类：开发规范 / 规范文档

这是国际化提示消息规范的提交版入口页。

## 硬性要求：禁止使用 `LegacyMessageTranslator`

- `LegacyMessageTranslator` 仅允许作为历史硬编码提示的临时兼容兜底，禁止新代码直接调用、封装调用或继续新增词条。
- 新增和改造后的后端返回消息、异常消息、参数校验消息，必须定义 `I18nPrompt` / 模块 Prompt 枚举，并在 `fx_i18n_message` 中维护多语言配置。
- 不允许在 Controller、Service、DTO 校验注解、全局异常处理分支中继续写死中文、英文或其他语言提示文案；确需默认文案时，只能作为 Prompt 默认模板和数据库初始化数据的来源。
- Bean Validation 等参数校验提示应走统一异常与数据库配置链路；不得通过向 `LegacyMessageTranslator` 补词条来“解决”新增校验文案的国际化问题。
- 发现缺失翻译时，优先补齐 `fx_i18n_message` 或对应配置页面数据；不能把硬编码翻译逻辑写回代码。

## 适用范围

- 后端 `I18nPrompt` / `R<T>` / 国际化异常
- 前端自动提示消息消费
- 统一消息模板设计与多语言回退
