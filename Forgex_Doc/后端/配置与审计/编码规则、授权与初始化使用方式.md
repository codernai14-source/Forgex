# 编码规则、授权与初始化使用方式

> 分类：后端 / 配置与审计
> 版本：**V0.6.5**
> 更新时间：**2026-04-17**

本文档说明编码规则、授权和初始化能力在实施和开发中的使用方式。

## 编码规则

适用场景：

- 物料编码
- 单据编码
- 业务流水号

推荐做法：

1. 业务只传稳定的 `ruleCode`。
2. 编码生成统一走服务层，不要页面或 Controller 自己拼。
3. 并发编号场景要验证 Redis / 序列一致性。

## 软件授权

常用接口：

| 接口 | 用途 |
|---|---|
| `GET /api/sys/license/status` | 查询授权状态 |
| `GET /api/sys/license/request-info` | 获取申请文件内容 |
| `POST /api/sys/license/refresh` | 刷新授权状态 |

推荐排查顺序：

1. 看授权目录是否存在 `request-info.json`
2. 看是否存在 `license.lic`
3. 校验公钥、机器码、实例编码
4. 刷新授权或重启网关与系统服务

## 初始化向导

首次安装时建议按顺序：

1. 查询初始化状态
2. 补齐数据库、基础配置、授权
3. 执行初始化申请
4. 验证登录和模块状态

## 常见排查

| 现象 | 排查点 |
|---|---|
| 编码重复 | 规则配置、序列缓存、并发控制 |
| 系统提示未授权 | 授权目录、公钥、实例编码、网关状态 |
| 初始化接口不可用 | 网关白名单、系统服务是否启动 |
