# 数据字典与日志

> 分类：后端 / 配置与审计  
> 版本：**V0.5.0**  
> 更新时间：**2026-04-13**

## 一、数据字典

### 1.1 概述

数据字典提供统一的枚举值、下拉选项、标签样式配置能力，支持树形结构、国际化标签和租户隔离。

**核心服务：** `com.forgex.sys.service.impl.DictServiceImpl`

### 1.2 数据表结构

**字典表 `sys_dict`：**

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | bigint | 主键 |
| `dict_type` | varchar | 字典类型编码 |
| `dict_code` | varchar | 字典码值 |
| `dict_name` | varchar | 字典显示名 |
| `dict_value` | varchar | 字典值 |
| `dict_value_i18n_json` | text | 多语言 JSON，如 `{"zh-CN":"启用","en-US":"Enabled"}` |
| `tag_style_json` | text | 标签样式 JSON |
| `parent_id` | bigint | 父节点 ID（支持树形） |
| `node_path` | varchar | 节点路径（如 `root/system/status`） |
| `order_num` | int | 排序号 |
| `tenant_id` | bigint | 租户 ID（0 为公共配置） |
| `status` | boolean | 是否启用 |

### 1.3 核心接口

| 接口 | 说明 |
|---|---|
| `GET /sys/dict/tree` | 获取字典树 |
| `GET /sys/dict/items?type=xxx` | 按类型获取字典项列表 |
| `GET /sys/dict/itemsByPath?path=xxx` | 按路径获取字典项列表 |
| `POST /sys/dict/save` | 保存字典项 |
| `POST /sys/dict/delete` | 删除字典项 |

### 1.4 国际化标签

字典项的显示名支持多语言，通过 `dict_value_i18n_json` 字段存储：

```json
{
  "zh-CN": "进行中",
  "en-US": "In Progress",
  "zh-TW": "進行中",
  "ja-JP": "進行中",
  "ko-KR": "진행 중"
}
```

查询时根据 `LangContext.get()` 自动选择对应语言，找不到则回退到 `dict_name`。

### 1.5 标签样式

通过 `tag_style_json` 配置前端标签展示样式：

```json
{
  "color": "#52c41a",
  "backgroundColor": "#f6ffed",
  "borderColor": "#b7eb8f"
}
```

前端 `DictTag` 组件直接消费该样式配置。

### 1.6 缓存架构

字典服务采用 **Caffeine 本地缓存 + Redis 分布式缓存** 二级方案：

```
查询：本地缓存(30s) → Redis(24h) → 数据库
写入：清除本地缓存 → 发布 Redis Pub/Sub 通知 → 其他节点清除本地缓存
```

**Redis Key 规则：**
- 按类型：`dict:{tenantId}:{lang}:{type}:{code}`
- 按路径：`dict:{tenantId}:{lang}:path:{nodePath}`

### 1.7 公共字典与租户字典

- `tenant_id = 0`：公共字典，所有租户共享
- `tenant_id > 0`：租户私有字典
- 查询时先查当前租户，未找到再回退到公共字典

## 二、日志系统

### 2.1 登录日志

**服务类：** `com.forgex.sys.service.impl.LoginLogServiceImpl`

| 功能 | 说明 |
|---|---|
| 记录登录成功 | 异步记录，包含用户、IP、地区、UA、Token |
| 记录登录失败 | 异步记录，包含账号、IP、失败原因 |
| 记录登出 | 回写登出时间到原登录日志 |

**表结构 `sys_login_log`：**

| 字段 | 说明 |
|---|---|
| `user_id` | 用户 ID |
| `account` | 账号 |
| `tenant_id` | 租户 ID |
| `login_ip` | 登录 IP |
| `login_region` | IP 归属地区 |
| `user_agent` | 浏览器 UA |
| `token_value` | Token（用于关联登出） |
| `login_time` | 登录时间 |
| `logout_time` | 登出时间 |
| `status` | 1=成功, 0=失败 |
| `fail_reason` | 失败原因 |

### 2.2 操作日志

通过 AOP 注解自动记录业务操作：

| 字段 | 说明 |
|---|---|
| `module` | 模块名 |
| `operation_type` | 操作类型（新增/编辑/删除/查询/导出等） |
| `method` | 方法签名 |
| `request_params` | 请求参数 JSON |
| `response_result` | 响应结果 JSON |
| `operator` | 操作人 |
| `ip` | 操作 IP |
| `cost_time` | 耗时（ms） |

### 2.3 日志特性

- **异步保存**：使用 `@Async` 避免阻塞业务请求
- **基本脱敏**：敏感字段（如 password）在日志中自动脱敏
- **详情模板化**：支持模板化的操作详情描述
