# 数据字典与日志实现逻辑

> 分类：后端 / 配置与审计
> 版本：**V0.6.5**
> 更新时间：**2026-04-17**

本文档说明字典配置、标签样式、缓存与日志审计在后端中的实现方式。

## 字典能力

`DictServiceImpl` 承担字典树、按类型取值、按路径取值、国际化标签解析等职责。

### 关键字段

| 字段 | 说明 |
|---|---|
| `dict_type` | 字典类型 |
| `dict_code` | 字典编码 |
| `dict_value_i18n_json` | 多语言显示值 |
| `tag_style_json` | 标签样式 |
| `tenant_id` | 租户归属 |

### 查询链路

```text
本地缓存
  -> Redis
  -> 数据库
  -> 回写缓存
```

字典查询默认带租户和语言维度，未命中时回退到 `tenant_id = 0`。

## 标签样式

前端 `DictTag` 会直接消费后端返回的 `tag_style_json`，因此字典既承载枚举值，也承载展示配置。

## 日志能力

### 登录日志

登录成功、失败、登出都通过专门服务异步记录，便于审计认证行为。

### 操作日志

业务操作一般通过注解和 AOP 方式自动记录模块、方法、请求参数、响应结果和耗时。

## 与其他能力的关系

- 与国际化：字典值按 `LangContext` 解析
- 与 Redis：依赖二级缓存与失效广播
- 与前端：支撑状态标签、下拉框、动态表格渲染
