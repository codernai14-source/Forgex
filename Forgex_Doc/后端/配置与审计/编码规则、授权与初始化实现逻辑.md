# 编码规则、授权与初始化实现逻辑

> 分类：后端 / 配置与审计  
> 版本：**V0.6.0**  
> 更新时间：**2026-04-17**

本文档说明编码规则、软件授权、初始化向导及相关系统级入口在后端中的实现结构。

## 入口控制器

| 控制器 | 作用 |
|---|---|
| `SysEncodeRuleController` | 编码规则配置、测试、生成 |
| `LicenseController` | 授权状态、申请信息、刷新 |
| `InitController` | 初始化状态与执行 |
| `SysModuleController` | 模块管理 |
| `OnlineUserController` | 在线用户 |
| `PersonalHomepageController` | 首页配置入口 |

## 编码规则链路

```text
业务请求生成编码
  -> 按 ruleCode 读取编码规则
  -> 根据日期、前缀、流水号配置生成
  -> 依赖 Redis / 数据库维持序列一致性
```

## 授权链路

```text
系统启动 / 网关请求
  -> 读取授权目录
  -> 解析 request-info.json / license.lic
  -> 校验签名、机器码、实例编码、有效期
  -> 网关决定放行或拒绝
```

## 初始化链路

初始化向导通常在系统首次安装后使用：

```text
GET /sys/init/status
  -> 判断是否已初始化
POST /sys/init/apply
  -> 执行初始化脚本与基础配置写入
```

## 系统级配置入口

模块管理、在线用户、租户初始化任务、首页配置等能力虽然分属不同页面，但都属于 `Forgex_Sys` 的平台配置型入口。
