# 编码规则、授权与初始化实现逻辑

> 分类：后端 / 配置与审计
> 版本：**V0.6.5**
> 更新时间：**2026-04-17**

本文档说明编码规则、软件授权、初始化向导及相关系统级入口在后端中的实现结构。

## 入口控制器

| 控制器 | 作用 |
|---|---|
| `SysEncodeRuleController` | 编码规则配置、测试、生成 |
| `LicenseController` | 授权状态、申请信息、刷新 |
| `InitController` | 初始化状态与执行 |
| `SysModuleController` | 模块管理 |
| `OnlineUserController` | 在线用户 |
| `PersonalHomepageController` | 首页配置入口 |

## 编码规则链路

```text
业务请求生成编码
  -> 按 ruleCode 读取编码规则
  -> 根据日期、前缀、流水号配置生成
  -> 依赖 Redis / 数据库维持序列一致性
```

编码规则分页、列表和详情查询会在租户插件忽略上下文中显式读取 `tenant_id in (0, 当前租户)` 的规则，确保公共初始化规则能够在当前租户页面展示。新增或编辑规则仍按当前登录租户写入；编码生成时优先匹配当前租户规则，未配置时可使用 `tenant_id=0` 的公共规则。

## 授权链路

```text
系统启动 / 网关请求
  -> 读取授权目录
  -> 解析 request-info.json / license.lic
  -> 校验签名、机器码、实例编码、有效期
  -> 网关决定放行或拒绝
```

## 初始化链路

初始化向导通常在系统首次安装后使用：

```text
GET /sys/init/status
  -> 判断是否已初始化
POST /sys/init/apply
  -> 执行初始化脚本与基础配置写入
```

初始化服务会写入系统管理、基础信息、审批管理等模块的默认菜单，并把菜单权限授权给系统管理员角色。

基础信息模块当前初始化菜单包括：

| 菜单 | component | 权限 |
|---|---|---|
| 基础信息主页 | `BasicDashboard` | `basic:dashboard:view` |
| 供应商管理 | `BasicSupplier` | `basic:supplier:query` |
| 客户管理 | `BasicCustomer` | `basic:customer:query` |
| 物料管理 | `BasicMaterial` | `basic:material:query` |
| 编码规则管理 | `BasicEncodeRule` | `basic:encodeRule:query` |

编码规则页面归属基础信息模块后，后端 `SysEncodeRuleController` 的权限码统一使用 `basic:encodeRule:*`。物料管理由一个菜单入口承载，前端页面内提供 `全部 / 原材料 / 半成品 / 成品` 四个 Tab，旧组件键 `BasicMaterialRaw`、`BasicMaterialSemiFinished`、`BasicMaterialFinished` 仅用于兼容历史缓存路由并统一加载 `BasicMaterial`。物料按钮共享 `basic:material:query/add/edit/delete` 权限，供应商初始化按钮权限与接口保持一致，包含导入、导出、生成供应商租户、审查和同步。

已有环境如存在空的 `basicInfo / 基础信息` 目录菜单、三类物料子菜单、缺少客户字段、缺少 `CustomerMasterTable` 或 `MaterialTable` 公共表格配置，可执行修复脚本：

```text
doc/sql/20260429_fix_basic_info_material_customer_encode_rule.sql
```

## 菜单打开次数与首次引导

系统管理模块菜单打开次数单独记录在 `sys_user_menu_open_count`，避免复用会裁剪 Top 6 的常用菜单表。

上报入口：

```text
POST /sys/menu/personal/open/report
```

请求包含菜单路径、标题、模块编码等信息，响应返回 `openCount` 和 `firstOpen`。前端目前仅对系统管理模块菜单上报打开次数；当 `firstOpen=true` 时触发对应菜单引导，引导组件支持“跳过引导”并复用原有跳过/关闭持久化逻辑。

## 系统级配置入口

模块管理、在线用户、租户初始化任务、首页配置等能力虽然分属不同页面，但都属于 `Forgex_Sys` 的平台配置型入口。
