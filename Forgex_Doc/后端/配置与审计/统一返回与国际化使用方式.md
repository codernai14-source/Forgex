# 统一返回与国际化使用方式

> 分类：后端 / 配置与审计  
> 版本：**V0.6.0**  
> 更新时间：**2026-04-17**

本文说明后端统一返回与国际化能力在开发中的推荐使用方式，重点覆盖“代码怎么写”和“页面怎么配合使用”。

## 代码如何使用

### 成功返回

```java
return R.ok();
return R.ok(data);
return R.ok(CommonPrompt.SAVE_SUCCESS);
return R.okWithArgs(CommonPrompt.SAVE_SUCCESS, "用户信息");
return R.okWithArgsAndData(CommonPrompt.UPDATE_SUCCESS, data, "部门");
```

### 失败返回

```java
return R.fail(CommonPrompt.OPERATION_FAILED);
return R.fail(StatusCode.NOT_FOUND, CommonPrompt.NOT_FOUND);
throw new I18nBusinessException(CommonPrompt.NOT_FOUND);
throw new I18nBusinessException(CommonPrompt.ACCOUNT_LOCKED, 30);
```

### 参数校验和异常

推荐：

- 业务可预期失败，优先 `I18nBusinessException`
- 普通错误收敛到 `R.fail(prompt)`
- 不要在 Controller 里直接返回中文字符串
- 存量 `BusinessException` 抛出点迁移时，需同步定义模块 Prompt 枚举，并在 `fx_i18n_message` 中补齐对应多语言配置

## 新模块接入步骤

1. 定义 Prompt 枚举并实现 `I18nPrompt`
2. Controller / Service 统一使用 `R` 或 `I18nBusinessException`
3. 在 `fx_i18n_message` 中维护多语言文案
4. 联调时确认请求头带 `X-Lang`

## 页面如何使用

### 前端统一请求层

前端真实入口：

- `Forgex_MOM/Forgex_Fronted/src/api/http.ts`

前端已经自动做了：

- 请求附带 `Accept-Language`
- 请求附带 `X-Lang`
- 错误消息统一提示
- 写操作成功消息自动提示

所以页面调用推荐直接：

```ts
await http.post('/sys/department/update', form)
```

不建议再额外写：

```ts
message.success('修改成功')
```

### 什么时候用静默请求

可以用 `silentHttp` 的场景：

- 定时刷新
- 未读数轮询
- 初始化查询
- 不想打断用户的预读接口

不建议用在：

- 保存
- 删除
- 提交
- 授权

这类需要明确反馈的操作。

## 设计建议

### 1. 共性消息进 `CommonPrompt`

例如：

- 保存成功
- 参数错误
- 未登录
- 数据不存在

### 2. 业务消息进模块 Prompt

例如：

- 工作流审批通过
- 报表发布失败
- 集成平台回调超时

### 3. 租户差异文案

`sys_response_message_template` 具备租户级返回文案覆盖能力，但当前没有对应的后台配置页面。配置页面落地前，默认仍以 `fx_i18n_message` 的公共多语言文案为准；确有租户差异需求时，应先补齐“返回消息模板配置”页面，再开放租户级覆盖维护，不建议临时在业务代码中分支硬编码。

## 检查清单

- `module + promptCode` 是否唯一
- 默认文案是否填写
- `fx_i18n_message` 中是否存在对应记录
- 如存在租户级覆盖需求，是否已有可维护的后台配置入口
- 前端请求是否已带 `X-Lang`
- 页面是否误用了 `silentHttp`

## 常见问题

| 现象 | 排查点 |
|---|---|
| 前端不弹提示 | 是否用了 `silentHttp`，返回是否带 `message` |
| 返回未翻译 | `X-Lang`、`fx_i18n_message`、`RMessageI18nAdvice` |
| 错误码不统一 | 是否绕过了 `R` 或全局异常处理 |
| 同接口需要不同租户文案 | 先确认是否已建设返回消息模板配置页面，再评估 `sys_response_message_template` 覆盖 |
