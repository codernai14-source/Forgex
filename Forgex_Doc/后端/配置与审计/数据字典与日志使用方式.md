# 数据字典与日志使用方式

> 分类：后端 / 配置与审计  
> 版本：**V0.6.0**  
> 更新时间：**2026-04-17**

本文档说明数据字典与日志能力在开发、配置和排查中的推荐用法。

## 字典接入建议

1. 状态、类型、枚举优先走数据字典，不要前后端重复硬编码。
2. 有颜色标签需求时直接配置 `tag_style_json`。
3. 多语言场景优先填写 `dict_value_i18n_json`。
4. 公共字典用 `tenant_id = 0`，租户差异字典单独配置。

## 常用接口

| 接口 | 用途 |
|---|---|
| `GET /sys/dict/tree` | 获取字典树 |
| `GET /sys/dict/items?type=xxx` | 按类型取字典项 |
| `GET /sys/dict/itemsByPath?path=xxx` | 按路径取字典项 |
| `POST /sys/dict/save` | 保存字典 |
| `POST /sys/dict/delete` | 删除字典 |

## 日志使用建议

- 登录相关异常优先看登录日志。
- 业务接口执行异常优先看操作日志和统一返回消息。
- 敏感字段不要直接写入自定义日志。

## 常见排查

| 现象 | 排查点 |
|---|---|
| 前端标签颜色不对 | `tag_style_json` 是否正确 |
| 下拉项不显示 | 字典类型、租户维度、缓存是否刷新 |
| 国际化字典未生效 | `LangContext`、`dict_value_i18n_json` |
| 日志没记录 | AOP 注解、异步线程、日志开关 |
