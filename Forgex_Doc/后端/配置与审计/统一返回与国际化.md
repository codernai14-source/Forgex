# 统一返回与国际化

> 分类：后端 / 配置与审计  
> 版本：**V0.5.0**  
> 更新时间：**2026-04-13**

## 一、概述

Forgex 的统一返回模型 `R<T>` 和国际化链路天然联动：Controller 返回 `R` 携带消息代码，`RMessageI18nAdvice` 在响应阶段完成翻译，前端根据 `code/message` 自动提示。

## 二、统一返回模型 `R<T>`

### 2.1 类路径

`com.forgex.common.web.R`

### 2.2 字段结构

| 字段 | 类型 | JSON 可见 | 说明 |
|---|---|---|---|
| `code` | Integer | ✅ | 状态码，200 成功，非 200 失败 |
| `message` | String | ✅ | 消息文本（翻译后） |
| `data` | T | ✅ | 返回数据 |
| `messageCode` | I18nPrompt | ❌ | 国际化消息代码（内部使用） |
| `i18n` | I18nMeta | ❌ | 国际化元信息（跨服务翻译） |

### 2.3 状态码 `StatusCode`

| 常量 | 值 | 说明 |
|---|---|---|
| `SUCCESS` | 200 | 成功 |
| `NOT_FOUND` | 404 | 资源不存在 |
| `BUSINESS_ERROR` | 500 | 业务失败（默认） |
| `UNAUTHORIZED` | 601 | 无权限 |
| `NOT_LOGIN` | 602 | 未登录 |
| `MODULE_OFFLINE` | 603 | 模块未上线 |

### 2.4 使用方式

**成功返回：**

```java
// 无数据
return R.ok();

// 带数据
return R.ok(userList);

// 带提示消息
return R.ok(CommonPrompt.SAVE_SUCCESS);

// 带数据和提示
return R.ok(CommonPrompt.SAVE_SUCCESS, user);

// 带参数的提示消息（模板：{0}保存成功）
return R.okWithArgs(CommonPrompt.SAVE_SUCCESS, "用户信息");
```

**失败返回：**

```java
// 默认失败
return R.fail();

// 自定义消息
return R.fail(CommonPrompt.OPERATION_FAILED);

// 自定义状态码
return R.fail(StatusCode.NOT_FOUND, CommonPrompt.NOT_FOUND);

// 带参数
return R.fail(CommonPrompt.ORDER_STATUS, "订单A,开工");
```

### 2.5 I18nPrompt 接口

所有提示消息枚举需实现 `I18nPrompt`：

```java
public interface I18nPrompt {
    String getModule();      // 模块标识，如 "common", "excel", "auth"
    String getPromptCode();  // 提示码，如 "SAVE_SUCCESS"
    String getDefaultTemplate(); // 默认模板，如 "保存成功"
}
```

**公共提示枚举 `CommonPrompt`** 已内置常用提示码。

### 2.6 自定义模块提示枚举

```java
@Getter
public enum MyModulePrompt implements I18nPrompt {
    ORDER_CREATED("ORDER_CREATED", "订单创建成功"),
    ORDER_NOT_FOUND("ORDER_NOT_FOUND", "订单不存在");

    private final String promptCode;
    private final String defaultTemplate;

    @Override
    public String getModule() { return "mymodule"; }
}
```

## 三、异常体系

### 3.1 BusinessException

普通业务异常，由全局异常处理器捕获转为 `R.fail()`：

```java
throw new BusinessException("物料不存在");
```

### 3.2 I18nBusinessException

国际化业务异常，支持消息代码和参数：

```java
throw new I18nBusinessException(CommonPrompt.NOT_FOUND);
throw new I18nBusinessException(StatusCode.BUSINESS_ERROR, CommonPrompt.ORDER_STATUS, "订单A", "已关闭");
```

### 3.3 全局异常处理器 `GlobalExceptionHandler`

| 异常类型 | 处理方式 |
|---|---|
| `BusinessException` | → `R.fail(500, msg)` |
| `I18nBusinessException` | → `R.failWithArgs(code, prompt, args)` |
| `MethodArgumentNotValidException` | → `R.fail(500, BAD_REQUEST, 字段错误信息)` |
| `CannotGetJdbcConnectionException` | → `R.fail(500, DB_CONNECT_FAILED)` |
| `DataAccessException` | → `R.fail(500, DATA_ACCESS_ERROR)` |
| 无权限异常 | → `R.fail(601, NO_PERMISSION)` |

## 四、国际化链路

### 4.1 多语言配置

#### 语言类型表 `fx_i18n_language_type`

| 字段 | 说明 |
|---|---|
| `lang_code` | 语言代码，如 `zh-CN`, `en-US`, `zh-TW`, `ja-JP`, `ko-KR` |
| `lang_name` | 语言名称（本地语言） |
| `lang_name_en` | 英文名称 |
| `icon` | 图标 |
| `enabled` | 是否启用 |
| `is_default` | 是否默认语言 |
| `order_num` | 排序号 |

**管理接口：**

| 接口 | 说明 |
|---|---|
| `POST /sys/i18n/languageType/listEnabled` | 获取启用的语言列表 |
| `POST /sys/i18n/languageType/listAll` | 获取所有语言 |
| `POST /sys/i18n/languageType/getDefault` | 获取默认语言 |
| `POST /sys/i18n/languageType/create` | 新增语言类型 |
| `POST /sys/i18n/languageType/update` | 更新语言类型 |
| `POST /sys/i18n/languageType/delete` | 删除语言类型 |
| `POST /sys/i18n/languageType/setDefault` | 设置默认语言 |
| `POST /sys/i18n/languageType/page` | 分页查询 |

### 4.2 消息文案表 `fx_i18n_message`

| 字段 | 说明 |
|---|---|
| `module` | 模块标识（如 `common`, `excel`） |
| `prompt_code` | 提示码（如 `SAVE_SUCCESS`） |
| `message_json` | 多语言 JSON |

示例：

```json
// module=common, prompt_code=SAVE_SUCCESS
{
  "zh-CN": "保存成功",
  "en-US": "Saved successfully",
  "zh-TW": "儲存成功",
  "ja-JP": "保存しました",
  "ko-KR": "저장 성공"
}
```

### 4.3 语言上下文 `LangContext`

```java
// 路径：com.forgex.common.i18n.LangContext
// 基于 ThreadLocal 存储当前请求语言

// 设置语言
LangContext.set("en-US");

// 获取当前语言
String lang = LangContext.get();  // 默认 "zh-CN"

// 清理（在请求结束时自动调用）
LangContext.clear();
```

**语言解析优先级：**
1. 请求头 `X-Lang`
2. 请求头 `Accept-Language`
3. 默认语言 `zh-CN`

**语言标准化规则：**

| 输入 | 标准化结果 |
|---|---|
| zh, zh-cn, zh_CN | zh-CN |
| zh-tw, zh-hk, zh_TW | zh-TW |
| en, en-us, en_US | en-US |
| ja-JP, ko-KR | 保持原样 |

### 4.4 语言上下文注入

由 `LangWebInterceptor` 和 `langContextFilter` 在请求进入时自动设置：

```java
// WebMvcConfig 中注册
@Bean
public FilterRegistrationBean<Filter> langContextFilter() {
    // 从请求头解析语言
    // LangContext.set(lang)
    // chain.doFilter()
    // finally: LangContext.clear()
}
```

### 4.5 响应翻译 `RMessageI18nAdvice`

`ResponseBodyAdvice` 实现，在响应写回前自动翻译 `R` 中的 message：

```
Controller 返回 R.ok(CommonPrompt.SAVE_SUCCESS)
  → RMessageI18nAdvice 拦截响应
  → 读取 R.messageCode (module=common, code=SAVE_SUCCESS)
  → 调用 I18nMessageService.resolve(module, code, lang, args)
  → 从 fx_i18n_message 查询对应语言的文案
  → 设置 R.message = "保存成功" 或 "Saved successfully"
  → 最终返回给前端
```

### 4.6 I18nMessageService

```java
// 路径：com.forgex.common.service.i18n.impl.I18nMessageServiceImpl

// 解析规则：
// 1. 按 (module + promptCode) 查 fx_i18n_message
// 2. 取 message_json 中 LangContext.get() 对应的语言
// 3. 支持 zh/en 前缀兜底（zh-CN 找不到尝试 zh）
// 4. 都找不到则返回 defaultTemplate
// 5. 用 MessageFormat 填充参数
```

## 五、网关层国际化

网关 `GatewayAuthGlobalFilter` 也支持国际化消息缓存：

```
1. 拦截微服务响应
2. 如果响应中包含 i18n 元信息
3. 从 Redis 缓存查询翻译（key: i18nmsg:{module}:{code}:{lang}）
4. 缓存未命中则调用 sys 服务的 /sys/i18n/message/resolve 接口
5. 结果写入 Redis（TTL 24h）
6. 替换响应中的 message 字段
```

## 六、前后端消息衔接

```
后端: return R.ok(CommonPrompt.SAVE_SUCCESS)
       ↓ RMessageI18nAdvice 翻译
       ↓ 网关转发
前端: http.ts 响应拦截器
       ↓ code === 200 → message.success(msg)
       ↓ code !== 200 → message.error(msg)
```

## 七、接入指南总结

### 新模块接入国际化步骤

1. **定义 Prompt 枚举**，实现 `I18nPrompt`
2. **Controller 使用 `R.ok(prompt)` / `R.fail(prompt)`**
3. **在 `fx_i18n_message` 表维护多语言文案**
4. **确保请求头携带 `X-Lang`**（前端自动处理）

### 检查清单

- [ ] 枚举的 `module` + `promptCode` 是否唯一
- [ ] `fx_i18n_message` 中是否有对应记录
- [ ] 默认语言（zh-CN）的文案是否已填写
- [ ] 前端是否在 HTTP 请求头中传了 `X-Lang`
