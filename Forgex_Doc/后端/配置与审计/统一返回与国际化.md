# 统一返回与国际化

> 分类：后端 / 配置与审计  
> 版本：**预览版 V0.5.0**

这是把 `R<T>` 和后端国际化链路合并后的镜像入口页，适合作为接口开发时的第一阅读页。

## 一、为什么要一起看

在 Forgex 中，这两个主题天然联动：

- Controller / Service 最终通过 `R<T>` 对外返回结果
- `R` 内部承载 `messageCode` / `i18n` 元信息
- `RMessageI18nAdvice` 在响应阶段完成消息翻译
- 前端 `http.ts` 再根据 `message` 自动提示

也就是说：**统一返回模型是国际化消息链路的一部分**。

## 三、适用场景

- 新写 Controller 返回值规范
- 写操作成功消息不显示 / 显示错误
- 多语言消息无法翻译
- 网关或跨服务需要保留二次翻译信息

## 四、要点速览

| 主题 | 关键点 |
|---|---|
| `R<T>` | 统一 `code / message / data` 出参模型 |
| `I18nPrompt` | 把消息模板与业务代码解耦 |
| `LangContext` | 当前请求语言上下文 |
| `RMessageI18nAdvice` | 响应出参统一翻译 |
| 前端 `http.ts` | 消费后端 message 并自动提示 |
