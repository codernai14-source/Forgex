# 多租户

> 分类：后端 / 租户与上下文  
> 版本：**V0.5.0**  
> 更新时间：**2026-04-13**

## 一、概述

Forgex 采用 **共享数据库 + 行级租户隔离** 方案，每张业务表通过 `tenant_id` 字段区分租户数据。核心目标：

- 所有业务 SQL 自动拼接 `tenant_id` 条件，无需开发者手动过滤
- 公共配置使用 `tenant_id = 0` 作为全局默认，租户配置优先覆盖
- 忽略租户隔离必须显式声明，可审计、可收敛

## 二、核心组件

| 类 / 组件 | 包路径 | 作用 |
|---|---|---|
| `TenantContext` | `com.forgex.common.tenant` | 基于 ThreadLocal 存储当前请求租户 ID |
| `UserContext` | `com.forgex.common.tenant` | 基于 ThreadLocal 存储当前请求用户 ID |
| `TenantContextIgnore` | `com.forgex.common.tenant` | 临时忽略租户隔离的开关 |
| `TenantIgnoreRegistry` | `com.forgex.common.tenant` | 全局注册表，维护表级/服务级/方法级忽略规则 |
| `TenantIgnoreAspect` | `com.forgex.common.tenant` | AOP 切面，自动判断当前调用是否应忽略租户隔离 |
| `TenantMetaObjectHandler` | `com.forgex.common.tenant` | MyBatis-Plus 元对象处理器，自动填充 `tenant_id`、审计字段 |
| `TenantIgnoreLoader` | `com.forgex.sys.config` / `com.forgex.auth.config` | 启动时从数据库加载忽略规则 |

## 三、租户隔离原理

### 3.1 MyBatis-Plus 租户插件

项目使用 MyBatis-Plus 的 `TenantLineHandler` 实现自动 SQL 拼接：

```java
// 所有 SELECT/UPDATE/DELETE 自动追加：
// WHERE tenant_id = #{currentTenantId}
// INSERT 自动填充 tenant_id 字段
```

### 3.2 上下文传递链路

```
前端请求 → 网关 Gateway
  → 从 Token/Redis 解析 userId、tenantId、lang
  → 写入请求头 X-User-Id / X-Tenant-Id / X-Lang
  → 转发到微服务

微服务拦截器
  → 读取请求头
  → TenantContext.set(tenantId)
  → UserContext.set(userId)
  → LangContext.set(lang)
  → 业务处理
  → finally: 清理 ThreadLocal
```

### 3.3 自动填充

`TenantMetaObjectHandler` 实现 `MetaObjectHandler`，在 INSERT 时自动填充：

| 字段 | 来源 | 说明 |
|---|---|---|
| `tenant_id` | `TenantContext.get()` | 当前租户 |
| `create_by` | `UserContext.get()` | 创建人 |
| `update_by` | `UserContext.get()` | 更新人 |
| `create_time` | `LocalDateTime.now()` | 创建时间 |
| `update_time` | `LocalDateTime.now()` | 更新时间 |
| `deleted` | `false` | 逻辑删除标记 |

## 四、忽略租户隔离

### 4.1 三种忽略粒度

| 粒度 | 配置字段 (scope) | 匹配目标 (matcher) | 说明 |
|---|---|---|---|
| 表级 | `TABLE` | 表名如 `sys_user` | 该表不拼接 `tenant_id` 条件 |
| 服务级 | `SERVICE` | 全限定类名 | 该 Service 所有方法忽略租户 |
| 方法级 | `MAPPER` | `类名#方法名` | 指定 Mapper 方法忽略 |

### 4.2 默认忽略的表

系统启动时 `TenantIgnoreRegistry.reset()` 会默认注册以下表：

```
sys_user, sys_tenant, sys_user_tenant, sys_config,
sys_tenant_ignore, sys_tenant_menu_copy_rule,
fx_i18n_message, fx_i18n_language_type
```

### 4.3 数据库配置

忽略规则存储在 `sys_tenant_ignore` 表中：

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | bigint | 主键 |
| `scope` | varchar | 忽略粒度：TABLE / SERVICE / MAPPER |
| `matcher` | varchar | 匹配规则值 |
| `enabled` | boolean | 是否启用 |
| `remark` | varchar | 备注说明 |

### 4.4 运行时热更新

```java
// 调用接口重新加载忽略规则（无需重启）
// POST /auth/tenant-ignore/reload
// POST /sys/tenant-ignore/reload

// 内部执行逻辑：
TenantIgnoreRegistry.reset();
// 从数据库读取所有 enabled=true 的记录
// 按 scope 分别调用 addIgnoreTable / addIgnoreService / addIgnoreMapperMethod
```

### 4.5 代码级临时忽略

```java
// 方式一：使用 TenantContextIgnore（推荐在 Service 层使用）
TenantContextIgnore.setIgnore(true);
try {
    // 此处 SQL 不拼接 tenant_id
    return mapper.selectList(wrapper);
} finally {
    TenantContextIgnore.clear();
}

// 方式二：封装为工具方法
private <T> T executeWithoutTenantIsolation(Supplier<T> supplier) {
    boolean oldIgnore = TenantContextIgnore.isIgnore();
    TenantContextIgnore.setIgnore(true);
    try {
        return supplier.get();
    } finally {
        if (oldIgnore) {
            TenantContextIgnore.setIgnore(true);
        } else {
            TenantContextIgnore.clear();
        }
    }
}
```

## 五、公共配置与回退机制

```
查询顺序：
1. 先查当前 tenant_id 的配置
2. 若不存在，再查 tenant_id = 0 的公共配置
3. 若都不存在，使用代码默认值
```

适用场景：表格配置、字典配置、消息模板配置等。

## 六、异步任务中的上下文透传

异步任务（如 `@Async`、线程池）需要手动传递上下文：

```java
Long tenantId = TenantContext.get();
Long userId = UserContext.get();
String lang = LangContext.get();

CompletableFuture.runAsync(() -> {
    TenantContext.set(tenantId);
    UserContext.set(userId);
    LangContext.set(lang);
    try {
        // 业务逻辑
    } finally {
        TenantContext.clear();
        UserContext.clear();
        LangContext.clear();
    }
});
```

## 七、最佳实践

1. **不要跳过租户隔离来"偷懒"**：所有数据查询默认走租户隔离
2. **新增忽略规则走数据库配置**，不要在代码中硬编码
3. **公共数据统一用 `tenant_id = 0`**，不要使用 null
4. **异步任务务必手动传递上下文**
5. **定期审计 `sys_tenant_ignore` 表**，确保忽略规则最小化
