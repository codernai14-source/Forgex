# 文件上传使用方式

> 分类：后端 / 模块专题  
> 版本：**V0.6.0**  
> 更新时间：**2026-04-17**

本文既说明后台代码如何使用文件上传能力，也说明前端页面如何接、如何配置、如何排查。

## 代码如何使用

### 1. 后端直接调用上传服务

最常见的方式是直接注入 `FileService`，调用 `upload`：

```java
String fileUrl = fileService.upload(file);
```

返回值是最终可访问 URL，不是相对路径。

### 2. 使用统一上传接口

前后端分离页面直接走系统接口：

```text
POST /sys/file/upload
Content-Type: multipart/form-data
字段名：file
```

接口同时支持可选参数 `moduleCode`、`moduleName`。上传完成后，后端会把文件记录写入 `sys_file_record`，其中会保存模块编码、模块名称、相对路径、访问 URL、存储类型等信息，前端直接使用返回的 URL 即可。

### 3. 自定义存储策略

不要在业务代码里手写 OSS/MinIO SDK 分支，统一通过配置驱动：

- 系统配置 `file.upload.settings`
- 或租户默认表 `sys_file_storage`

### 4. 读取本地公开文件

本地模式下可通过：

```text
GET /files/{relativePath}
```

适合图片预览、头像展示、公开静态附件访问。

### 5. 敏感文件接入加密

如果上传的是敏感附件，推荐在后端上传链路里组合 `FileEncryptService`：

1. 接收原始文件
2. 加密为临时文件或加密流
3. 再走 `FileService` 或底层存储服务上传
4. 下载/预览时按业务规则解密

### 页面接法

常规页面接法是：

1. 选择文件
2. 调上传 API
3. 拿到返回 URL
4. 把 URL 回填到表单字段
5. 页面使用该 URL 预览或展示

头像上传就是一个典型参考实现，适合复用到其他图片上传场景。

当前登录页背景图 / 背景视频、系统 Logo、用户头像、租户 Logo 都已经走统一上传接口，并在上传时写入对应模块信息，便于在 `sys_file_record` 中追踪。

系统基础配置 `system.basic` 中的 `systemLogo`、`loginBackgroundImage`、`loginBackgroundVideo` 字段在保存和读取时会统一规范化：如果值是 `/files/**` 或 `/api/files/**`，会转换为 `publicBaseUrl + /files/**` 这种完整网关 URL，避免登录页在前端域名下请求相对路径。

## 配置如何使用

### 1. LOCAL 网关访问推荐配置

建议配置：

```json
{
  "storageType": "LOCAL",
  "localUploadPath": "C:/forgex/data/uploads",
  "publicBaseUrl": "http://192.168.1.10:9000/api",
  "accessPrefix": "/files"
}
```

说明：

1. `localUploadPath` 优先从系统配置 `file.upload.settings` 读取。
2. `localUploadPath` 在配置页面通过服务器目录选择器选择，选择的是后端服务所在机器的目录，不是浏览器客户端电脑目录。
3. `publicBaseUrl` 在 LOCAL 模式下建议使用网关地址 `http://<服务器IP>:9000/api`。
4. `accessPrefix` 保持 `/files` 时，最终上传返回 URL 形如 `http://<服务器IP>:9000/api/files/xxx.png`。
5. 用户只填写 `192.168.x.x` 或 `http://192.168.x.x` 时，系统会补全为 `http://192.168.x.x:9000/api`，避免浏览器默认访问 80 端口导致无法预览。
6. 前端页面不要自行拼接文件地址，直接使用上传接口返回的 URL 或数据库中的 `access_url`。

配置页面还提供以下辅助能力：

| 接口 | 用途 |
|---|---|
| `GET /sys/config/file-upload/runtime-defaults` | 返回本机 IP 候选、推荐 `publicBaseUrl`、访问示例 |
| `GET /sys/config/file-upload/folders/roots` | 返回后端服务器可选根目录 |
| `POST /sys/config/file-upload/folders/children` | 懒加载服务器子目录 |
| `POST /sys/config/file-upload/folders/create` | 在服务器当前目录下新建文件夹 |

### 2. MinIO 环境

`providerConfigJson` 推荐形如：

```json
{
  "endpoint": "http://127.0.0.1:9000",
  "accessKey": "minioadmin",
  "secretKey": "minioadmin",
  "bucketName": "forgex",
  "domain": "http://127.0.0.1:9000"
}
```

### 3. OSS 环境

`providerConfigJson` 推荐形如：

```json
{
  "endpoint": "oss-cn-hangzhou.aliyuncs.com",
  "accessKeyId": "your-ak",
  "accessKeySecret": "your-sk",
  "bucketName": "forgex-files",
  "domain": "https://cdn.example.com"
}
```

1. 后端先加密文件
2. 再走上传
3. 只对有权限的下载接口开放解密

## 常见问题排查

| 现象 | 排查方式 |
|---|---|
| 上传接口 200，但页面显示不出来 | 检查返回 URL 是否可直接访问 |
| 本地能看，生产看不到 | 检查 OSS/MinIO 域名、桶权限、跨域配置 |
| 返回 `http://192.168.x.x/files/xxx.png` 无法预览 | URL 缺少端口，浏览器会访问默认 80 端口；LOCAL 推荐配置为 `publicBaseUrl=http://<服务器IP>:9000/api`、`accessPrefix=/files` |
| 访问 `/api/files/**` 404 | 检查网关是否存在 `/api/files/** -> forgex-sys` 路由，并确认文件确实在 `localUploadPath` 下 |
| 访问 `/files/**` 404 | 这是直连系统服务路径；如果从浏览器走网关，请使用 `/api/files/**` |
| 同一个页面上传两次路径不一致 | 这是正常现象，文件名由 UUID 生成 |
| OSS/MinIO 模式仍尝试本地预览 | 检查前端是否错误改写了后端返回 URL |
