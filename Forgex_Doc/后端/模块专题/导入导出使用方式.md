# 导入导出使用方式

> 分类：后端 / 模块专题
> 版本：**V0.6.0**
> 更新时间：**2026-04-17**

本文档说明 Excel 导入导出能力在项目中的实际使用方式，重点覆盖：

1. 后端代码如何接入导出和模板下载。
2. 页面如何维护导入导出配置。
3. 新业务如何定义自己的 `tableCode`。
4. 模板下拉、字段映射、导入校验问题如何排查。

## 1. 后端代码如何使用

### 1.1 先定义稳定的 `tableCode`

新业务要接 Excel 导入导出时，第一步不是写 Controller，而是先定义稳定的 `tableCode`。

要求：

- 唯一
- 可读
- 不要频繁改名

推荐示例：

- `sys_user_table`
- `login_log_table`
- `basic_material_table`

后续这些能力都会依赖它：

- 导出配置
- 导入配置
- 模板下载
- 业务导出

### 1.2 后端接导出

典型接法是：

1. 前端传 `tableCode + 查询条件`
2. 服务层按 `tableCode` 取导出配置
3. 查询业务数据
4. 调 `excelFileService.buildExportFile(...)`
5. 输出到响应流

参考现有服务：

- `ExcelExportServiceImpl.exportLoginLog(...)`
- `ExcelExportServiceImpl.exportUser(...)`

示例思路：

```java
String tableCode = body.getTableCode();
FxExcelExportConfigDTO cfg = excelConfigService.getExportConfigByCode(tableCode);
List<DemoExportRow> rows = demoService.queryExportRows(body.getQuery());
byte[] bytes = excelFileService.buildExportFile(cfg, rows);
```

接入要点：

- 数据 DTO 字段名要和导出配置里的字段映射一致
- 文件下载统一走已有响应头处理逻辑
- 不要在业务里再手写一套列头

### 1.3 后端接模板下载

模板下载的典型流程是：

1. 前端传 `tableCode`
2. 服务端按 `tableCode` 取导入配置
3. 调 `buildImportTemplateXlsx(...)`
4. 返回下载流

示例思路：

```java
FxExcelImportConfigDTO cfg = excelConfigService.getImportConfigByCode(tableCode);
byte[] bytes = excelFileService.buildImportTemplateXlsx(cfg);
```

适用场景：

- 业务导入前先下载标准模板
- 页面让用户按统一格式填数据

### 1.4 后端接公共导入执行

公共导入升级后，业务页面不再各自上传 Excel 到业务接口。推荐流程是：

1. 前端公共导入组件按 `tableCode` 拉取导入配置。
2. 前端用 `xlsx` 在本地解析文件、做表头和必填基础校验。
3. 前端调用统一接口 `/sys/excel/import/execute`。
4. 后端按 `tableCode` 读取导入配置，并校验配置上的 `importPermission`。
5. 后端根据配置上的 `handlerBeanName` 找到业务处理器 Bean。
6. 后端把导入模式、导入配置、导入数据一起传给业务处理器。

统一执行参数是：

```java
FxExcelImportExecuteParam param = new FxExcelImportExecuteParam();
param.setTableCode(tableCode);
param.setImportMode(importMode);
param.setImportConfig(importConfig);
param.setImportData(importData);
```

`importMode` 固定使用：

- `ADD`：新增，已存在的数据跳过。
- `UPDATE`：修改，只更新已存在的数据。
- `COVER`：覆盖，由业务处理器按业务边界先清空或覆盖目标数据，再导入全部数据。

### 1.5 业务处理器如何实现

业务处理器需要实现 `FxExcelImportHandler`，并用配置中的 `handlerBeanName` 注册为 Spring Bean。

示例：

```java
@Component("demoImportHandler")
public class DemoImportHandler implements FxExcelImportHandler {

    @Override
    public FxExcelImportResultDTO handle(FxExcelImportExecuteParam param) {
        // 根据 param.getImportMode() 区分新增、修改、覆盖
        // 根据 param.getImportData() 读取导入数据并完成业务校验与落库
        return FxExcelImportResultDTO.success(total, created, updated, skipped, failed);
    }
}
```

处理器职责包括：

- 定义本业务的唯一键判断方式。
- 做业务合法性校验、关联数据校验和租户边界控制。
- 按 `ADD / UPDATE / COVER` 执行不同落库策略。
- 返回 `created / updated / skipped / failed` 等统一统计。

### 1.6 新增 Provider

如果模板下拉项不适合用字典或静态 JSON，可以新增 Provider。

接入步骤：

1. 实现 `TemplateOptionProvider`
2. 给 Provider 分配唯一编码
3. 注册到 `TemplateOptionProviderRegistry`
4. 在导入配置里把数据源类型设为 `PROVIDER`
5. 数据源值写 Provider 编码或其扩展参数

推荐场景：

- 下拉项依赖当前组织、工厂、角色
- 下拉项需要根据上下文动态变化

## 2. 页面如何使用

### 2.1 导出配置页面

页面位置：

- `D:\mine_product\forgex\Forgex_MOM\Forgex_Fronted\src\views\system\excelExportConfig\index.vue`

页面用途：

1. 维护某个 `tableCode` 的导出配置
2. 定义导出列、标题、字段映射、顺序、样式

页面使用顺序：

1. 先新建或定位目标 `tableCode`
2. 配置导出列
3. 保存配置
4. 再到业务页面触发真实导出验证效果

### 2.2 导入配置页面

页面位置：

- `D:\mine_product\forgex\Forgex_MOM\Forgex_Fronted\src\views\system\excelImportConfig\index.vue`

相关组件：

- `ExcelImportConfigModal.vue`
- `DataSourceConfig.vue`
- `FieldConfigList.vue`

页面用途：

1. 维护导入模板列
2. 配置字段类型、是否必填、数据源类型、下拉来源
3. 配置处理器 Bean 名称和导入权限码
4. 支持 `DICT / JSON / PROVIDER`
5. 支持通过 `sheetCode / sheetName` 配置多 Sheet 模板

页面使用顺序：

1. 选择或创建目标 `tableCode`
2. 配置 `handlerBeanName`，例如 `sysUserImportHandler`
3. 配置 `importPermission`，例如 `sys:user:import`
4. 配好列头、字段映射、必填规则
5. 多 Sheet 导入时，为每列配置对应的 `sheetCode` 和 `sheetName`
6. 如果有下拉项，再补数据源配置
7. 保存后下载模板验证

### 2.3 页面怎么触发公共导入

前端公共导入组件位置：

- `D:\mine_product\forgex\Forgex_MOM\Forgex_Fronted\src\components\excel\CommonImportDialog.vue`

业务页面只需要传入稳定的 `tableCode` 和弹窗状态，不再维护自己的上传解析逻辑。

示例：

```vue
<CommonImportDialog
  v-model:open="importDialogOpen"
  table-code="sys_user"
  @success="loadData"
/>
```

公共导入组件负责：

- 下载当前 `tableCode` 的模板。
- 解析 `.xlsx / .xls / .csv` 文件。
- 校验 10MB 文件大小、文件类型、表头顺序、必填字段和空行。
- 展示前 5 行预览、校验结果和查看全部数据弹窗。
- 调用 `/sys/excel/import/execute` 提交结构化数据。

### 2.4 页面怎么触发业务导出

前端接口封装位置：

- `D:\mine_product\forgex\Forgex_MOM\Forgex_Fronted\src\api\system\excel.ts`

页面使用建议：

1. 业务页面持有自己的 `tableCode`
2. 导出按钮点击时携带 `tableCode + 查询条件`
3. 后端按配置导出

重点不是页面自己拼列，而是页面只负责告诉后端“我现在要导哪个 `tableCode` 的数据”。

## 3. 新功能接入步骤

一个新业务要接入导入导出，推荐顺序如下：

1. 定义稳定 `tableCode`
2. 后台配置导出项
3. 后台配置导入主表的 `handlerBeanName` 和 `importPermission`
4. 后台配置导入项，单 Sheet 使用 `main`，多 Sheet 使用稳定的 `sheetCode`
5. 业务服务补导出查询逻辑
6. 新增业务导入处理器，实现 `FxExcelImportHandler`
7. 页面接导出按钮和公共导入弹窗
8. 用真实文件联调新增、修改、覆盖三种模式

## 4. 常见问题排查

### 4.1 业务页面点导出没反应

优先检查：

1. 页面是否真的传了 `tableCode`
2. 该 `tableCode` 是否存在导出配置
3. 后端是否正确设置了下载响应头
4. 当前用户是否有导出权限

### 4.2 导出的列不对

优先检查：

1. 后台导出配置是否改过
2. 列顺序和字段映射是否正确
3. 业务 DTO 字段名是否变更但没同步配置

### 4.3 下载模板没有下拉框

优先检查：

1. 导入配置的数据源类型是否正确
2. `DICT / JSON / PROVIDER` 的配置值是否正确
3. Provider 是否注册成功

### 4.4 导入文件本地校验失败

优先检查：

1. 模板是否是系统最新下载的
2. Excel 列顺序和列头是否被用户手改
3. 多 Sheet 文件的 Sheet 名称是否和配置匹配
4. 必填字段是否为空
5. CSV 是否错误用于多 Sheet 导入

### 4.5 导入能提交但后端执行失败

优先检查：

1. 导入配置是否填写了 `handlerBeanName`
2. 对应 Spring Bean 是否存在并实现了 `FxExcelImportHandler`
3. 导入配置是否填写了 `importPermission`
4. 当前用户是否拥有该导入权限
5. 业务唯一性校验、外键校验、字典校验是否失败
6. 当前租户上下文是否正确

### 4.6 改了 `tableCode` 后配置全丢

优先检查：

1. 页面、接口、后台配置是否仍然使用同一个 `tableCode`
2. 是否只改了前端代码，没迁移配置数据

## 5. 推荐实践

- 一个功能一个稳定 `tableCode`，不要多个页面混用又各自魔改。
- 模板列、导出列优先后台配置化，不要在代码里重复维护。
- 前端公共组件只做基础文件校验，业务唯一性和落库逻辑放到处理器中。
- 处理器 Bean 名称和权限码必须跟导入配置保持一致。
- Provider 只解决下拉项来源问题，不负责业务数据合法性。
