# 任务调度

> 分类：后端 / 模块专题  
> 版本：**V0.5.0**  
> 更新时间：**2026-04-15**

本专题用于说明 `Forgex_Job` 模块在当前仓库中的职责边界、接入方式和现阶段状态。

## 一、模块定位

`Forgex_Job` 是 Forgex 的任务调度与异步任务接入模块，当前主要承担：

- 定时任务服务启动入口
- Spring `@EnableScheduling` 调度能力
- Spring `@EnableAsync` 异步执行能力
- 公共 Feign 客户端复用
- 后续统一任务编排、重试、消息推送、数据清理等扩展位

当前服务名：`forgex-job`  
默认端口：`9004`

## 二、当前实现状态

### 2.1 已落地能力

- 启动类：`ForgexJobApplication`
- 已启用 `@EnableScheduling`
- 已启用 `@EnableAsync`
- 已启用注册发现与 Feign 客户端

### 2.2 当前边界

- 当前模块尚未提供独立业务 Controller
- 更适合作为“任务调度接入骨架”而不是“已完成的任务管理中心”
- 文档应明确该模块已纳入正式架构，但业务任务会按后续需要逐步沉淀

## 三、典型使用场景

- 数据清理与归档
- 审批超时重试
- 定时消息推送
- 定时统计汇总
- 外部系统数据同步

## 四、与其他模块的关系

| 模块 | 关系 |
|---|---|
| `Forgex_Workflow` | 可承接审批超时补偿、重试、提醒类任务 |
| `Forgex_Sys` | 可承接消息、授权、系统级定时任务 |
| `Forgex_Common` | 复用公共上下文、国际化、Redis、Feign 能力 |

## 五、文档约定

- 当 `Forgex_Job` 增加任务配置、任务日志、任务管理页面后，应优先在本页补充正式入口。
- 若任务能力直接挂到工作流或系统管理中，也要在本页增加“实际功能入口”说明，避免模块名存在但无导航。
