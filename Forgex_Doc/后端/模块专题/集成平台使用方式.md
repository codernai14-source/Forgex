# 集成平台使用方式

> 分类：后端 / 模块专题  
> 版本：**V0.7.0**  
> 更新时间：**2026-04-20**

本文说明当前接口平台在代码中、外部联调中和配置层面的标准使用方式。当前版本以统一执行引擎为中心，不再保留旧路由兼容层。

## 1. 使用前准备

推荐按以下顺序完成平台配置：

1. 新增第三方系统。
2. 新增接口配置，明确接口方向、调用模式、目标地址或处理器 Bean。
3. 配置参数结构。
4. 配置参数映射。
5. 配置第三方授权。
6. 如为入站接口，补充解释器 Bean。
7. 完成联调并通过调用日志、任务结果表观察执行结果。

数据库需要先执行升级脚本：

- `D:\mine_product\forgex\doc\sql\20260420_upgrade_integration_engine.sql`

运行参数位于：

- `D:\mine_product\forgex\Forgex_MOM\Forgex_Backend\Forgex_Integration\src\main\resources\application.yml`

重点配置包括：

- 缓存过期时间
- Redis 队列键
- 异步结果保留天数
- 同步线程池
- 异步消费线程池
- 日志缓冲线程池

## 2. 内部服务如何调用外部接口

### 2.1 标准入口

内部服务统一注入 `IntegrationFacade`，只传接口编码和请求实体：

```java
@Resource
private IntegrationFacade integrationFacade;

public void demo() {
    DemoRequest request = new DemoRequest();
    request.setOrderNo("SO20260420001");
    request.setAmount(new BigDecimal("128.50"));

    IntegrationExecuteResult result = integrationFacade.invoke("ERP_ORDER_PUSH", request);
}
```

标准接口位置：

- `D:\mine_product\forgex\Forgex_MOM\Forgex_Backend\Forgex_Integration\src\main\java\com\forgex\integration\service\IntegrationFacade.java`

### 2.2 平台内部会做什么

调用 `invoke` 后，平台会自动完成以下动作：

1. 按 `apiCode + tenantId + OUTBOUND` 查接口定义快照。
2. 将请求实体转为 `Map`。
3. 按参数映射规则组装 Body、Query、Header、Path 参数。
4. 调用外部 HTTP 接口。
5. 记录原始参数、组装后参数、返回值、结果类型、响应耗时。

### 2.3 返回值说明

出站调用默认是同步返回，常见字段包括：

- `accepted`：平台是否受理成功。
- `success`：执行是否成功。
- `traceId`：链路号。
- `status`：平台状态，常见如 `SUCCESS`。
- `resultType`：结果类型，常见如 `SUCCESS`、`SYSTEM_FAIL`、`TIMEOUT`。
- `data`：外部接口响应体。

## 3. 外部系统如何调用平台

### 3.1 统一调用入口

对外入口固定为：

- `POST /api/integration/public/invoke`

请求体必须包含 `jiekou_code`，其余字段都视为业务参数。例如：

```json
{
  "jiekou_code": "WMS_STOCK_IN",
  "orderNo": "RK20260420001",
  "warehouseCode": "WH001",
  "items": [
    {
      "skuCode": "SKU001",
      "qty": 10
    }
  ]
}
```

控制器位置：

- `D:\mine_product\forgex\Forgex_MOM\Forgex_Backend\Forgex_Integration\src\main\java\com\forgex\integration\controller\PublicIntegrationController.java`

### 3.2 同步与异步规则

同步或异步不是调用方传入的，而是由接口配置里的 `invokeMode` 固定决定：

- `SYNC`：直接进入解释器 Bean，执行完成后将结果返回给外部系统。
- `ASYNC`：先写任务表 `fx_api_task`，再推送 Redis 队列，立即返回受理结果。

异步返回通常包含：

- `accepted`
- `success`
- `taskId`
- `traceId`
- `status`
- `resultType`

### 3.3 任务结果查询

异步任务统一通过下面的接口查询结果：

- `GET /api/integration/public/task/{taskId}`

返回字段包括：

- `status`
- `resultType`
- `resultData`
- `errorMessage`
- `costTimeMs`
- `finishedTime`

## 4. 入站解释器如何接入

所有入站业务处理统一实现 `ApiInboundInterpreter`：

```java
@Component("demoInboundInterpreter")
public class DemoInboundInterpreter implements ApiInboundInterpreter {

    @Override
    public Object handle(ApiExecutionContext context, Object payload) {
        // 这里执行业务逻辑
        return Map.of("success", true);
    }
}
```

解释器标准接口位置：

- `D:\mine_product\forgex\Forgex_MOM\Forgex_Backend\Forgex_Integration\src\main\java\com\forgex\integration\spi\ApiInboundInterpreter.java`

使用约束：

- `processorBean` 必须配置为对应 Spring Bean 名称。
- 入站参数会先经过参数组装，再进入解释器。
- 同步执行和异步执行最终都调用同一个解释器接口。

## 5. 参数组装规则

当前版本统一支持以下确定性规则：

- 路径拷贝
- 默认值
- 常量填充
- 基础类型转换
- 字典翻译
- 日期格式化
- 大小写处理
- trim

首版不支持任意脚本表达式，避免性能和安全风险。

## 6. 高并发下需要关注的配置

### 6.1 接口级配置

在接口配置中重点关注以下字段：

- `invokeMode`
- `httpMethod`
- `contentType`
- `timeoutMs`
- `retryCount`
- `retryIntervalMs`
- `maxConcurrent`
- `queueLimit`
- `authType`

### 6.2 运行时配置

在 `application.yml` 中重点关注：

- `forgex.integration.cache`
- `forgex.integration.queue`
- `forgex.integration.log-buffer`
- `forgex.integration.sync-executor`
- `forgex.integration.async-consumer-executor`
- `forgex.integration.log-executor`

## 7. 结果查看与排障

出现问题时，建议按以下顺序排查：

1. 先查接口配置是否启用且方向正确。
2. 再查参数结构和映射是否完整。
3. 如为入站接口，确认 `processorBean` 对应解释器是否存在。
4. 如为异步接口，查看 `fx_api_task_result` 是否已有最终结果。
5. 查看调用日志月表，确认原始参数、组装参数、结果类型和错误信息。

日志重点字段：

- `trace_id`
- `task_id`
- `api_code`
- `raw_request_data`
- `assembled_request_data`
- `result_type`
- `cost_time_ms`

## 8. 当前版本说明

- 旧的 `IntegrationController` 已移除。
- 旧的 `IApiRouterService` / `ApiRouterServiceImpl` 已移除。
- 当前正式入口为 `IntegrationFacade` 和 `PublicIntegrationController`。
- 首版对外协议只覆盖 HTTP，不包含 TCP 实际执行能力。
