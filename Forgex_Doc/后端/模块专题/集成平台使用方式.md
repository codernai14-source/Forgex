# 集成平台使用方式

> 分类：后端 / 模块专题
> 版本：**V0.6.0**
> 更新时间：**2026-04-17**

本文档面向后端开发、前端开发、联调人员和实施人员，说明接口平台模块在代码中如何使用、页面上如何操作，以及遇到典型问题时如何排查。

## 1. 功能使用顺序

集成平台模块推荐按下面顺序使用：

1. 先新增第三方系统
2. 再新增接口配置
3. 再配置参数结构
4. 再配置参数映射
5. 再配置第三方授权
6. 最后发起联调并查看调用记录

如果直接跳到后面的步骤，通常会因为基础配置缺失导致报错。

## 2. 后端代码如何使用

### 2.1 新增第三方系统

后端接口由 `ThirdSystemController` 提供，前端封装在：

- `D:\mine_product\forgex\Forgex_MOM\Forgex_Fronted\src\api\system\integration.ts`

典型接口包括：

- 分页：`/integration/third-system/page`
- 详情：`/integration/third-system/detail/{id}`
- 新增：`/integration/third-system/create`
- 编辑：`/integration/third-system/update`
- 删除：`/integration/third-system/delete/{id}`

后端接入要点：

1. 当前请求必须有租户上下文
2. 当前用户必须有 `integration:third-system:add`
3. DTO 必填字段要完整
4. 表结构要支持租户和审计字段

### 2.2 新增接口配置

接口配置相关接口由 `ApiConfigController` 提供，前端封装同样在 `integration.ts` 中。

常用接口：

- 分页：`/integration/api-config/page`
- 详情：`/integration/api-config/detail/{id}`
- 新增：`/integration/api-config/create`
- 编辑：`/integration/api-config/update`
- 启用：`/integration/api-config/enable/{id}`
- 停用：`/integration/api-config/disable/{id}`
- 删除：`/integration/api-config/delete/{id}`

后端使用建议：

1. 新增配置后立即补参数配置和映射配置
2. 启停接口要区分“配置存在”和“配置可用”两个状态
3. `processorBean`、`moduleCode`、`apiPath` 等字段要与路由或处理器逻辑保持一致

### 2.3 配置参数与映射

参数配置与映射分别由：

- `ApiParamConfigController`
- `ApiParamMappingController`

提供。

开发时建议把它理解为两个层次：

1. 参数配置
   定义接口本身有哪些字段、层级、类型、方向。
2. 参数映射
   定义平台字段与第三方字段之间如何转换。

如果 API 配置已经建好，但调用仍然失败，经常不是 API 主记录的问题，而是参数配置或映射没配完整。

### 2.4 配置授权

第三方授权相关功能由 `ThirdAuthorizationController` 提供，主要权限码是：

- `integration:third-system:auth`

常见场景：

- token 模式
- IP 白名单模式

授权配好之前，即使第三方系统和 API 配置都存在，也不代表可以真正接收或发起请求。

### 2.5 查看调用记录

调用记录接口由 `ApiCallLogController` 提供，前端封装为：

- `getApiCallLogList`
- `getApiCallLogDetail`

调用记录是定位联调问题的第一入口，通常可以查到：

- 调用方向
- 调用状态
- 调用时间
- 请求参数
- 响应参数
- 错误信息
- 调用耗时

## 3. 页面如何使用

### 3.1 第三方系统页面

页面位置：

- `D:\mine_product\forgex\Forgex_MOM\Forgex_Fronted\src\views\integrationPlatform\thirdSystem\index.vue`

页面使用方式：

1. 进入列表页查看当前租户已配置的第三方系统
2. 点击“新增”打开表单弹窗
3. 填写系统编码、系统名称、IP、联系人、备注、状态
4. 保存后可继续打开授权弹窗配置鉴权

如果保存时报错，优先看浏览器请求参数和后端异常堆栈。

### 3.2 接口配置页面

页面位置：

- `D:\mine_product\forgex\Forgex_MOM\Forgex_Fronted\src\views\integrationPlatform\apiConfig\index.vue`

页面使用方式：

1. 进入接口配置页查看已配置的 API
2. 点击“新增”打开接口配置弹窗
3. 填写接口编码、名称、方向、路径、调用方式、目标地址、处理器、模块编码、超时时间、状态
4. 保存后可继续点击“参数配置”进入参数树配置
5. 根据需要继续进入参数映射配置

页面上新增按钮受以下权限控制：

- `integration:api-config:add`

因此如果页面看不到新增按钮，先查权限，不要先查页面组件。

### 3.3 调用记录页面

页面位置：

- `D:\mine_product\forgex\Forgex_MOM\Forgex_Fronted\src\views\integrationPlatform\apiCallLog\index.vue`

页面使用方式：

1. 输入接口 ID、调用方向、状态、IP、时间范围等条件
2. 点击查询
3. 查看表格中的状态、耗时、错误信息
4. 点击详情查看请求体和响应体

调用记录页面最关键的筛选条件是时间范围，因为后端会按时间范围决定查询哪些月表。

## 4. 联调顺序建议

### 4.1 新系统接入

推荐顺序：

1. 新增第三方系统
2. 给第三方系统配置授权
3. 新增接口配置
4. 配参数结构
5. 配参数映射
6. 执行实际调用
7. 去调用记录核对结果

### 4.2 页面与后端联调

建议按以下顺序拆分问题：

1. 页面按钮是否显示
2. 新增保存请求是否发出
3. 后端接口是否进入 Controller
4. Service 是否保存成功
5. 数据是否落库
6. 调用日志是否落表

这样可以避免把“权限问题、入参问题、数据库问题、分表问题”混在一起。

## 5. 典型问题排查

### 5.1 第三方系统配置新增报错

建议按下面顺序排查：

1. 看前端提交参数是否缺少必填项
2. 看当前用户是否有 `integration:third-system:add`
3. 看请求是否已经建立租户上下文
4. 看 `ThirdSystemServiceImpl` 保存前是否有字段转换问题
5. 看数据库唯一索引或字段长度是否冲突

尤其是租户字段问题，经常会表现成“同样的页面别人能新增，我这边报错”。

### 5.2 接口配置页面看不到新增按钮

建议按下面顺序排查：

1. 当前用户是否有 `integration:api-config:add`
2. 当前角色是否配置了该按钮权限
3. 当前租户下是否正确加载了角色权限
4. 前端 `v-permission="'integration:api-config:add'"` 是否正常工作

当前页面代码里已经写了按钮，不显示通常不是因为页面漏写。

### 5.3 调用记录报错

建议按下面顺序排查：

1. 查询时是否传了合法的时间范围
2. 对应月份的 `fx_api_call_log_yyyyMM` 表是否存在
3. 月表结构是否完整一致
4. 详情查询时 `id + callTime` 是否匹配
5. 跨月查询时是否覆盖到了所有目标月份

如果是“只有某个月的数据查不了”，优先怀疑该月表结构或建表逻辑。

### 5.4 首页菜单名显示的是代码

建议按下面顺序排查：

1. 菜单配置里保存的是名称还是国际化 key
2. 前端国际化资源是否有对应翻译
3. 菜单渲染时是否直接把 code 字段展示出来
4. 当前语言包是否缺失该菜单文本

这类问题一般不是接口平台逻辑错误，而是菜单配置或国际化资源缺失。

## 6. 推荐使用实践

- 第三方系统、接口配置、授权配置尽量由同一套租户内管理员维护，避免跨租户串配。
- 接口配置保存成功后，不要忘了继续配参数和授权，否则无法真正落地。
- 调用记录问题优先查时间范围和月表，不要只看 Controller 返回。
- 页面按钮不显示时，优先查权限码链路，不要先改页面代码。
