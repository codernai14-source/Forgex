# 集成平台使用方式

> 分类：后端 / 模块专题  
> 版本：**V0.8.0**  
> 更新时间：**2026-04-22**

本文说明当前接口平台在代码中、外部联调中和配置层面的标准使用方式。当前版本以统一执行引擎为中心，支持多目标出站调用。

## 1. 使用前准备

推荐按以下顺序完成平台配置：

1. 新增第三方系统（`ThirdSystem`）。
2. 配置第三方授权（`ThirdAuthorization`），设置 Token 或 IP 白名单。
3. 新增接口配置（`ApiConfig`），明确接口方向、调用模式、目标地址或处理器 Bean。
4. 如为出站接口，配置出站目标（`ApiOutboundTarget`），可配置多个目标系统。
5. 配置参数结构（`ApiParamConfig`），定义请求/响应参数树。
6. 配置参数映射（`ApiParamMapping`），定义字段转换规则。
7. 如为入站接口，补充解释器 Bean（`ApiInboundInterpreter`）。
8. 完成联调并通过调用日志、任务结果表观察执行结果。

运行参数位于：

- `Forgex_Integration/src/main/resources/application.yml`

重点配置包括：

- 缓存过期时间（`forgex.integration.cache`）
- Redis 队列键（`forgex.integration.queue`）
- 异步结果保留天数（`forgex.integration.async-result-retention-days`）
- 同步线程池（`forgex.integration.sync-executor`）
- 异步消费线程池（`forgex.integration.async-consumer-executor`）
- 日志缓冲线程池（`forgex.integration.log-executor`）

## 2. 内部服务如何调用外部接口

### 2.1 标准入口

内部服务统一注入 `IntegrationFacade`，只传接口编码和请求实体：

```java
@Resource
private IntegrationFacade integrationFacade;

public void demo() {
    DemoRequest request = new DemoRequest();
    request.setOrderNo("SO20260422001");
    request.setAmount(new BigDecimal("128.50"));

    IntegrationExecuteResult result = integrationFacade.invoke("ERP_ORDER_PUSH", request);
}
```

标准接口位置：

- `Forgex_Integration/src/main/java/com/forgex/integration/service/IntegrationFacade.java`
- `Forgex_Integration/src/main/java/com/forgex/integration/service/impl/IntegrationFacadeImpl.java`

### 2.2 平台内部会做什么

调用 `invoke` 后，平台会自动完成以下动作：

1. 按 `apiCode + tenantId + OUTBOUND` 查接口定义快照。
2. 将请求实体转为 `Map`。
3. 查询关联的出站目标配置（`ApiOutboundTarget`）。
4. 如有多个目标配置，按 `orderNum` 顺序处理：
   - 每个目标独立组装 Body、Query、Header、Path 参数。
   - 每个目标可配置独立的 `invokeMode`（同步/异步）。
   - 同步目标立即执行，异步目标入队等待。
5. 调用外部 HTTP 接口。
6. 记录原始参数、组装后参数、返回值、结果类型、响应耗时。

### 2.3 单目标与多目标模式

**单目标模式（无出站目标配置）**：

使用接口配置中的默认参数：
- `targetUrl` - 默认目标地址
- `httpMethod` - 默认 HTTP 方法
- `invokeMode` - 默认调用模式

**多目标模式（有出站目标配置）**：

每个目标独立配置：
- `targetUrl` - 目标专属地址
- `httpMethod` - 目标专属 HTTP 方法
- `invokeMode` - 目标专属调用模式
- `timeoutMs` - 目标专属超时时间
- `retryCount` - 目标专属重试次数
- `orderNum` - 执行顺序

### 2.4 返回值说明

出站调用返回 `IntegrationExecuteResult`：

| 字段 | 说明 |
|------|------|
| `accepted` | 平台是否受理成功 |
| `success` | 所有同步目标是否成功 |
| `traceId` | 链路号 |
| `status` | 平台状态：SUCCESS/PARTIAL_FAIL/WAITING |
| `resultType` | 结果类型：SUCCESS/SYSTEM_FAIL 等 |
| `data` | 单目标时返回响应体，多目标时返回 `syncResults` |
| `syncResults` | 同步执行结果列表 |
| `asyncResults` | 异步执行结果列表（包含 taskId） |

单目标同步调用示例：

```java
IntegrationExecuteResult result = integrationFacade.invoke("ERP_ORDER_PUSH", request);
if (result.isSuccess()) {
    Object data = result.getData(); // 外部接口响应体
}
```

多目标混合调用示例：

```java
IntegrationExecuteResult result = integrationFacade.invoke("MULTI_TARGET_API", request);
// 查看同步执行结果
for (IntegrationTargetResult syncResult : result.getSyncResults()) {
    System.out.println("目标系统: " + syncResult.getTargetSystemName());
    System.out.println("执行结果: " + syncResult.isSuccess());
}
// 查看异步执行任务
for (IntegrationTargetResult asyncResult : result.getAsyncResults()) {
    System.out.println("任务 ID: " + asyncResult.getTaskId());
}
```

## 3. 外部系统如何调用平台

### 3.1 统一调用入口

对外入口固定为：

- `POST /api/integration/public/invoke`

请求体必须包含 `jiekou_code`（接口编码），其余字段都视为业务参数。例如：

```json
{
  "jiekou_code": "WMS_STOCK_IN",
  "orderNo": "RK20260422001",
  "warehouseCode": "WH001",
  "items": [
    {
      "skuCode": "SKU001",
      "qty": 10
    }
  ]
}
```

控制器位置：

- `Forgex_Integration/src/main/java/com/forgex/integration/controller/PublicIntegrationController.java`

请求参数类：

- `Forgex_Integration/src/main/java/com/forgex/integration/domain/param/PublicInvokeRequest.java`

**请求参数解析逻辑**：

```java
@JsonProperty("jiekou_code")
private String interfaceCode;

private final Map<String, Object> payload = new LinkedHashMap<>();

@JsonAnySetter
public void put(String key, Object value) {
    if (!"jiekou_code".equals(key)) {
        payload.put(key, value);
    }
}
```

通过 Jackson 注解 `@JsonProperty("jiekou_code")` 将请求体的 `jiekou_code` 字段映射为 `interfaceCode`，其余所有字段通过 `@JsonAnySetter` 收集到 `payload` Map 中。

### 3.2 授权校验

入站调用时，平台会校验请求中的 `token` 字段：

```json
{
  "jiekou_code": "WMS_STOCK_IN",
  "token": "your-token-value",
  "orderNo": "RK20260422001"
}
```

授权校验由 `IThirdAuthorizationService.validateToken(token)` 完成。

校验逻辑（`ApiGatewayServiceImpl`）：

```java
String token = rawPayload == null ? null : String.valueOf(rawPayload.getOrDefault("token", ""));
if (token != null && !token.isBlank() && !thirdAuthorizationService.validateToken(token)) {
    return IntegrationExecuteResult.builder()
        .accepted(false)
        .success(false)
        .status("AUTH_FAIL")
        .resultType(ApiResultTypeEnum.AUTH_FAIL.name())
        .errorMessage("authorization failed")
        .build();
}
```

### 3.3 同步与异步规则

同步或异步由接口配置中的 `invokeMode` 决定：

- `SYNC`：直接进入解释器 Bean，执行完成后将结果返回给外部系统。
- `ASYNC`：先写任务表 `fx_api_task`，再推送 Redis 队列，立即返回受理结果。

异步返回通常包含：

```json
{
  "accepted": true,
  "success": true,
  "taskId": "abc123def456",
  "traceId": "trace001",
  "status": "WAITING",
  "resultType": "SUCCESS"
}
```

### 3.4 任务结果查询

异步任务统一通过以下接口查询结果：

- `GET /api/integration/public/task/{taskId}`

返回 `ApiTaskResultDTO`，包含：

| 字段 | 说明 |
|------|------|
| `taskId` | 任务唯一标识 |
| `traceId` | 链路追踪 ID |
| `apiCode` | 接口编码 |
| `direction` | 调用方向 |
| `status` | 任务状态：SUCCESS/FAIL |
| `resultType` | 结果类型 |
| `resultData` | 执行结果数据 |
| `errorMessage` | 错误信息 |
| `costTimeMs` | 执行耗时 |
| `finishedTime` | 完成时间 |
| `outboundTargetId` | 出站目标 ID |
| `targetSystemCode` | 目标系统编码 |
| `targetSystemName` | 目标系统名称 |

## 4. 入站解释器如何接入

所有入站业务处理统一实现 `ApiInboundInterpreter`：

```java
@Component("wmsStockInInterpreter")
public class WmsStockInInterpreter implements ApiInboundInterpreter {

    @Override
    public Object handle(ApiExecutionContext context, Object payload) {
        // payload 已经过参数组装
        Map<String, Object> data = (Map<String, Object>) payload;
        
        // 执行业务逻辑
        String orderNo = (String) data.get("orderNo");
        String warehouseCode = (String) data.get("warehouseCode");
        
        // 返回结果
        return Map.of("success", true, "message", "入库成功");
    }
}
```

解释器标准接口位置：

- `Forgex_Integration/src/main/java/com/forgex/integration/spi/ApiInboundInterpreter.java`

解释器注册表位置：

- `Forgex_Integration/src/main/java/com/forgex/integration/service/impl/ApiInterpreterRegistry.java`

使用约束：

- `processorBean` 必须配置为对应 Spring Bean 名称（如 `wmsStockInInterpreter`）。
- 入站参数会先经过参数组装，再进入解释器。
- 同步执行和异步执行最终都调用同一个解释器接口。

执行上下文（`ApiExecutionContext`）提供：

| 字段 | 说明 |
|------|------|
| `tenantId` | 租户 ID |
| `apiConfigId` | 接口配置 ID |
| `outboundTargetId` | 出站目标 ID |
| `apiCode` | 接口编码 |
| `targetSystemCode` | 目标系统编码 |
| `targetSystemName` | 目标系统名称 |
| `direction` | 方向 |
| `traceId` | 链路号 |
| `taskId` | 任务号（异步时） |
| `callerIp` | 调用方 IP |
| `invokeMode` | 调用模式 |
| `startTime` | 开始时间 |

## 5. 参数组装规则

当前版本统一支持以下确定性规则：

### 5.1 值来源类型（ApiMappingValueTypeEnum）

| 类型 | 说明 | 配置字段 |
|------|------|----------|
| `SOURCE` | 从源字段路径读取值 | `sourceFieldPath` |
| `DEFAULT` | 使用配置的默认值 | `defaultValue` |
| `CONSTANT` | 使用配置的常量值 | `constantValue` |

### 5.2 目标范围类型（ApiMappingTargetScopeEnum）

| 范围 | 说明 | 用途 |
|------|------|------|
| `BODY` | 请求体 | 出站调用时组装 JSON Body |
| `QUERY` | URL 参数 | 出站调用时组装 Query String |
| `HEADER` | 请求头 | 出站调用时设置 HTTP Header |
| `PATH` | 路径变量 | 出站调用时替换 URL 中的 `{var}` |

### 5.3 转换规则（transformRule）

| 规则 | 说明 | 示例 |
|------|------|------|
| `trim` | 去除前后空格 | `" ABC "` → `"ABC"` |
| `uppercase` | 转大写 | `"abc"` → `"ABC"` |
| `lowercase` | 转小写 | `"ABC"` → `"abc"` |
| `number` | 数字 | `"123"` → `123L` |
| `boolean` | 转布尔 | `"true"` → `true` |
| `date:pattern` | 日期格式化 | `date:yyyy-MM-dd` |

### 5.4 参数组装示例

出站参数映射配置：

```json
[
  {
    "sourceFieldPath": "order.orderNo",
    "targetFieldPath": "order_code",
    "targetScope": "BODY",
    "valueType": "SOURCE",
    "transformRule": "uppercase"
  },
  {
    "sourceFieldPath": "order.createTime",
    "targetFieldPath": "created_at",
    "targetScope": "BODY",
    "valueType": "SOURCE",
    "transformRule": "date:yyyy-MM-dd HH:mm:ss"
  },
  {
    "targetFieldPath": "sys_code",
    "targetScope": "BODY",
    "valueType": "CONSTANT",
    "constantValue": "FORGEX"
  },
  {
    "targetFieldPath": "version",
    "targetScope": "HEADER",
    "valueType": "DEFAULT",
    "defaultValue": "1.0"
  },
  {
    "sourceFieldPath": "order.id",
    "targetFieldPath": "orderId",
    "targetScope": "PATH",
    "valueType": "SOURCE"
  }
]
```

## 6. 高并发下需要关注的配置

### 6.1 接口级配置

在接口配置中重点关注以下字段：

| 字段 | 说明 | 建议值 |
|------|------|--------|
| `invokeMode` | 调用模式 | 高频接口建议 ASYNC |
| `httpMethod` | HTTP 方法 | 根据目标系统要求 |
| `contentType` | 内容类型 | 通常 `application/json` |
| `timeoutMs` | 超时时间 | 建议 5000-30000 |
| `retryCount` | 重试次数 | 建议 0-3 |
| `retryIntervalMs` | 重试间隔 | 建议 1000-5000 |
| `maxConcurrent` | 最大并发 | 根据目标系统承受能力 |
| `queueLimit` | 队列限制 | 根据业务峰值 |
| `authType` | 授权类型 | 根据安全要求 |

### 6.2 出站目标配置

每个出站目标可独立配置：

| 字段 | 说明 |
|------|------|
| `targetUrl` | 目标专属地址 |
| `httpMethod` | 目标专属 HTTP 方法 |
| `invokeMode` | 目标专属调用模式 |
| `timeoutMs` | 目标专属超时时间 |
| `retryCount` | 目标专属重试次数 |
| `orderNum` | 执行顺序（多目标时） |

### 6.3 运行时配置

在 `application.yml` 中重点关注：

```yaml
forgex:
  integration:
    cache:
      local-expire-seconds: 300    # 本地缓存 TTL
      redis-expire-seconds: 300    # Redis 缓存 TTL
      max-size: 2000               # 本地缓存容量
    queue:
      redis-key: integration:task:queue
      batch-size: 100
      lease-seconds: 60            # 任务租约时长
      compensate-limit: 200        # 补偿批次
    log-buffer:
      batch-size: 100              # 日志批量刷库条数
      flush-interval-ms: 1000      # 日志刷库间隔
    sync-executor:
      core-pool-size: 8
      max-pool-size: 32
      queue-capacity: 500
    async-consumer-executor:
      core-pool-size: 8
      max-pool-size: 32
      queue-capacity: 500
    async-result-retention-days: 7 # 异步结果保留天数
```

配置类位置：

- `Forgex_Integration/src/main/java/com/forgex/integration/config/IntegrationProperties.java`
- `Forgex_Integration/src/main/java/com/forgex/integration/config/IntegrationRuntimeConfig.java`

## 7. 第三方授权配置

### 7.1 Token 授权

为第三方系统生成 Token：

```java
// 调用 API 生成
POST /api/integration/third-authorization/generate-token/{thirdSystemId}?expireHours=24

// 返回 Token 值
"abc123def456ghi789"
```

外部调用时传入 Token：

```json
{
  "jiekou_code": "WMS_STOCK_IN",
  "token": "abc123def456ghi789",
  "data": { ... }
}
```

### 7.2 IP 白名单

配置 IP 白名单：

```json
{
  "thirdSystemId": 1,
  "authType": "WHITELIST",
  "whitelistIps": "192.168.1.100,192.168.1.101",
  "status": 1
}
```

校验 IP：

```java
GET /api/integration/third-authorization/check-ip-whitelist/{thirdSystemId}?ipAddress=192.168.1.100
```

## 8. 结果查看与排障

出现问题时，建议按以下顺序排查：

1. 先查接口配置是否启用且方向正确（`fx_api_config` 表）。
2. 如为出站接口，查出站目标配置是否正确（`fx_api_outbound_target` 表）。
3. 再查参数结构和映射是否完整（`fx_api_param_config`、`fx_api_param_mapping` 表）。
4. 如为入站接口，确认 `processorBean` 对应解释器是否存在。
5. 如为异步接口，查看 `fx_api_task_result` 是否已有最终结果。
6. 查看调用日志，确认原始参数、组装参数、结果类型和错误信息。

日志重点字段：

| 字段 | 说明 |
|------|------|
| `trace_id` | 链路号（用于追踪） |
| `task_id` | 任务号（异步时） |
| `api_code` | 接口编码 |
| `raw_request_data` | 原始请求参数 |
| `assembled_request_data` | 组装后参数 |
| `result_type` | 结果类型 |
| `cost_time_ms` | 执行耗时 |
| `outbound_target_id` | 出站目标 ID |
| `target_system_code` | 目标系统编码 |
| `target_system_name` | 目标系统名称 |
| `error_message` | 错误信息 |
