# 集成平台

> 分类：后端 / 模块专题  
> 版本：**V0.8.0**  
> 更新时间：**2026-04-22**

接口平台已升级为统一的双向执行引擎，面向两类核心场景：

1. 内部服务调用外部接口  
   统一通过 `IntegrationFacade.invoke(String apiCode, Object requestEntity)` 发起调用，平台自动查配置、组装参数、执行外呼、记录日志。支持多目标系统并发调用。
2. 外部系统调用内部业务  
   统一通过 `POST /api/integration/public/invoke` 进入平台，请求体必须包含 `jiekou_code`，平台按接口编码找到解释器 Bean，按配置同步执行或异步入任务。

## 核心能力

- 双向执行：同时支持 `OUTBOUND` 出站调用和 `INBOUND` 入站调用。
- 多目标出站：一个出站接口可配置多个目标系统（`ApiOutboundTarget`），支持同步/异步混合执行。
- 统一入口：内部统一 Bean 门面，对外统一 HTTP 入口。
- 高吞吐缓存：接口定义在运行期编译为快照，走本地 Caffeine 缓存 + Redis 缓存二级缓存。
- 异步任务：采用 `任务表 + Redis 队列 + 定时补偿` 模型，兼顾吞吐与可靠性。
- 全链路记录：保留调用日志，并扩展记录原始参数、组装参数、结果类型、耗时、任务号、链路号、目标系统信息。
- 结果查询：异步任务执行完成后写入结果表，外部系统通过 `taskId` 轮询结果。
- 第三方授权：支持 Token 授权和 IP 白名单两种方式。

## 当前标准入口

### 内部调用入口

- 接口：`IntegrationFacade`
- 方法：`IntegrationExecuteResult invoke(String apiCode, Object requestEntity)`
- 代码位置：
  - `Forgex_Integration/src/main/java/com/forgex/integration/service/IntegrationFacade.java`
  - `Forgex_Integration/src/main/java/com/forgex/integration/service/impl/IntegrationFacadeImpl.java`

### 对外统一入口

- `POST /api/integration/public/invoke` - 入站调用入口
- `GET /api/integration/public/task/{taskId}` - 异步任务结果查询
- 控制器位置：
  - `Forgex_Integration/src/main/java/com/forgex/integration/controller/PublicIntegrationController.java`

请求参数类：

- `Forgex_Integration/src/main/java/com/forgex/integration/domain/param/PublicInvokeRequest.java`

**请求参数说明**：

请求体必须包含 `jiekou_code` 字段（通过 Jackson 注解映射为 `interfaceCode`），其余字段均视为业务参数。

```java
@JsonProperty("jiekou_code")
private String interfaceCode;

private final Map<String, Object> payload = new LinkedHashMap<>();

@JsonAnySetter
public void put(String key, Object value) {
    if (!"jiekou_code".equals(key)) {
        payload.put(key, value);
    }
}
```

## 关键运行结构

### 配置快照

运行期不再每次调用都拼装多张配置表，而是将以下配置编译为 `ApiDefinitionSnapshot`：

- `ApiConfigDTO` - 接口基础配置
- `ApiOutboundTargetDTO` - 出站目标系统配置（支持多目标）
- `ApiParamConfigDTO` - 请求/响应参数树
- `ApiParamMappingDTO` - 入站/出站参数映射

快照模型字段：

```java
@Data
@Builder
public class ApiDefinitionSnapshot {
    private Long tenantId;
    private String cacheKey;
    private ApiConfigDTO apiConfig;
    private List<ApiParamConfigDTO> requestParamTree;
    private List<ApiParamConfigDTO> responseParamTree;
    private List<ApiParamMappingDTO> inboundMappings;
    private List<ApiParamMappingDTO> outboundMappings;
    private List<ApiOutboundTargetDTO> outboundTargets;
}
```

快照服务位置：

- `Forgex_Integration/src/main/java/com/forgex/integration/service/impl/ApiDefinitionServiceImpl.java`

缓存键格式：

```text
integration:definition:{tenantId}:{direction}:{apiCode}
```

### 入站解释器

入站接口统一使用解释器 SPI：

- 接口：`ApiInboundInterpreter`
- 标准签名：`Object handle(ApiExecutionContext context, Object payload)`
- 注册表：`ApiInterpreterRegistry`

SPI 位置：

- `Forgex_Integration/src/main/java/com/forgex/integration/spi/ApiInboundInterpreter.java`
- `Forgex_Integration/src/main/java/com/forgex/integration/service/impl/ApiInterpreterRegistry.java`

解释器注册表在启动时通过 Spring ApplicationContext 自动扫描所有 `ApiInboundInterpreter` 实现 Bean 并注册。

### 异步任务

异步入站接口先落库，再推 Redis 队列，消费成功后写结果表并删除任务表记录：

- 任务表：`fx_api_task`
- 结果表：`fx_api_task_result`
- Redis 队列键：`integration:task:queue`（可配置）

任务状态流转：

- `WAITING` → `QUEUED` → `RUNNING` → `SUCCESS` / `FAIL` / `RETRY`

### 调用日志

调用日志记录完整调用信息：

- 日志表：`fx_api_call_log`
- 重点字段：
  - `api_code` - 接口编码
  - `trace_id` - 链路号
  - `task_id` - 任务号
  - `invoke_mode` - 调用模式
  - `raw_request_data` - 原始请求参数
  - `assembled_request_data` - 组装后参数
  - `response_code` - 响应码
  - `result_type` - 结果类型
  - `cost_time_ms` - 耗时
  - `outbound_target_id` - 出站目标 ID
  - `target_system_code` - 目标系统编码
  - `target_system_name` - 目标系统名称

日志采用批量缓冲后刷库，降低高并发下单条插入压力。

| 值 | 说明 |
|----|------|
