# 集成平台

> 分类：后端 / 模块专题  
> 版本：**V0.7.0**  
> 更新时间：**2026-04-20**

接口平台已升级为统一的双向执行引擎，面向两类核心场景：

1. 内部服务调用外部接口  
   统一通过 `IntegrationFacade.invoke(String apiCode, Object requestEntity)` 发起调用，平台自动查配置、组装参数、执行外呼、记录日志。
2. 外部系统调用内部业务  
   统一通过 `POST /api/integration/public/invoke` 进入平台，请求体必须包含 `jiekou_code`，平台按接口编码找到解释器 Bean，按配置同步执行或异步入任务。

首版已经移除旧的兼容路由和旧执行入口，不再使用 `IntegrationController`、`IApiRouterService`、`ApiRouterServiceImpl`。

## 核心能力

- 双向执行：同时支持 `OUTBOUND` 出站调用和 `INBOUND` 入站调用。
- 统一入口：内部统一 Bean 门面，对外统一 HTTP 入口。
- 高吞吐缓存：接口定义在运行期编译为快照，走本地缓存 + Redis 缓存。
- 异步任务：采用 `任务表 + Redis 队列 + 定时补偿` 模型，兼顾吞吐与可靠性。
- 全链路记录：保留调用日志，并扩展记录原始参数、组装参数、结果类型、耗时、任务号、链路号。
- 结果查询：异步任务执行完成后写入结果表，外部系统通过 `taskId` 轮询结果。

## 关键运行结构

### 配置快照

运行期不再每次调用都拼装多张配置表，而是将以下配置编译为 `ApiDefinitionSnapshot`：

- `ApiConfig`
- `ApiParamConfig`
- `ApiParamMapping`
- 授权配置

快照服务位置：

- `D:\mine_product\forgex\Forgex_MOM\Forgex_Backend\Forgex_Integration\src\main\java\com\forgex\integration\service\impl\ApiDefinitionServiceImpl.java`

### 入站解释器

入站接口不再走反射调用，统一使用解释器 SPI：

- 接口：`ApiInboundInterpreter`
- 标准签名：`Object handle(ApiExecutionContext context, Object payload)`

SPI 位置：

- `D:\mine_product\forgex\Forgex_MOM\Forgex_Backend\Forgex_Integration\src\main\java\com\forgex\integration\spi\ApiInboundInterpreter.java`

### 异步任务

异步入站接口先落库，再推 Redis 队列，消费成功后写结果表并删除任务表记录：

- 任务表：`fx_api_task`
- 结果表：`fx_api_task_result`
- 升级脚本：
  - `D:\mine_product\forgex\doc\sql\20260420_upgrade_integration_engine.sql`

### 调用日志

调用日志仍按月分表落库，但新增以下关键信息：

- `api_code`
- `trace_id`
- `task_id`
- `invoke_mode`
- `raw_request_data`
- `assembled_request_data`
- `response_code`
- `result_type`

日志改为批量缓冲后刷库，降低高并发下单条插入压力。
