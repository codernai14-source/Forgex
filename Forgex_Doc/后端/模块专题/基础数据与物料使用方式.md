# 基础数据与物料使用方式

> 分类：后端 / 模块专题  
> 版本：**V0.6.0**  
> 更新时间：**2026-04-17**

本文档说明基础数据与物料模块的使用与阅读方式。

## 适用场景

- 开发物料管理
- 开发供应商主数据
- 接入自动编码
- 排查基础模块启用状态

## 推荐做法

1. 物料编码统一走编码规则。
2. 状态、类型优先走字典。
3. 批量导入导出优先复用公共能力。

## 菜单与权限

基础数据模块菜单显示名统一为“基础信息”，基础信息主页是一级菜单，不再通过二级占位页进入。

| 菜单 | 前端组件 | 主权限 |
|---|---|---|
| 基础信息主页 | `BasicDashboard` | `basic:dashboard:view` |
| 供应商管理 | `BasicSupplier` | `basic:supplier:query` |
| 客户管理 | `BasicCustomer` | `basic:customer:query` |
| 物料管理 | `BasicMaterial` | `basic:material:query` |
| 计量单位 | `BasicUnit` | `basic:unit:query` |
| 编码规则管理 | `BasicEncodeRule` | `basic:encodeRule:query` |

物料管理是单页面入口，页面内通过 `全部 / 原材料 / 半成品 / 成品` 四个 Tab 区分查询范围。`全部` Tab 不传 `materialType`，其他 Tab 通过同一个 `/basic/material/page` 接口传入对应物料类型。历史路由组件键 `BasicMaterialRaw`、`BasicMaterialSemiFinished`、`BasicMaterialFinished` 仅作为兼容映射保留，统一加载新的 `BasicMaterial` 页面。

物料管理共用物料按钮权限：`basic:material:add/edit/delete`。编码规则归属基础信息模块后，权限码使用 `basic:encodeRule:*`。客户管理使用公共表格配置 `CustomerMasterTable`，物料管理使用公共表格配置 `MaterialTable`。

已有环境可执行修复脚本同步菜单、权限和打开次数表：

```text
doc/sql/20260429_basic_homepage_module_homepage_guide.sql
doc/sql/20260429_fix_basic_info_material_customer_encode_rule.sql
```

## 供应商主数据

供应商主数据前端入口：

```text
Forgex_MOM/Forgex_Fronted/src/views/basic/supplier/index.vue
```

页面通过 `FxDynamicTable` 承载列表和列配置，新增 / 编辑 / 详情共用 `BaseFormDialog`。当前供应商新增弹窗在调用处显式启用 `:mask-closable="true"`，因此打开新增供应商后点击弹窗外部遮罩会关闭弹窗。

使用建议：

1. 新增供应商时可启用自动生成编码，保存时由后端编码规则生成供应商编码。
2. 合作状态、信用等级、风险等级、供应商分级、审查状态等下拉项优先走数据字典。
3. 联系人和资质作为供应商主数据的子表信息随主表一起维护。
4. 导入、导出、模板下载优先复用公共导入导出能力。
5. 供应商资质审查流程使用工作流自定义表单路径 `/basic/supplier/review`，前端注册组件为 `src/views/workflow/form/SupplierReviewForm.vue`；从通用发起审批页进入时，由该组件选择待审供应商并调用 `/basic/supplier/review/start`，确保供应商审查状态同步进入“审查中”。

## 物料管理

物料管理前端入口：

```text
Forgex_MOM/Forgex_Fronted/src/views/basic/material/index.vue
```

页面通过 `FxDynamicTable` 和 `MaterialTable` 承载列表列配置。四个 Tab 共用同一套新增、编辑、详情、删除逻辑，后端仍按 `materialType` 区分原材料、半成品和成品。

使用建议：

1. 物料编码统一走编码规则，新增时优先使用自动编码。
2. 分类、单位、状态和审批状态字段优先走字典或基础资料接口。
3. 表格列缺失时先检查公共表格配置 `MaterialTable` 是否存在，再检查返回字段与列配置字段是否一致。

## 计量单位

计量单位前端入口：

```text
Forgex_MOM/Forgex_Fronted/src/views/basic/unit/index.vue
```

页面左侧显示计量单位类型树，右侧使用 `FxDynamicTable` 和公共表格配置 `BasicUnitTable` 承载单位主数据。选中左侧类型后，右侧列表按 `unitTypeId` 查询。

计量单位初始化与升级脚本为 `Forgex_Doc/数据库/脚本与修复/20260502_basic_unit_and_table_upgrade.sql`，用于在 `forgex_admin` 同步三表结构、兼容旧版 `basic_unit` 单表并补齐 `unit_type_id` 与默认类型，在 `forgex_common` 初始化 `BasicUnitTable` 公共表格配置，并补齐 `BasicUnit` 菜单、`basic:unit:*` 按钮权限、管理员授权、页面文案和后端返回消息。

使用建议：

1. 先维护单位类型，再在类型下新增单位主数据。
2. 单位编码在同一类型下唯一，不同类型可使用相同编码。
3. 点击单位行的“转换配置”维护多条目标单位关系，例如 `1 m = 100 cm`、`1 m = 1000 mm`。
4. 删除类型前需要先删除子类型和单位；删除单位前需要先删除涉及该单位的换算关系。
5. 其它业务服务需要换算时，优先通过 `BasicUnitConversionFeignClient.convert` 调用公共换算能力，按 `ID/CODE` 明确指定实体内单位字段类型。

## 客户主数据

客户管理前端入口：

```text
Forgex_MOM/Forgex_Fronted/src/views/basic/customer/index.vue
```

客户主数据使用公共表格配置 `CustomerMasterTable`。已有环境如缺少 `customer_full_name`、价值等级、信用等级、地址、审批状态等字段，应执行 `doc/sql/20260429_fix_basic_info_material_customer_encode_rule.sql` 补齐表结构和公共列配置。
