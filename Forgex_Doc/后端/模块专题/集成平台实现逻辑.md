# 集成平台实现逻辑

> 分类：后端 / 模块专题  
> 版本：**V0.6.0**  
> 更新时间：**2026-04-20**

本文说明当前接口平台统一执行引擎的实际实现结构，重点覆盖入口、缓存、参数组装、同步执行、异步任务、日志缓冲和运行时线程池。

## 1. 总体架构

当前实现围绕 `ApiGatewayServiceImpl` 组织，分为两条主链路：

1. 出站链路  
   内部服务调用 `IntegrationFacade`，平台按接口编码查找 `OUTBOUND` 定义，组装外呼参数并执行 HTTP 请求。
2. 入站链路  
   外部系统调用 `PublicIntegrationController`，平台按 `jiekou_code` 查找 `INBOUND` 定义，参数组装后按配置同步执行或异步入任务。

核心代码位置：

- `D:\mine_product\forgex\Forgex_MOM\Forgex_Backend\Forgex_Integration\src\main\java\com\forgex\integration\service\impl\IntegrationFacadeImpl.java`
- `D:\mine_product\forgex\Forgex_MOM\Forgex_Backend\Forgex_Integration\src\main\java\com\forgex\integration\controller\PublicIntegrationController.java`
- `D:\mine_product\forgex\Forgex_MOM\Forgex_Backend\Forgex_Integration\src\main\java\com\forgex\integration\service\impl\ApiGatewayServiceImpl.java`

## 2. 统一入口层

### 2.1 内部统一门面

`IntegrationFacadeImpl` 只保留一个标准方法：

```java
IntegrationExecuteResult invoke(String apiCode, Object requestEntity)
```

该门面不关心业务方实体结构，只负责将调用交给 `IApiGatewayService.invokeOutbound(...)`。

### 2.2 对外统一 HTTP 入口

`PublicIntegrationController` 暴露两个外部接口：

- `POST /api/integration/public/invoke`
- `GET /api/integration/public/task/{taskId}`

控制器层只做轻量逻辑：

- 解析请求体
- 提取 `jiekou_code`
- 获取调用方 IP
- 返回统一响应

控制器不再承载旧版路由转发能力。

## 3. 配置快照与缓存

### 3.1 快照模型

运行期调用不会每次重新查询并拼装多张表，而是由 `ApiDefinitionServiceImpl` 将配置编译成 `ApiDefinitionSnapshot`。

快照内容包括：

- `ApiConfigDTO`
- 请求参数树
- 响应参数树
- 入站映射
- 出站映射

### 3.2 二级缓存

`ApiDefinitionServiceImpl` 使用两级缓存：

1. 本地 Caffeine 缓存  
   适合实例内高频读取，减少重复反序列化和 Redis 往返。
2. Redis 缓存  
   适合多实例共享定义快照。

缓存键格式：

```text
integration:definition:{tenantId}:{direction}:{apiCode}
```

这保证了不同租户、不同方向的配置不会互相污染。

实现位置：

- `D:\mine_product\forgex\Forgex_MOM\Forgex_Backend\Forgex_Integration\src\main\java\com\forgex\integration\service\impl\ApiDefinitionServiceImpl.java`

## 4. 参数组装引擎

`ApiGatewayServiceImpl` 在执行前统一调用 `IApiParamAssembler`，分别处理：

- 入站组装：`assembleInbound`
- 出站组装：`assembleOutbound`

首版执行规则限定为确定性转换：

- 字段路径拷贝
- 默认值填充
- 常量填充
- 基础类型转换
- 字典翻译
- 日期格式化
- 大小写处理
- trim

不支持任意脚本表达式，目的是保证性能稳定和执行安全。

## 5. 出站执行链路

### 5.1 执行流程

内部服务调用 `IntegrationFacade.invoke(...)` 后，实际流程如下：

```text
IntegrationFacade
  -> ApiGatewayServiceImpl.invokeOutbound
  -> ApiDefinitionServiceImpl.getSnapshot(apiCode, OUTBOUND)
  -> IApiParamAssembler.assembleOutbound
  -> IApiOutboundExecutor.execute
  -> ApiLogBufferServiceImpl.buffer
```

### 5.2 关键行为

`ApiGatewayServiceImpl.invokeOutbound(...)` 会完成：

1. 读取出站定义快照。
2. 将调用方实体转为 `Map`。
3. 组装 `Body / Query / Header / Path`。
4. 执行外部 HTTP 请求。
5. 记录原始参数、组装后参数、响应结果、耗时、结果类型。

### 5.3 HTTP 客户端

运行时通过 `IntegrationRuntimeConfig` 提供统一 `RestTemplate`，底层使用 `HttpComponentsClientHttpRequestFactory`。

配置位置：

- `D:\mine_product\forgex\Forgex_MOM\Forgex_Backend\Forgex_Integration\src\main\java\com\forgex\integration\config\IntegrationRuntimeConfig.java`

## 6. 入站执行链路

### 6.1 同步执行

入站同步调用流程如下：

```text
PublicIntegrationController
  -> ApiGatewayServiceImpl.invokeInbound
  -> ApiDefinitionServiceImpl.getSnapshot(apiCode, INBOUND)
  -> 授权校验
  -> IApiParamAssembler.assembleInbound
  -> ApiInterpreterRegistry.get(processorBean)
  -> integrationSyncExecutor 提交执行
  -> ApiInboundInterpreter.handle(context, payload)
  -> ApiLogBufferServiceImpl.buffer
```

同步执行时，解释器运行在线程池 `integrationSyncExecutor` 中，而不是直接复用 Web 请求线程。

### 6.2 解释器 SPI

入站业务处理统一实现 `ApiInboundInterpreter`：

```java
Object handle(ApiExecutionContext context, Object payload)
```

运行期通过 `processorBean` 从解释器注册表中直接查找对应 Bean，不再使用反射调用通用 `handle(Object)`。

SPI 位置：

- `D:\mine_product\forgex\Forgex_MOM\Forgex_Backend\Forgex_Integration\src\main\java\com\forgex\integration\spi\ApiInboundInterpreter.java`

## 7. 异步任务模型

### 7.1 提交流程

当入站接口配置为 `ASYNC` 时，`ApiGatewayServiceImpl.invokeInbound(...)` 会调用 `IApiTaskService.submit(...)`，流程如下：

```text
invokeInbound
  -> 写 fx_api_task
  -> 推 Redis 就绪队列
  -> 立即返回 accepted/taskId/traceId/status
```

任务表字段重点包括：

- `task_id`
- `trace_id`
- `api_code`
- `processor_bean`
- `request_payload`
- `assembled_payload`
- `status`
- `retry_count`
- `next_execute_time`
- `lease_expire_time`
- `result_type`
- `error_message`

### 7.2 消费流程

`ApiTaskConsumer` 通过定时拉取 Redis 队列中的 `taskId`，然后投递到 `integrationAsyncConsumerExecutor` 并发执行。

处理逻辑如下：

1. 从 Redis 队列取出 `taskId`。
2. 调用 `apiTaskService.claim(taskId)` 抢占任务。
3. 恢复租户上下文。
4. 读取任务中的组装参数。
5. 按 `processorBean` 找到解释器。
6. 执行业务逻辑。
7. 成功则写结果表并删除任务表。
8. 失败则进入重试或最终失败。

实现位置：

- `D:\mine_product\forgex\Forgex_MOM\Forgex_Backend\Forgex_Integration\src\main\java\com\forgex\integration\service\impl\ApiTaskServiceImpl.java`
- `D:\mine_product\forgex\Forgex_MOM\Forgex_Backend\Forgex_Integration\src\main\java\com\forgex\integration\service\impl\ApiTaskConsumer.java`

### 7.3 补偿与可靠性

`ApiTaskServiceImpl.requeueExpiredTasks()` 使用定时任务扫描以下任务并重新入队：

- `WAITING` 且到达执行时间
- `RETRY` 且到达下次执行时间
- `RUNNING` 且租约过期

这使平台具备以下能力：

- Redis 推送失败后的补偿捞起
- 消费实例宕机后的租约超时重试
- 异步任务可控重试

### 7.4 结果保存

异步任务完成后，结果不会继续留在任务表，而是写入 `fx_api_task_result`，然后删除 `fx_api_task` 中的记录。

对外结果查询固定按 `taskId` 查询，不再从日志月表中查最终结果。

## 8. 调用日志与可观测性

### 8.1 日志写入方式

`ApiGatewayServiceImpl` 会在同步、异步受理、同步失败等节点统一构造 `ApiCallLog`，然后交给 `ApiLogBufferServiceImpl.buffer(...)`。

日志不会立即单条入库，而是先进入内存队列，再通过定时任务批量刷库。

### 8.2 日志缓冲实现

`ApiLogBufferServiceImpl` 使用 `ConcurrentLinkedQueue<ApiCallLog>` 暂存日志，定时批量执行 `apiCallLogService.batchSaveLogs(batch)`。

这种做法的目标是：

- 降低高并发下数据库插入压力
- 减少一次调用一次 insert 的放大效应
- 保持调用主链路尽可能短

实现位置：

- `D:\mine_product\forgex\Forgex_MOM\Forgex_Backend\Forgex_Integration\src\main\java\com\forgex\integration\service\impl\ApiLogBufferServiceImpl.java`

### 8.3 结果类型

当前平台统一结果类型枚举包括：

- `SUCCESS`
- `BUSINESS_FAIL`
- `SYSTEM_FAIL`
- `TIMEOUT`
- `REJECTED`
- `AUTH_FAIL`

## 9. 线程池与运行参数

平台不再复用通用异步线程池，而是独立配置三类线程池：

- `integrationSyncExecutor`
- `integrationAsyncConsumerExecutor`
- `integrationLogExecutor`

线程池与缓存、队列参数统一由 `IntegrationProperties` 管理，并在 `IntegrationRuntimeConfig` 中初始化。

代码位置：

- `D:\mine_product\forgex\Forgex_MOM\Forgex_Backend\Forgex_Integration\src\main\java\com\forgex\integration\config\IntegrationProperties.java`
- `D:\mine_product\forgex\Forgex_MOM\Forgex_Backend\Forgex_Integration\src\main\java\com\forgex\integration\config\IntegrationRuntimeConfig.java`
- `D:\mine_product\forgex\Forgex_MOM\Forgex_Backend\Forgex_Integration\src\main\resources\application.yml`

当前支持的主要运行参数：

- 本地缓存 TTL
- Redis 缓存 TTL
- 本地缓存容量
- Redis 队列键
- 队列补偿批次
- 任务租约时长
- 日志批量刷库条数
- 日志刷库间隔
- 三类线程池的 core/max/queue/keepAlive/rejectPolicy
- 异步结果保留天数

## 10. 已移除的旧实现

由于用户尚未正式使用旧接口平台，当前版本已直接移除旧兼容层，文档和代码均以新执行引擎为准。以下类不再是现行入口：

- `IntegrationController`
- `IApiRouterService`
- `ApiRouterServiceImpl`

后续如需扩展，只应基于 `IntegrationFacade`、`PublicIntegrationController`、`ApiGatewayServiceImpl` 和解释器 SPI 继续演进。
