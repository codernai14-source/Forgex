# 文件上传

> 分类：后端 / 模块专题  
> 版本：**V0.5.0**  
> 更新时间：**2026-04-13**

## 一、概述

Forgex 提供统一的文件上传、下载、删除能力，支持 **本地存储** 和 **OSS 对象存储** 两种方式，通过工厂模式自动选择存储策略。

## 二、核心组件

| 组件 | 包路径 | 作用 |
|---|---|---|
| `FileController` | `com.forgex.sys.controller` | 文件上传入口 Controller |
| `FileService` | `com.forgex.sys.service` | 文件服务接口 |
| `FileServiceImpl` | `com.forgex.sys.service.impl` | 文件服务实现 |
| `FileStorageService` | `com.forgex.sys.service.storage` | 存储服务接口 |
| `LocalStorageService` | `com.forgex.sys.service.storage` | 本地文件存储实现 |
| `OssStorageService` | `com.forgex.sys.service.storage` | OSS 对象存储实现 |
| `FileStorageFactory` | `com.forgex.sys.service.storage` | 存储策略工厂 |

## 三、接口列表

### 3.1 上传文件

```
POST /sys/file/upload
Content-Type: multipart/form-data
参数: file (MultipartFile)

返回: R<String>  → 文件访问 URL
```

### 3.2 下载文件

```
GET /sys/file/download?filename=xxx
返回: 文件流 (application/octet-stream)
```

### 3.3 获取文件

```
GET /sys/file/get?filename=xxx
返回: 文件流（自动识别 MediaType）
```

## 四、存储策略

### 4.1 工厂优先级

`FileStorageFactory.getDefault()` 按以下优先级选择存储策略：

```
1. sys_config 全局配置 (key: file.upload.settings)
   → 若 storageType 有值，使用对应存储
2. sys_file_storage 租户表配置
   → 若租户配置了存储信息，使用租户存储
3. 兜底使用本地存储 LocalStorageService
```

### 4.2 本地存储

**配置方式：**

```yaml
# application.yml
file:
  upload:
    path: ./uploads       # 上传目录
  access:
    prefix: /files        # 访问前缀
```

也可通过 `sys_config` 的 `file.upload.settings` 键动态配置：

```json
{
  "storageType": "local",
  "localUploadPath": "/data/forgex/uploads",
  "accessPrefix": "/files"
}
```

**文件命名规则：**
- UUID 替换文件名，保留原始扩展名
- 路径格式：`{uploadPath}/{uuid}.{ext}`

### 4.3 OSS 对象存储

**配置方式：**

```json
// sys_config: file.upload.settings
{
  "storageType": "oss",
  "endpoint": "https://oss-cn-hangzhou.aliyuncs.com",
  "accessKeyId": "...",
  "accessKeySecret": "...",
  "bucketName": "forgex-files"
}
```

**文件命名规则：**
- 路径格式：`yyyy/MM/dd/{uuid}.{ext}`
- 如：`2026/04/13/a1b2c3d4e5f6.xlsx`

## 五、FileStorageService 接口

```java
public interface FileStorageService {
    /** 上传文件，返回文件相对路径 */
    String upload(MultipartFile file) throws IOException;

    /** 下载文件到 HTTP 响应 */
    void download(String filePath, HttpServletResponse response) throws IOException;

    /** 删除文件 */
    boolean delete(String filePath);

    /** 获取文件访问 URL */
    String getUrl(String filePath);
}
```

## 六、上传流程

```
前端上传文件
  → POST /sys/file/upload (multipart/form-data)
  → FileController.upload()
  → FileServiceImpl.upload()
  → FileStorageFactory.getDefault() 获取存储策略
  → StorageService.upload(file) 执行上传
  → StorageService.getUrl(relativePath) 获取访问 URL
  → 返回 R.ok(url)
```

## 七、最佳实践

1. **生产环境建议使用 OSS**，避免单机存储风险
2. **文件大小限制**通过 Spring Boot 配置：
   ```yaml
   spring:
     servlet:
       multipart:
         max-file-size: 100MB
         max-request-size: 100MB
   ```
3. **敏感文件启用加密存储**（参见 [加密功能](./加密功能.md)）
4. 文件 URL 不应直接暴露存储路径，应通过访问前缀映射
