# 代码生成使用方式

> 分类：后端 / 模块专题  
> 版本：**V0.1.0**  
> 更新时间：**2026-04-22**

本文说明 Forgex 代码生成模块的使用方式，涵盖前端配置界面、API 调用、生成文件说明和集成步骤。

## 1. 使用流程概览

代码生成的标准使用流程如下：

1. **配置数据源**：添加数据库连接配置
2. **选择表结构**：从数据源中选择目标表
3. **配置生成参数**：设置模块名、实体名、包名、菜单等
4. **预览生成结果**：在线预览生成的代码文件
5. **下载 ZIP 包**：下载并集成到项目中
6. **执行 SQL 脚本**：执行生成的菜单、表格配置等 SQL

## 2. 前端配置界面

### 2.1 数据源管理

通过 `/sys/codegen/datasource` 管理数据源：

| API | 说明 |
|-----|------|
| `POST /page` | 分页查询数据源列表 |
| `POST /list` | 查询数据源列表 |
| `GET /detail/{id}` | 获取数据源详情 |
| `POST /create` | 创建数据源 |
| `POST /update` | 更新数据源 |
| `POST /delete/{id}` | 删除数据源 |

数据源配置字段：

| 字段 | 说明 | 示例 |
|------|------|------|
| `datasourceCode` | 数据源编码 | `primary` |
| `datasourceName` | 数据源名称 | `主数据源` |
| `datasourceType` | 数据源类型 | `MySQL` |
| `host` | 主机地址 | `192.168.1.100` |
| `port` | 端口 | `3306` |
| `username` | 用户名 | `root` |
| `password` | 密码 | `******`（加密存储） |
| `database` | 数据库名 | `forgex_mom` |

### 2.2 元数据查询

通过 `/sys/codegen/meta` 查询数据库元数据：

| API | 说明 |
|-----|------|
| `POST /schemas` | 查询 schema 列表 |
| `POST /tables` | 查询表列表 |
| `POST /columns` | 查询列列表 |
| `POST /table-meta` | 查询表元数据详情 |

请求参数示例：

```json
{
  "datasourceId": 1,
  "schemaName": "forgex_mom",
  "tableName": "fx_material"
}
```

### 2.3 配置记录管理

通过 `/sys/codegen/config` 管理生成配置记录：

| API | 说明 |
|-----|------|
| `POST /page` | 分页查询配置记录 |
| `POST /detail` | 查询配置详情 |
| `POST /save` | 保存配置 |
| `POST /delete` | 删除配置 |

保存配置后，可通过 `configId` 直接预览或下载。

### 2.4 代码生成

通过 `/sys/codegen` 执行生成：

| API | 说明 |
|-----|------|
| `POST /preview` | 预览生成代码 |
| `POST /download` | 下载 ZIP 包 |

## 3. API 调用方式

### 3.1 预览生成代码

```bash
POST /sys/codegen/preview
Content-Type: application/json

{
  "datasourceId": 1,
  "schemaName": "forgex_mom",
  "pageType": "SINGLE",
  "mainTableName": "fx_material",
  "moduleName": "base",
  "bizName": "material",
  "entityName": "Material",
  "packageName": "com.forgex.base",
  "author": "Forgex Team",
  "menuName": "物料管理",
  "menuIcon": "AppstoreOutlined",
  "parentMenuPath": "/base",
  "generateItems": ["backend", "frontend", "sql"]
}
```

响应示例：

```json
{
  "code": 200,
  "data": {
    "zipFileName": "codegen-material-20260422001.zip",
    "files": [
      {
        "path": "backend/com/forgex/base/domain/entity/Material.java",
        "name": "Material.java",
        "category": "backend",
        "content": "// 生成的实体类代码..."
      },
      {
        "path": "frontend/src/views/base/material/index.vue",
        "name": "index.vue",
        "category": "frontend",
        "content": "// 生成的 Vue 页面代码..."
      }
    ]
  }
}
```

### 3.2 下载 ZIP 包

```bash
POST /sys/codegen/download
Content-Type: application/json

{
  "datasourceId": 1,
  "schemaName": "forgex_mom",
  "pageType": "SINGLE",
  "mainTableName": "fx_material",
  "moduleName": "base",
  "bizName": "material"
}
```

响应为 ZIP 文件流，浏览器自动下载。

### 3.3 通过配置 ID 生成

如果已保存配置记录：

```bash
POST /sys/codegen/preview
Content-Type: application/json

{
  "configId": 123
}
```

## 4. 生成请求参数说明

### 4.1 基础参数

| 参数 | 必填 | 说明 | 默认值 |
|------|------|------|--------|
| `datasourceId` | 是 | 数据源 ID | - |
| `schemaName` | 是 | schema 名称 | - |
| `pageType` | 是 | 页面类型：SINGLE / MASTER_DETAIL | - |
| `mainTableName` | 是 | 主表名称 | - |

### 4.2 主子表关联参数（MASTER_DETAIL 必填）

| 参数 | 必填 | 说明 |
|------|------|------|
| `subTableName` | 是 | 子表名称 |
| `mainPkColumn` | 是 | 主表主键列 |
| `subFkColumn` | 是 | 子表外键列（关联主表主键） |
| `subPkColumn` | 是 | 子表主键列 |

### 4.3 命名与包配置

| 参数 | 必填 | 说明 | 默认值 |
|------|------|------|--------|
| `moduleName` | 否 | 模块编码 | `system` |
| `bizName` | 否 | 业务标识 | 实体名小写 |
| `entityName` | 否 | 实体名称 | 表名驼峰转换 |
| `subEntityName` | 否 | 子实体名称 | 子表名驼峰转换 |
| `packageName` | 否 | 包名 | `com.forgex.{moduleName}` |
| `author` | 否 | 作者 | `Forgex CodeGen` |

### 4.4 菜单配置

| 参数 | 必填 | 说明 | 默认值 |
|------|------|------|--------|
| `menuName` | 否 | 菜单名称 | 表注释或实体名 |
| `menuIcon` | 否 | 菜单图标 | `TableOutlined` |
| `parentMenuPath` | 否 | 父级菜单路径 | `/system/codegen` |
| `tableCodePrefix` | 否 | 表格编码前缀 | 模块编码 |

### 4.5 生成项配置

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `generateItems` | 生成项列表 | `["backend", "frontend", "sql"]` |

可选值：
- `backend` - 后端代码（Entity、Controller、Service、Mapper、Param）
- `frontend` - 前端代码（Vue 页面、API 接口）
- `sql` - SQL 脚本（菜单权限、表格配置、国际化）

### 4.6 字段配置

| 参数 | 说明 |
|------|------|
| `mainColumns` | 主表字段配置列表（覆盖自动采集） |
| `subColumns` | 子表字段配置列表 |

字段配置（`ColumnMetaDTO`）可覆盖自动推断的特性：

| 字段 | 说明 |
|------|------|
| `queryType` | 查询类型：input、select、date |
| `queryOperator` | 查询操作符：like、eq、gt、lt、between |
| `formType` | 表单类型：input、textarea、number、select、date、datetime |
| `dictCode` | 字典编码 |
| `required` | 是否必填 |
| `width` | 列宽度 |
| `align` | 对齐方式 |
| `sortable` | 是否可排序 |
| `renderType` | 渲染类型 |

## 5. 主子表使用示例

### 5.1 主子表场景

适用于订单-订单明细、入库单-入库明细等主子表联动场景。

### 5.2 主子表请求示例

```json
{
  "datasourceId": 1,
  "schemaName": "forgex_mom",
  "pageType": "MASTER_DETAIL",
  "mainTableName": "fx_purchase_order",
  "subTableName": "fx_purchase_order_item",
  "mainPkColumn": "id",
  "subFkColumn": "order_id",
  "subPkColumn": "id",
  "moduleName": "purchase",
  "bizName": "order",
  "entityName": "PurchaseOrder",
  "subEntityName": "PurchaseOrderItem",
  "packageName": "com.forgex.purchase",
  "menuName": "采购订单",
  "parentMenuPath": "/purchase"
}
```

### 5.3 主子表关联约束

系统会校验以下约束：

1. 主表主键列必须存在
2. 子表外键列必须存在且类型与主表主键一致
3. 子表主键列必须存在
4. 子表默认排序字段（优先 `sort_no`、`sort_num`、`order_num`、`create_time`、`id`）

## 6. 生成的 SQL 脚本执行

### 6.1 菜单权限 SQL

生成文件：`sql/01_menu_permission.sql`

内容包括：
- 菜单记录（`sys_menu`）
- 权限标识（`sys_menu_perm`）
- 角色权限关联（如需默认授权）

执行顺序：
1. 先执行菜单 SQL，创建菜单入口
2. 登录系统后可在新菜单下操作

### 6.2 表格配置 SQL

生成文件：`sql/02_table_config.sql`

内容包括：
- 表格配置（`fx_table_config`）
- 列配置（`fx_table_column_config`）

执行后，前端 `FxDynamicTable` 会自动读取配置显示列。

### 6.3 国际化 SQL

生成文件：`sql/03_i18n_message.sql`

内容包括：
- 中文提示消息（`i18n_message_zh_CN`）
- 英文提示消息（`i18n_message_en_US`）

### 6.4 响应消息 SQL

生成文件：`sql/04_response_message.sql`

内容包括：
- 响应提示枚举（对应 `PromptEnum`）

## 7. 生成文件集成步骤

### 7.1 后端代码集成

1. 将 `backend/` 目录下的代码复制到对应模块
2. 确保包名与模块目录结构一致
3. 添加 MyBatis-Plus Mapper XML（如需自定义 SQL）
4. 在启动类或配置类中确保 Mapper 扫描路径正确

### 7.2 前端代码集成

1. 将 `frontend/src/views/` 下的页面复制到前端项目
2. 将 `frontend/src/api/` 下的 API 文件复制到前端项目
3. 在路由配置中添加页面路由（或通过菜单配置自动加载）
4. 确保前端项目已配置好 API 请求基础路径

### 7.3 SQL 脚本执行

按顺序执行：
1. `01_menu_permission.sql`
2. `02_table_config.sql`
3. `03_i18n_message.sql`
4. `04_response_message.sql`

执行后刷新系统缓存：

```bash
# 清除 Redis 缓存（如有）
redis-cli DEL "forgex:table:config:*"
redis-cli DEL "forgex:i18n:*"
```

## 8. 常见问题

### 8.1 表名转换规则

表名（下划线命名）会自动转换为驼峰实体名：

- `fx_material` → `Material`
- `fx_purchase_order` → `PurchaseOrder`
- `sys_user_info` → `UserInfo`

### 8.2 字段类型映射

数据库类型自动映射为 Java 类型：

| 数据库类型 | Java 类型 |
|------------|-----------|
| `varchar`, `char`, `text` | `String` |
| `int`, `tinyint` | `Integer` |
| `bigint` | `Long` |
| `decimal`, `numeric` | `BigDecimal` |
| `datetime`, `timestamp` | `LocalDateTime` |
| `date` | `LocalDate` |
| `boolean`, `bit` | `Boolean` |

### 8.3 基础字段过滤

以下字段会被识别为基础字段，不生成在查询/表单配置中：

- `id`
- `tenant_id`
- `create_time`
- `create_by`
- `update_time`
- `update_by`
- `deleted`

### 8.4 生成项选择

如只需生成部分代码，可指定 `generateItems`：

```json
{
  "generateItems": ["backend", "sql"]
}
```

`doc` 类（README.md）始终生成。
