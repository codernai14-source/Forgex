# 加密功能使用方式

> 分类：后端 / 模块专题
> 版本：**V0.6.5**
> 更新时间：**2026-04-17**

本文重点说明两件事：

- 代码里怎么正确接入加密能力
- 页面和配置层面怎么启用、验证和排查

## 使用原则

| 场景 | 推荐方式 |
|---|---|
| 登录密码存储 | `bcrypt` 或更强哈希策略 |
| 登录口令传输保护 | `SM2` 或 `RSA` 公钥加密 |
| 数据库存敏感字段 | `@FieldEncrypt` |
| 敏感附件或归档文件 | `FileEncryptService` |
| 业务密钥、第三方密钥托管 | `KmsService` |

## 代码如何使用

### 1. 密码或口令策略

统一通过 `CryptoProviders` 解析，不要手写算法分支：

```java
CryptoPasswordProvider provider = CryptoProviders.resolve("bcrypt", configService);
String cipher = provider.encrypt("plain-text");
boolean ok = provider.verify("plain-text", cipher);
```

### 2. 字段透明加密

实体字段上直接标注 `@FieldEncrypt`：

```java
@FieldEncrypt(algorithm = "sm4")
private String phone;
```

接入要求：

1. 字段必须是 `String`
2. 数据通过 MyBatis 链路读写
3. 不要在 Service 层再次手工加密同一字段

### 3. 文件加密

需要对附件落密文时，使用 `FileEncryptService`：

```java
fileEncryptService.encryptFile(sourceFile, encryptedFile, "aes");
fileEncryptService.decryptFile(encryptedFile, plainFile, "aes");
```

### 4. KMS 业务密钥管理

需要“生成密钥并长期托管”时，接 `KmsService`：

```java
Long keyId = kmsService.generateKey("third-party-sign-key", "AES", 256, "第三方签名密钥");
String activeKey = kmsService.getActiveKey("third-party-sign-key");
Long newKeyId = kmsService.rotateKey("third-party-sign-key");
```

## 页面如何使用

### 1. 系统配置页面

如果项目已接入系统配置管理页，需要维护这些配置键：

- `security.crypto.transport`
- `security.crypto.sm4`
- `security.crypto.aes`
- `security.crypto.rsa`
- `security.kms.master`

页面操作重点：

1. 先确认配置键名称完全一致
2. 再填算法所需密钥或参数
3. 保存后重新验证对应链路

### 2. 登录页面

如果登录页启用了前端公钥加密，需要先拿公钥，再对密码做加密提交。这个使用方式和认证模块联动，详细入口见认证文档。

### 3. 文件上传页面

如果上传页面对应的是敏感附件场景，要在后端上传流程里串上 `FileEncryptService`。前端页面本身仍按正常上传交互走，不需要额外感知算法细节。

常见页面入口可参考：

- `Forgex_MOM/Forgex_Fronted/src/components/AvatarUpload.vue`
- `Forgex_MOM/Forgex_Fronted/src/views/profile/index.vue`
- `Forgex_MOM/Forgex_Fronted/src/api/system/file.ts`

## 常见误区

### 误区 1：字段加密后还在 Service 手工加密

会造成双重加密，查询后解不回来。只要用了 `@FieldEncrypt`，就不要再手工二次处理。

### 误区 2：开发环境让系统自动生成密钥，生产也这样做

生产环境必须固定配置密钥并纳入配置管理。否则一旦环境重建，历史数据就可能无法解密。

### 误区 3：把 KMS 当成普通配置表

业务接入时应优先使用服务接口，不要直接操作表。

## 排查清单

| 现象 | 先查什么 |
|---|---|
| 登录密文校验失败 | 传输算法、前端加密方式、后端解密配置是否一致 |
| 字段查出来还是密文 | 是否加了 `@FieldEncrypt`，是否带 `ENC:` 前缀 |
| 文件加密后打不开 | 上传时算法和下载时算法是否一致，密钥是否一致 |
| 换环境后历史数据解不开 | `security.crypto.*` 与 `security.kms.master` 是否迁移 |
| KMS 轮换后业务报错 | 是否仍在读取旧版本或写死旧别名 |
