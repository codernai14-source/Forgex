# 基础数据与物料实现逻辑

> 分类：后端 / 模块专题  
> 版本：**V0.6.0**  
> 更新时间：**2026-04-17**

本文档说明 `Forgex_Basic` 模块中基础业务探针和物料管理的实现结构。

## 模块定位

`Forgex_Basic` 承担基础业务扩展位，当前重点是物料管理与模块探针。

## 核心入口

| 入口 | 作用 |
|---|---|
| `BasicModuleController` | 模块健康探针 |
| `BasicMaterialController` | 物料管理 |
| `SupplierController` | 供应商主数据管理 |
| `BasicUnitController` | 计量单位类型、单位主数据与换算关系 |
| `BasicUnitInternalController` | 计量单位公共换算内部接口 |

## 物料链路

```text
物料新增 / 更新
  -> 编码规则生成物料编码
  -> 字典解析类型、状态
  -> 多租户入库
  -> 可联动导入导出
```

## 供应商主数据链路

供应商主数据位于 `Forgex_Basic` 模块，前端页面位于：

```text
Forgex_MOM/Forgex_Fronted/src/views/basic/supplier/index.vue
```

核心实现分层：

| 层级 | 说明 |
|---|---|
| `SupplierController` | 暴露分页、详情、新增、更新、删除、导入导出、生成租户、发起审查等接口 |
| `ISupplierService` / `SupplierServiceImpl` | 负责供应商主表、详情、联系人、资质、租户生成和工作流联动 |
| `BasicSupplier` | 供应商主表实体 |
| `BasicSupplierDetail` | 供应商扩展详情 |
| `BasicSupplierContact` | 供应商联系人 |
| `BasicSupplierQualification` | 供应商资质 |

新增 / 更新链路：

```text
前端供应商弹窗
  -> 校验供应商编码或自动生成配置
  -> 调用 supplierApi.create / supplierApi.update
  -> 后端保存主表、详情、联系人、资质
  -> 刷新 FxDynamicTable 列表
```

供应商新增弹窗使用公共 `BaseFormDialog`，并在业务调用处开启 `maskClosable`。该行为只作用于供应商页面，不改变公共弹窗默认的复杂表单保护策略。

## 计量单位链路

计量单位采用“类型树 + 单位主数据 + 换算关系”三表模型：

| 表 | 说明 |
|---|---|
| `basic_unit_type` | 计量单位类型树，维护类型编码、类型名称、父 ID 和层级路径 |
| `basic_unit` | 计量单位主数据，按类型维护单位编码、单位名称和备注 |
| `basic_unit_conversion` | 计量单位换算关系，表示 `1 源单位 = conversion_value 目标单位` |

计量单位三张业务表随 `Forgex_Basic` 默认主数据源存放在 `forgex_admin`；公共表格配置、国际化提示等公共配置仍存放在 `forgex_common`。旧版单表 `basic_unit` 升级时会按租户创建 `DEFAULT/默认类型` 并回填 `unit_type_id`，避免列表接口查询新字段时报错。

后端实现位置：

```text
Forgex_MOM/Forgex_Backend/Forgex_Basic/src/main/java/com/forgex/basic/material/controller/BasicUnitController.java
Forgex_MOM/Forgex_Backend/Forgex_Basic/src/main/java/com/forgex/basic/material/service/impl/BasicUnitServiceImpl.java
Forgex_MOM/Forgex_Backend/Forgex_Common/src/main/java/com/forgex/common/api/dto/UnitConvertRequestDTO.java
Forgex_MOM/Forgex_Backend/Forgex_Common/src/main/java/com/forgex/common/api/feign/BasicUnitConversionFeignClient.java
```

删除保护规则在服务层统一执行：类型下存在子类型或单位时禁止删除；单位存在源或目标换算关系时禁止删除。换算配置保存按源单位执行全量替换：前端提交的多行关系会保留或更新，未提交的源单位旧关系会被删除。

公共换算方法使用严格模式。调用方传入实体数据、目标单位编码、实体内单位字段名、单位字段类型 `ID/CODE` 和待换算数字字段名数组。服务会查询源单位、目标单位与直接关系；缺少直接关系时允许使用反向关系并执行除法。同单位直接返回原值。单位、字段、关系缺失或字段非数字时抛业务异常，计算使用 `BigDecimal`，不做默认四舍五入。
