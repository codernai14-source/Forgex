# 代码生成实现逻辑

> 分类：后端 / 模块专题  
> 版本：**V0.6.0**  
> 更新时间：**2026-04-22**

本文说明 Forgex 代码生成模块的核心实现结构，涵盖策略模式、模板渲染、数据库元数据采集和生成流程。

## 1. 总体架构

代码生成模块采用**策略模式 + Beetl 模板引擎**的设计，核心组件如下：

- **CodeGenService**：生成服务入口，负责组装上下文、调用策略、打包输出
- **CodeGenStrategyFactory**：策略工厂，根据页面类型选择对应的生成策略
- **CodeGenStrategy**：策略接口，定义生成文件列表的标准行为
- **AbstractCodeGenStrategy**：策略抽象基类，封装模板渲染和文件添加逻辑
- **CodegenTemplateRenderer**：Beetl 模板渲染器，将上下文绑定到模板并输出代码
- **DbMetaService**：数据库元数据服务，采集表结构、列信息

代码位置：

- `Forgex_Sys/src/main/java/com/forgex/sys/service/impl/CodeGenServiceImpl.java`
- `Forgex_Sys/src/main/java/com/forgex/sys/service/codegen/`
- `Forgex_Sys/src/main/java/com/forgex/sys/service/codegen/template/CodegenTemplateRenderer.java`

前端数据源配置入口：

- `Forgex_MOM/Forgex_Fronted/src/views/system/codegenDatasource/index.vue`
- `Forgex_MOM/Forgex_Fronted/src/views/system/codegenDatasource/components/CodegenDatasourceFormDialog.vue`

数据源表单弹窗使用公共 `BaseFormDialog` 承载，页面层负责打开、回显、测试连接、保存与关闭。当前数据源新增 / 编辑弹窗在调用处开启 `maskClosable`，仅影响该页面的遮罩点击关闭行为，不改变公共弹窗组件默认策略。

## 2. 策略模式设计

### 2.1 策略接口

```java
public interface CodeGenStrategy {

    /**
     * 生成文件列表
     *
     * @param context 生成上下文

}
```

### 2.2 策略工厂

策略工厂通过 Spring 自动注入所有 `CodeGenStrategy` 实现，根据页面类型匹配：

```java
@Component
public class CodeGenStrategyFactory {
    private final List<CodeGenStrategy> strategies;

    public CodeGenStrategy getStrategy(String pageType) {
        return strategies.stream()
            .filter(strategy -> strategy.supports(pageType))
            .findFirst()
            .orElseThrow(() -> new I18nBusinessException(...));
    }
}
```

策略工厂位置：

- `Forgex_Sys/src/main/java/com/forgex/sys/service/codegen/CodeGenStrategyFactory.java`

### 2.4 策略抽象基类

抽象基类封装通用逻辑：

```java
public abstract class AbstractCodeGenStrategy implements CodeGenStrategy {
    protected final CodegenTemplateRenderer templateRenderer;

    protected void addFile(List<CodegenRenderFileDTO> files, String category, String path,
                           String templateName, CodeGenContextDTO context) {
        if (!shouldGenerate(category, context)) {
            return;
        }
        files.add(new CodegenRenderFileDTO(path, category, 
            templateRenderer.render(templateName, context)));
    }

    protected boolean shouldGenerate(String category, CodeGenContextDTO context) {
        if ("doc".equalsIgnoreCase(category)) {
            return true;
        }
        return context.getGenerateItems().stream()
            .anyMatch(item -> item != null && item.equalsIgnoreCase(category));
    }
}
```

抽象基类位置：

- `Forgex_Sys/src/main/java/com/forgex/sys/service/codegen/AbstractCodeGenStrategy.java`

```java
@Component
public class CodegenTemplateRenderer {
    private final GroupTemplate groupTemplate;

    public String render(String templateName, CodeGenContextDTO context) {
        Template template = groupTemplate.getTemplate(templateName);
        template.binding("ctx", context);
        template.binding("moduleName", context.getModuleName());
        template.binding("entityName", context.getEntityName());
        template.binding("mainColumns", context.getMainColumns());
        // ... 更多绑定

    }
```

### 5.2 字段特性推断

上下文组装时会根据字段类型自动推断默认特性：

```java
private String resolveQueryType(String javaType) {
    if ("LocalDate".equals(javaType) || "LocalDateTime".equals(javaType)) {
        return "date";
    }
    return "input";
}

private String resolveFormType(ColumnMetaDTO column) {
    String javaType = column.getJavaType();
    if ("LocalDate".equals(javaType)) return "date";
    if ("LocalDateTime".equals(javaType)) return "datetime";
    if ("BigDecimal".equals(javaType) || "Integer".equals(javaType)) return "number";
    if ("Boolean".equals(javaType)) return "select";
    if (column.getCharacterMaximumLength() > 200) return "textarea";
    return "input";
}

private Integer resolveWidth(String javaType) {
    if ("LocalDateTime".equals(javaType)) return 180;
    if ("LocalDate".equals(javaType)) return 140;
    if ("Integer".equals(javaType) || "BigDecimal".equals(javaType)) return 120;
    return 160;
}

private String resolveAlign(String javaType) {
    if ("Integer".equals(javaType) || "BigDecimal".equals(javaType)) return "right";
    if ("LocalDate".equals(javaType)) return "center";
    return "left";
}
```

推断逻辑位于：

## 9. 扩展策略

如需扩展新的页面类型，只需：

1. 实现 `CodeGenStrategy` 接口
2. 添加 `@Component` 注解，Spring 会自动注册到工厂
3. 在 `resources/templates/codegen/` 目录下添加对应模板文件

示例：

```java
@Component
public class TreeTableCodeGenStrategy extends AbstractCodeGenStrategy {
    @Override
    public boolean supports(String pageType) {
        return "TREE".equalsIgnoreCase(pageType);
    }

    @Override
    public List<CodegenRenderFileDTO> generate(CodeGenContextDTO context) {
        List<CodegenRenderFileDTO> files = newFiles();
        // 添加树表特有的模板文件
        addFile(files, "backend", ..., "tree/backend/Entity.java.btl", context);
        return files;
    }
}
```
