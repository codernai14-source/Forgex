# 加密功能

> 分类：后端 / 模块专题  
> 版本：**V0.5.0**  
> 更新时间：**2026-04-13**

## 一、概述

Forgex 提供 **多层次、可配置** 的加密能力，覆盖密码存储、传输加密、字段透明加密、文件加密和 KMS 密钥管理。所有加密算法通过 `CryptoProviders` 工厂统一管理，支持 Provider 扩展。

## 二、密码存储加密

### 2.1 Provider 架构

```java
// com.forgex.common.crypto.CryptoProviders
public static CryptoPasswordProvider resolve(String store, ConfigService cfg) {
    switch (store.toLowerCase()) {
        case "bcrypt": return new BcryptPasswordProvider();
        case "sm2":    return new Sm2PasswordProvider(cfg);
        case "sm4":    return new Sm4PasswordProvider(cfg);
        case "aes":    return new AESPasswordProvider(cfg);
        case "rsa":    return new RSAPasswordProvider(cfg);
        default:       return new BcryptPasswordProvider();
    }
}
```

### 2.2 Provider 接口

```java
public interface CryptoPasswordProvider {
    String name();                     // 算法名称
    boolean supportsEncrypt();         // 是否支持可逆加密
    boolean supportsHash();            // 是否支持不可逆哈希
    String encrypt(String plain);      // 加密
    String decrypt(String cipher);     // 解密（仅可逆算法支持）
    String hash(String plain);         // 哈希
    boolean verify(String raw, String encoded); // 校验
}
```

### 2.3 各算法对比

| 算法 | 类型 | 可逆 | 场景 | Provider 类 |
|---|---|---|---|---|
| bcrypt | 哈希 | 否 | 密码存储（默认） | `BcryptPasswordProvider` |
| SM2 | 非对称 | 是 | 国密场景 | `Sm2PasswordProvider` |
| SM4 | 对称 | 是 | 国密场景 | `Sm4PasswordProvider` |
| AES | 对称 | 是 | 通用加密 | `AESPasswordProvider` |
| RSA | 非对称 | 是 | 通用场景 | `RSAPasswordProvider` |

### 2.4 密码策略配置

通过 `sys_config` 的 `security.password.policy` 键配置：

```json
{
  "store": "bcrypt",
  "defaultPassword": "Aa123456"
}
```

## 三、传输加密（SM2）

### 3.1 配置

通过 `sys_config` 的 `security.crypto.transport` 键配置：

```json
{
  "algorithm": "SM2",
  "publicKey": "04...",
  "privateKey": "...",
  "cipher": "HEX"
}
```

### 3.2 前后端交互

```
前端：
  1. GET /auth/publicKey → 获取 SM2 公钥
  2. password = SM2.encryptToHex(rawPassword, publicKey)
  3. POST /auth/login { password: encrypted }

后端：
  1. 读取 security.crypto.transport 配置
  2. SM2 sm2 = new SM2(privateKey, publicKey)
  3. rawPassword = sm2.decryptStr(password) 或 sm2.decryptFromBcd(password)
```

### 3.3 密码格式

| cipher 配置 | 说明 |
|---|---|
| `HEX` | 十六进制编码（默认） |
| `BCD` | BCD 编码格式 |

## 四、字段透明加密

### 4.1 注解 `@FieldEncrypt`

```java
@FieldEncrypt(algorithm = "SM4")
private String phone;

@FieldEncrypt(algorithm = "AES")
private String idCard;
```

### 4.2 拦截器 `FieldEncryptInterceptor`

MyBatis 拦截器，自动在 INSERT/UPDATE 时加密、SELECT 时解密：

```
INSERT/UPDATE:
  1. 扫描实体类带 @FieldEncrypt 的字段
  2. 通过 CryptoProviders 获取对应 Provider
  3. 加密后添加前缀 "ENC:" 存入数据库

SELECT:
  1. 扫描返回对象带 @FieldEncrypt 的字段
  2. 检查值是否以 "ENC:" 开头
  3. 去除前缀后解密还原
```

### 4.3 配置开关

通过 `sys_config` 的 `security.field.encrypt` 键配置：

```json
{
  "enabled": true,
  "defaultAlgorithm": "SM4"
}
```

## 五、文件加密

通过 `sys_config` 的 `security.file.encrypt` 键配置：

```json
{
  "enabled": true,
  "defaultAlgorithm": "aes"
}
```

支持上传文件时自动加密、下载文件时自动解密。

## 六、KMS 密钥管理

### 6.1 配置

通过 `sys_config` 的 `security.kms.master` 键配置：

```json
{
  "masterKeyHex": "64字符Hex（256位主密钥）",
  "rotateRemindDays": 90
}
```

### 6.2 密钥生成接口

```
POST /sys/config/crypto/generate/sm4  → 生成 SM4 密钥（128位，32字符Hex）
POST /sys/config/crypto/generate/aes  → 生成 AES 密钥
POST /sys/config/crypto/generate/rsa  → 生成 RSA 密钥对
POST /sys/config/crypto/generate/sm2  → 生成 SM2 密钥对
```

## 七、加密配置管理接口

```
GET  /sys/config/crypto       → 获取完整加密配置
PUT  /sys/config/crypto       → 保存加密配置
```

返回结构 `CryptoConfig`：

```json
{
  "sm4":          { "keyHex": "..." },
  "aes":          { "keyHex": "..." },
  "rsa":          { "publicKey": "...", "privateKey": "...", "keySize": 2048 },
  "kms":          { "masterKeyHex": "...", "rotateRemindDays": 90 },
  "fileEncrypt":  { "enabled": false, "defaultAlgorithm": "aes" },
  "fieldEncrypt": { "enabled": false, "defaultAlgorithm": "SM4" }
}
```

## 八、与数据库 TDE 的协同

- **应用层加密**（本文档）：保护特定敏感字段，应用可控
- **数据库 TDE**：整表/表空间级透明加密，防物理介质泄露
- 两者互补，建议同时启用
