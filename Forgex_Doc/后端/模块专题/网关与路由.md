# 网关与路由

> 分类：后端 / 模块专题  
> 版本：**V0.5.0**  
> 更新时间：**2026-04-15**

本专题用于说明 `Forgex_Gateway` 的职责、认证过滤、授权校验、路由转发和部署关注点。

## 一、模块定位

`Forgex_Gateway` 是 Forgex 的统一网关入口，负责：

- API 路由转发
- 登录态解析
- 网关级鉴权
- 授权状态校验
- 国际化消息兜底翻译
- Fallback 与熔断链路

当前服务名：`forgex-gateway`  
默认端口：`9000`

## 二、关键入口

| 入口 | 说明 |
|---|---|
| `ForgexGatewayApplication` | 网关服务启动类 |
| `GatewayAuthGlobalFilter` | 全局认证过滤器 |
| `FallbackController` | 路由熔断/兜底入口 |
| `application.yml` | 路由、限流、熔断、授权文件等配置 |

## 三、当前网关规则

### 3.1 受保护资源

网关当前会对以下路径进行统一认证判断：

- `/api/sys/**`
- `/api/basic/**`
- `/api/files/**`
- `/api/app/**`

### 3.2 已放行入口

当前已显式放行的典型路径包括：

- `/api/auth/**`
- `/api/sys/config/system-basic`
- `/api/sys/config/login-captcha`
- `/api/sys/i18n/languageType/listEnabled`
- `/api/sys/init/status`
- `/api/sys/init/apply`
- `/api/sys/license/status`
- `/api/sys/license/request-info`
- `/api/sys/license/logs`
- `/api/sys/license/refresh`
- `/api/basic/module/ping`
- `/api/files/**` 的只读访问

### 3.3 授权联动

网关会在登录态检查前先执行授权状态校验：

- 未授权：`LICENSE_REQUIRED`
- 已过期：`LICENSE_EXPIRED`
- 宽限期：`LICENSE_GRACE`
- 非法授权：`LICENSE_INVALID`

授权状态由 `LicenseManager` 提供，结果由 `GatewayAuthGlobalFilter` 返回统一 `R` 响应。

## 四、路由与部署注意事项

根据当前配置，网关已存在对以下业务域的路由治理：

- `api-sys`
- `api-basic`
- `api-files`
- `api-wf`
- `api-app`
- `api-integration`

并已使用：

- `RequestRateLimiter`
- `CircuitBreaker`

因此部署文档和环境配置应默认把网关视为所有 Web/移动端入口，而不是绕过网关直连业务服务。
