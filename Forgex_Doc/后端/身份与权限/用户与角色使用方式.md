# 用户与角色使用方式

> 分类：后端 / 身份与权限
> 版本：**V0.6.5**
> 更新时间：**2026-04-17**

本文档说明用户、角色相关能力在日常开发和联调中的推荐用法。

## 适用场景

- 新增用户管理接口
- 做角色授权、菜单授权
- 调整默认密码、重置密码逻辑
- 排查用户无菜单、无按钮、无租户权限问题

## 推荐实践

1. 用户新增、编辑、启停都走统一 Service，不要在多个模块重复维护账号状态。
2. 角色授权完成后，注意同步刷新当前登录用户权限或要求重新登录验证。
3. 默认密码、密码重置口径统一受配置控制，不在业务模块中硬编码。
4. 涉及跨租户查用户时，要明确是否忽略租户隔离。

## 新增用户接口示例

```java
@RequirePerm("sys:user:create")
@PostMapping("/create")
public R<Boolean> create(@RequestBody SysUserDTO dto) {
    userService.create(dto);
    return R.ok(CommonPrompt.SAVE_SUCCESS);
}
```

## 角色授权注意点

- 菜单权限与按钮权限需要一起维护。
- 角色删除前要检查是否仍有用户绑定。
- 数据权限范围调整后，要联动验证列表查询结果。

## 常见排查

| 现象 | 排查点 |
|---|---|
| 用户能登录但没有菜单 | `sys_user_role`、`sys_role_menu`、当前租户 |
| 修改角色后前端无变化 | 重新登录、刷新权限缓存 |
| 重置密码后无法登录 | 密码算法、传输加密、公钥配置 |
| 用户查不到所属租户数据 | `sys_user_tenant`、默认租户、TenantContext |
