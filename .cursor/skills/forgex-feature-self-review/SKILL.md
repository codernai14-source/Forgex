# Forgex 功能自审查

## 一、何时使用

在以下场景完成后应用此技能：

- **功能开发完成**: 新增功能、API 接口、页面组件
- **Bug 修复**: 修复问题、回归修复
- **代码重构**: 重构代码、优化性能
- **API 变更**: 接口参数变更、响应结构变更
- **行为变更**: 对话框行为、交互逻辑变更
- **配置变更**: 数据库字段、Nacos 配置变更

## 二、自审查检查清单

### 2.1 代码质量检查

- [ ] 代码符合命名规范
- [ ] 注释完整（类、方法、字段）
- [ ] 分层结构正确
- [ ] 异常处理合理
- [ ] 事务注解使用正确
- [ ] 权限注解配置正确

### 2.2 功能完整性检查

- [ ] API 接口与前端调用匹配
- [ ] 请求参数和响应格式正确
- [ ] 错误处理覆盖所有异常场景
- [ ] 国际化文本完整

### 2.3 文档同步检查

**核心原则**：
- ✅ **必须更新功能文档**：确保文档与代码一致
- ✅ **保持文档版本不变**：除非用户明确要求
- ✅ **更新实现逻辑文档**：技术实现变更时
- ✅ **更新使用方式文档**：接入方式变更时
- ❌ **不要修改文档版本号**：除非用户明确要求

### 2.4 数据库检查

- [ ] 表结构变更有对应的 SQL 脚本
- [ ] SQL 脚本命名符合规范
- [ ] 审计字段完整
- [ ] 索引设计合理

### 2.5 前端检查

- [ ] 组件使用公共组件
- [ ] 国际化文本完整
- [ ] 权限按钮配置正确
- [ ] 表格列配置正确

## 三、文档同步流程

### 3.1 识别需要更新的文档

根据代码变更类型，确定需要更新的文档：

| 变更类型 | 需要更新的文档 |
|---------|--------------|
| 新增 API 接口 | 后端使用方式文档、前端 API 对接文档 |
| 修改 API 参数 | 后端使用方式文档、前端 API 对接文档 |
| 新增前端组件 | 前端使用方式文档、组件实现逻辑文档 |
| 修改业务逻辑 | 实现逻辑文档、使用方式文档 |
| 修改数据库结构 | 数据库脚本与修复文档 |
| 修改配置 | 部署文档、Nacos 配置文档 |

### 3.2 更新文档内容

**步骤**：

1. **定位文档路径**: 在 `Forgex_Doc/` 目录下找到对应文档
2. **对比代码变更**: 识别变更点和影响范围
3. **更新实现逻辑**: 修改技术实现描述、代码示例、流程图
4. **更新使用方式**: 修改接入示例、参数说明、返回值说明
5. **检查关联文档**: 确保关联文档也同步更新
6. **保持版本不变**: 不要修改版本号字段

### 3.3 文档更新示例

**变更**: 用户 API 新增 `batchCreate` 接口

**需要更新的文档**:
- `Forgex_Doc/后端/身份与权限/用户与角色使用方式.md`
- `Forgex_Doc/后端/身份与权限/用户与角色实现逻辑.md`

**更新内容**:
```markdown
## 批量创建用户

### 使用方式

```java
@PostMapping("/batch-create")
public R<Void> batchCreate(@RequestBody List<SysUserParam> params) {
    userService.batchCreate(params);
    return R.ok();
}
```

### 参数说明

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| params | List<SysUserParam> | 是 | 用户参数列表 |

### 实现逻辑

1. 校验参数列表非空
2. 批量校验用户名唯一性
3. 批量创建用户（事务）
4. 返回成功结果
```

### 3.4 文档版本管理

**重要规则**：

1. **默认不改版本号**: 功能更新只更新文档内容，不修改版本号
2. **用户明确要求**: 仅当用户明确要求更新版本号时，才修改版本号
3. **版本号格式**: `V{major}.{minor}.{patch}`
4. **版本号位置**: 文档开头的 YAML frontmatter 或标题下方

## 四、常见问题排查

### 4.1 文档与代码不一致

**症状**: 文档描述的接口/参数/返回值与实际代码不匹配

**解决**:
1. 对比代码和文档
2. 更新文档使其与代码一致
3. 确保不修改版本号

### 4.2 遗漏更新关联文档

**症状**: 更新了主文档，但关联文档未同步更新

**解决**:
1. 检查文档中的关联阅读链接
2. 确认所有相关文档都已更新
3. 使用模块文档映射表检查

### 4.3 文档版本误修改

**症状**: 功能更新时误修改了文档版本号

**解决**:
1. 检查 Git 历史
2. 回退版本号修改
3. 保持内容更新

## 五、最佳实践

### 5.1 文档更新时机

- **功能开发完成后立即更新**: 避免遗忘
- **一次变更一次更新**: 不要批量更新
- **代码审查时检查文档**: 确保 PR 包含文档更新

### 5.2 文档更新质量

- **准确性优先**: 确保文档描述准确
- **示例代码可执行**: 提供的示例代码应该可以直接运行
- **图文结合**: 复杂流程配合流程图或截图
- **保持简洁**: 删除过时和无用的内容

### 5.3 文档维护原则

1. **代码为准**: 当代码和文档冲突时，以代码为准
2. **及时同步**: 代码变更后立即更新文档
3. **保持简洁**: 文档应该简洁明了
4. **版本控制**: 通过 Git 管理文档变更历史
